# Content Plans

这个目录用于保存各栏目内部规划文件，不属于 VitePress 站点内容，不会直接展示在网站上。

目前每个栏目各有一份规划说明，方便后续逐步补充内容。

建议这些规划文件至少包含下面几类信息：

- 栏目定位：这一栏解决什么问题，不解决什么问题
- 内容结构：章节、模块、问题列表或专题列表
- 状态标记：哪些是 `idea`、`outline`、`draft`、`published`
- 资源计划：哪些页面需要图片、图表、示意图或分享图

推荐把规划文件当成“结构数据”而不是随手笔记：

- 目录结构尽量和最终站点 slug 对齐
- 问题标题尽量在规划阶段就固定下来
- 准备做脚本生成的栏目，优先在这里维护结构源，而不是手写侧边栏

如果某个栏目后续要做半自动生成，优先在这里沉淀：

- 章节名
- 模块名
- 页面 slug
- 页面状态
- 是否需要配图
