# 某小厂 Agent 面经分享

这组题目覆盖 Agent 基础、项目拆解、RAG、记忆、规划、权限、多 Agent 协作与知识库工程化，问题链比较完整，适合拿来做模拟面试。

## 基本信息

- 方向：AI Agent
- 轮次：面经整理
- 面试公司：某小厂
- 面试时间：2026 年 3 月 29 日

## 题目清单

1. 实习挖掘了什么？
2. 项目样打？
3. 请具体介绍这两个项目的工作流以及用到的技术栈。
4. 数据清洗这块跑出来哪些类？这些步骤起到什么作用？
5. 微调数据集的结构是怎样的？
6. 微调有哪几种方法？它们有什么区别？
7. 对于各种文档，你们做了哪些处理方式？每种处理方式有什么优缺点？
8. 你们采用了子块和父文档关联的方式，这种方式在实际应用中的优点和缺点是什么？
9. 如果一个包有多级标题（如 1 级、2 级、3 级）的父子树，块粒度一个大类可能爬取下所有子类内容都带出来，你会怎么设计？
10. 在构建知识库进行检索时，可以涉及哪些算法？为了提高召回率，多路召回会涉及哪些方法？
11. 在多路混合检索时，如何平衡不同检索方法之间的权重？
12. 知识库与模型召回，如何评估它的效果和能力？具体指标如何衡量？
13. 在检索阶段，我们通过某种方法提高了召回率，但召了许多低质片段。在报文图片质量低估太粗糙时，我们是否会给模型吗？还是有什么方式可以提高模型的推理效果？
14. 在设计智能体时，你是如何进行不同类型智能体的划分？智能体之间如何协作？它们的记忆如何管理？
15. 多智能体是如何协同的？智能体之间的流转状态是如何管理的？
16. 你们的实现有没有依赖 LangGraph 这类框架，是基于什么考虑？
17. 如何对智能体进行权限控制？
18. MCP 有哪几种协议？它们之间有什么区别？

## 这组题主要考什么

- 前半段集中在项目经历拆解，要求你能把做过的 Agent 或 RAG 项目讲到工作流、数据处理和模型调优层面。
- 中段重点考知识库构建、检索召回、效果评估和低质上下文处理，比较偏工程落地。
- 后半段转向智能体设计，包括记忆体系、状态流转、多 Agent 协作、框架选型、MCP 和权限控制。

## 练习建议

- 把第 3 到第 13 题串起来，当成一套完整的 RAG 项目复盘来回答。
- 把第 14 到第 18 题单独整理成智能体系统设计题，重点准备架构图、状态机和权限边界。
- 第 2 题和第 13 题原截图文字不够清晰，后续如果你拿到更高清版本，建议再精修一遍题面。
