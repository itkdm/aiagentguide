# 1.2 RAG 系统的基本组成

这部分会从系统视角拆开 RAG 的核心组成，带你理解一个最小可用系统和一个生产级系统之间的差别。

前面在“定义与定位”里，你会知道 RAG 本质上是“先找资料，再基于资料回答”。而这一节要进一步回答的是：

**这件事在系统里到底是由哪些模块共同完成的？**

如果不把系统组成拆开看，学习 RAG 时很容易出现两种问题：

- 把 RAG 理解成一个单点能力，好像只要“接个知识库”就行
- 把所有问题都归咎给模型，而忽略前面数据、检索、排序和上下文构造的作用

这一节的目标，就是把这条链路从“抽象概念”变成“可以逐层分析的系统”。

## 这一节会回答什么问题

- [一个最小可用的 RAG 系统由哪些部分组成？](./q01-rag)
- [一个生产级 RAG 系统相比 Demo 多了哪些能力？](./q02-rag-demo)
- [RAG 的完整工作流程是什么？](./q03-rag)
- [从用户提问到模型回答，中间数据流是怎么走的？](./q04-item)

## 学这一节时最该带着的问题

1. 一个能跑的 Demo，最少需要哪些模块？
2. 一个真正能上线的系统，除了“能回答”之外还缺什么？
3. 当回答效果不好时，应该往链路的哪一层去找原因？

带着这三个问题往下看，你会更容易理解为什么后面整本 RAG 教程会把数据、切块、检索、重排、评测、治理都单独展开。
