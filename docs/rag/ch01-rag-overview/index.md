# 第 1 章 RAG 全景图与核心认知

本章目标：建立对 RAG 的整体认知，先理解它是什么、适合解决什么问题、系统基本由哪些部分构成，再辨析常见误解和边界。

这一章最大的作用，是先把对 RAG 的预期建立正确。

很多后续学习成本，其实不是出在知识点太难，而是出在一开始把 RAG 看错了。比如：

- 把它看成一个模型能力，而不是一条系统链路
- 把它看成万能解法，而不是解决特定问题的方法
- 把它看成 Demo 级拼接，而不是需要完整治理的工程系统

所以这一章不会一上来就深入讲切块、索引和检索算法，而是先帮你把“这件事到底是什么、该怎么理解、容易误解什么”建立起来。

## 学完这一章后，你应该做到什么

- 能用自己的话解释 RAG 是什么，不再把它和搜索、知识库问答、Agent 混为一谈
- 能从系统视角理解 RAG 至少包含哪些关键环节
- 能识别几种最常见的错误预期，避免后面一上来就把优化方向走偏
