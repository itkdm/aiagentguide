# 7.2 Rerank 的作用与价值

这一节开始正式进入：

- 为什么很多 RAG 系统在初次召回之后，还要再加一层 `Rerank`

前一节已经讲清楚了：

- 召回到了不等于最后答得好
- 正确候选在池子里，也不等于会被优先使用

那么很自然的下一步就是：

- 既然候选顺序这么重要，系统能不能再有一层专门做“重新排序”

这就是 Rerank 的位置。

## 学这一节时，最值得先建立的判断

- Rerank 解决的是候选优先级问题，不是替代召回
- 它通常发生在初次召回之后，而不是之前
- Rerank 的价值常常体现在“把对的结果顶上来”，而不是“凭空补回完全没召到的结果”
- 不是所有系统都必须先上 Reranker，但很多一到复杂场景就会明显受益
