# 14.1.6 RAG 效果差时，为什么不能第一反应就怪模型？

先给结论：*RAG 的质量瓶颈通常不在模型，而在“数据质量 + 检索链路 + 评估与观测”。*

很多问题看似是“模型不给力”，但真正原因往往是：

- 文档噪声过多  
- 切块策略不合理  
- 召回不稳定  
- 没有评估与观测导致问题不可定位  

## 为什么模型常被“误伤”

模型输出是最后一步，所有前序错误都会集中体现在这里。  
如果你没有观测和评估，模型就成了“最容易背锅的环节”。citeturn1search1turn1search0

## 一个更可靠的定位顺序

出现效果问题时，先按顺序排查：

1. 数据是否正确、是否更新  
2. 切块是否破坏语义  
3. 召回是否稳定  
4. 评估集是否能复现问题  

只有当前几项都确认无误，才去考虑模型本身。

## 一句话总结

RAG 效果差时，先查数据和检索链路，再查模型。没有观测与评估，模型只会被误判。
