# 第 14 章 常见误区与设计原则总结

先给结论：*RAG 的真正难点不是“工具不够强”，而是“判断不够稳”。这一章把常见误区和设计原则压缩成一张可复用的判断框架。*

本章目标是帮你在实际项目里更快做出正确判断：  
哪些问题是“参数问题”，哪些是“结构问题”；  
哪些优化值得先做，哪些只是成本更高的噪声放大。
