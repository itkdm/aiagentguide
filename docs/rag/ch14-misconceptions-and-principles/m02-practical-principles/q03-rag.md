# 14.2.3 一个成熟 RAG 系统最重要的设计原则有哪些？

先给结论：*成熟 RAG 的核心原则是“可用、可控、可评测、可更新、可回退”。*

下面给出一组可以长期复用的原则清单。

## 1. 可观测优先

没有追踪和日志，问题就无法定位。  
把“每次检索与生成发生了什么”记录下来，是第一优先级。citeturn1search1

## 2. 评估先于优化

没有评估就没有优化。  
哪怕是最小评测集，也要先建立，再做调整。citeturn1search0

## 3. 证据优先于模型

答案质量的上限由证据质量决定。  
先保证召回稳定，再谈生成优化。

## 4. 更新与回退必须内置

数据与索引一定会变。没有更新与回退，系统最终会失效。citeturn1search1

## 5. 权限与范围控制必须前置

生产环境里权限与范围是刚性约束，不能后补。  
metadata/payload 过滤是最常见的落地方式。citeturn1search2

## 6. 复杂能力延后

重排、多路召回、多模态与 Agentic Retrieval 都有效，但必须建立在稳定链路之上。

## 一句话总结

成熟 RAG 的设计原则是“先可控，后高级”。可观测、可评测、可更新、可回退，是长期稳定的基础。
