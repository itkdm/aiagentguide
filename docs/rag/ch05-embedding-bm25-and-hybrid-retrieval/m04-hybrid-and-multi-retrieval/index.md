# 5.4 混合检索与多路召回

这一节要讨论的，是很多 RAG 系统从“能用”走向“更稳”时都会遇到的一步：

- 不再只依赖单一路召回
- 而是开始组合多种检索信号和多条召回路径

前面已经讲清楚了：

- 向量检索强在语义覆盖
- BM25 强在精确命中

那么很自然的下一步就是：

- 既然两者各有盲点，为什么不把它们一起用

这就是混合检索和多路召回的出发点。

## 学这一节时，最值得先建立的判断

- 混合检索不是“多加几层功能”，而是在补不同召回信号的盲区
- 多路召回不是一定更复杂才更好，而是为了降低单一路径的不稳定性
- 单一路召回最大的风险，不是完全没结果，而是“总有点像，但不够稳”
- 混合检索和多路召回的价值，往往体现在长尾问题、术语问题和边界复杂问题上
