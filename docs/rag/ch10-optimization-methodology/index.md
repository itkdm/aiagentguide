# 第 10 章 RAG 调优方法论

先给结论：**RAG 调优不是“哪个环节都动一点”，而是先找到主矛盾，再对准一层下手，最后用评测闭环验证是否真的改善。**

真实系统里，“效果不好”几乎从来不是一个点的问题。你需要一套方法来回答三个核心问题：

- 先查哪里
- 怎么取舍
- 如何验证

这就是本章的核心目标。

## 本章最值得先建立的判断

- 优化顺序比优化动作更重要
- 召回、准确率、成本、延迟往往相互牵动
- 常见优化抓手各有边界，不存在万能方案
- 没有闭环，优化很容易局部变好但整体变差
