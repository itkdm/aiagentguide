# 10.3 常见优化抓手

先给结论：**RAG 调优里没有“万能抓手”。`Chunk`、混合检索、`Reranker`、上下文压缩和复杂文档优化，分别作用在不同链路阶段。只有把这些抓手放回整条链路里理解，你才知道什么时候该优先动哪一个。**

很多团队在调优时，最容易出现的问题不是“知道得太少”，而是：

- 抓手知道很多
- 但不知道什么时候该用哪个

结果就是：

- 看见效果不好，就轮流把这些手段都试一遍

这会让调优变得又慢又乱。因为每种抓手真正擅长解决的问题并不一样。

## 学这一节时，最值得先建立的判断

- 同一个失败现象，可能对应完全不同的优化抓手
- 每个抓手都有更适合的问题类型和阶段边界
- 抓手越多，不代表系统越强；关键是是否命中当前主矛盾
- 把抓手放回链路理解，比记住手段名字更重要
