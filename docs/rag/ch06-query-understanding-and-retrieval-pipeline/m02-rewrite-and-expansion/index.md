# 6.2 Query Rewrite 与 Query Expansion

这一节要讨论的是查询理解里最常见、也最容易被混在一起的两类动作：

- `Query Rewrite`
- `Query Expansion`

它们看起来都像“把查询改一改”，但真正解决的问题并不完全一样。

可以先粗略理解成：

- `Rewrite` 更像把问题改得更清晰、更适合检索
- `Expansion` 更像给查询补更多可能有用的召回信号

如果这两层不分清，后面很容易出现两种常见问题：

- 改写太多，把用户真实意图改偏了
- 扩写太猛，把噪声和错误候选一起带进来了

## 学这一节时，最值得先建立的判断

- Query Rewrite 的重点是“重述意图”，不是简单换个说法
- Query Expansion 的重点是“补充召回信号”，不是无限扩词
- 多轮对话里，重写往往是先把问题补完整
- 对话历史如果处理不好，不是帮助检索，而是直接污染检索
