# 13.2.1 一个 Demo 版和生产版最大的差别是什么？

先给结论：*Demo 证明“能跑通”，生产证明“能长期稳定跑”。最大的差别不是功能，而是可观测、可评估、可更新。*citeturn0search1turn0search2turn1search3

## 三个核心差别

### 1. 可观测

Demo 阶段你只关心“有没有答案”，生产阶段你必须回答：

- 答案来自哪里  
- 检索质量是否波动  
- 哪一步出现了问题  

这要求你有**链路追踪、日志、召回结果可视化**等能力。citeturn0search1turn1search3

### 2. 可评估

Demo 常靠直觉判断“还不错”，生产必须有可重复的评估集和评估指标。没有评估就无法稳定迭代。citeturn0search2

### 3. 可更新

生产系统一定会遇到：

- 数据更新  
- 索引更新  
- 模型版本更新  

如果没有更新与回滚机制，系统只能“越来越乱”。citeturn0search2

## Demo 能容忍，生产不能容忍的事

- 偶发错误  
- 单点故障  
- 无法解释的结果波动  

这就是为什么“Demo 很顺”并不代表“生产可用”。

## 一个最小生产门槛

当你满足以下三点，才算跨过“Demo 到生产”的最低门槛：

- 有可观测链路（知道每次检索与生成发生了什么）  
- 有可重复评估（同一问题可稳定对比）  
- 有更新与回退机制（数据或模型出错可回滚）  

## 一句话总结

Demo 证明“能跑通”，生产证明“能稳定长期跑”。最大的差距是可观测、可评估、可更新。
