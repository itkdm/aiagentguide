# 13.3.3 一个务实的 RAG 演进路线应该是什么样？

先给结论：*路线应该是“闭环 -> 可控 -> 可扩展 -> 高级形态”。每一阶段都必须能稳定运行，再进入下一阶段。*

下面给出一个 4 阶段的务实路线图。

## 阶段 0：最小闭环

目标：从文档到答案跑通。  
关键动作：

- 小规模高质量数据  
- 固定问题集  
- 简单检索 + 生成  

## 阶段 1：可控性

目标：能定位问题、能稳定迭代。  
关键动作：

- 最小评估集  
- 召回与生成的日志与追踪  
- 基础过滤  

评估与追踪是稳定迭代的核心能力。citeturn1search0turn1search1

## 阶段 2：可扩展

目标：能处理更多数据与更多场景。  
关键动作：

- 增量更新与索引重建  
- 多数据源接入  
- 简单混合检索  

混合检索通常在这个阶段引入，需要配合评估。citeturn0search2

## 阶段 3：高级形态

目标：提升上限与复杂任务覆盖。  
关键动作：

- 重排、多路召回  
- Agentic Retrieval  
- 多模态扩展  

## 一句话总结

务实路线的关键是“每一步可控、可复现”。先闭环，再扩展，最后再冲上限。
