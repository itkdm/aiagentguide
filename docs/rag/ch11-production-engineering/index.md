# 第 11 章 生产级 RAG 工程实践

先给结论：**生产级 RAG 的核心难点不在“能不能答”，而在“能不能稳定答、可控答、可回退答”。主链路只是基础，工程能力决定上线成败。**

这一章的目标是帮助你建立生产视角：一个能上线的 RAG 系统需要哪些支撑能力，哪些风险必须提前防线化。

## 本章最值得先建立的判断

- 工程能力和主链路同等重要
- 缓存、并发控制、降级策略是上线门槛
- 没有日志和观测，调优只能靠猜
- 灰度与回滚不是运维手段，而是系统能力

读完这一章后，你最好能更稳地判断：

- 哪些工程能力是上线必需项，哪些可以后置
