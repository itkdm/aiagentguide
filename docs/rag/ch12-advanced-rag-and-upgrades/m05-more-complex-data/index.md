# 12.5 为了解决“数据形态更复杂”

先给结论：*当数据不再只是“长文本”，RAG 的问题就不再是“分块 + 相似度”，而是“如何让不同形态的数据都能被检索和解释”。*

这一节聚焦三类复杂数据：

- 结构化数据（数据库、指标、表格）  
- 多模态数据（图片、图表、视频）  
- 半结构化数据（表格 + 文本、代码 + 文档）  

## 学这一节时，最值得先建立的判断

- 结构化数据不适合“纯向量检索”，必须保留结构化查询能力。  
- 多模态数据的难点不是“看得见”，而是“检索到 + 解释对”。  
- 复杂数据形态的核心不是“更多 embedding”，而是“保留结构 + 建立对齐”。
