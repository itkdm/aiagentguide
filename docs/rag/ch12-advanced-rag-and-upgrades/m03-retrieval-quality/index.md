# 12.3 为了解决“检索质量不稳定”

先给结论：*当检索质量不稳定时，问题通常不是“检索能力不够”，而是“缺少检索结果自检与纠错”。纠错型 RAG 和 Self-RAG 的核心价值，是把“自检 + 纠错”变成可执行流程。*

这一节会把“检索质量波动”拆成三个层面：

- 检索结果是否相关
- 证据是否足够支撑答案
- 生成是否需要进入二次检索

你会看到两类思路：

- **纠错型 RAG（CRAG）**：先评估检索结果质量，再触发纠错动作  
- **Self-RAG**：把“是否检索、是否支持、是否使用”内化为模型可预测的反思信号  

## 学这一节时，最值得先建立的判断

- 质量不稳定的根因通常是“检索质量不可控”，而不是“模型能力不足”。
- 自检不是“再问一遍”，而是“对检索结果和证据做结构化判断”。
- 如果没有明确的自检信号与纠错策略，检索质量不稳定就无法被系统性修复。
