# 12.2 为了解决“大文档与复杂结构”

先给结论：*当文档规模很大或结构复杂时，问题往往不是“chunk 不够小”，而是“结构被丢失”。你需要用分层检索和递归检索把结构找回来，再让检索在结构里工作，而不是在碎片里盲选。*

这一节聚焦的不是“调参数”，而是“改结构”。核心是两件事：

- 用层级结构表达文档，让检索可以先定位到“正确的层”，再进入细节
- 用递归或多阶段检索，把大问题拆成多次、有限范围的检索

## 学这一节时，最值得先建立的判断

- 大文档的问题不是“信息太多”，而是“信息在层级里”，结构丢了就会检索错层。
- 当问题需要“先定位章节，再找具体条款”时，单次 top-k chunk 会天然失效。
- 分层检索和递归检索不是更复杂的花活，而是大文档场景下的基本结构。
