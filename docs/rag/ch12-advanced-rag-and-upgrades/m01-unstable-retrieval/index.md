# 12.1 为了解决“检索不稳”

先给结论：**检索不稳不是“偶发噪声”，而是检索路径对问题类型和数据形态不匹配的表现。要稳定检索，需要的不是只调 `TopK`，而是建立多路召回、路由和分层的结构。**

很多系统会出现下面这些现象：

- 同类问题有时答对、有时答错
- 一换问法就召不回关键证据
- 某些问题类型稳定性明显更差

这些现象往往说明：

- 单一路径无法覆盖所有问题类型

这一节就是从“结构升级”的角度，讲清几种专门用来对抗检索不稳的方案。

## 学这一节时，最值得先建立的判断

- 检索不稳往往是“路径不匹配”，不是“参数没调好”
- 多路召回是为了补盲，不是为了堆复杂度
- 路由检索的价值在于：不同问题走不同检索路径
