# 12.4 为了解决“任务不只是问答”

先给结论：*当问题从“回答”变成“完成任务”，RAG 只是一个能力模块，真正的主流程必须由 Agent 来负责规划与执行。*

这节不是讲“更大的模型”，而是讲“更强的流程”。你会看到：

- **RAG 的边界**：它擅长“找证据”，不擅长“做事情”  
- **Agent 的角色**：负责分解任务、选择工具、组织步骤  
- **协同的关键**：把检索变成工具，不让检索绑架流程  

## 学这一节时，最值得先建立的判断

- 只要任务需要“多步行动 + 工具调用”，就不再是纯 RAG 问题。  
- Agent 不等于“更复杂”，它是“让流程可控”的结构。  
- 最常见的失败不是模型不够聪明，而是“没有流程”。
