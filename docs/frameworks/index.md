# 框架

这一栏关注的是框架选型，而不是单纯罗列名字。不同框架背后的设计重点并不一样，有的强调状态和流程控制，有的强调多 Agent 协作，有的更偏平台化和可视化交付。

## 你会在这里看到什么

- 主流 AI Agent 框架的定位差异
- 适用场景与学习门槛
- 代码驱动和平台驱动路线的区别
- 框架选型时最容易忽视的判断点

## 这一栏适合什么时候看

- 已经理解 Agent 基本概念，准备开始做技术选型
- 想知道不同框架到底差别在哪里
- 想避免上来就盲目跟风某一个热门框架

## 阅读建议

这一栏更适合带着目标来看。先想清楚自己要做的是单 Agent、多 Agent、工作流编排，还是平台型应用，再来看框架差异，会比“从名字倒推用途”更有效。
