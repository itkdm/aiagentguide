# 框架

这一栏既关心“怎么选框架”，也关心“框架具体怎么用”。不同框架背后的重点并不一样，有的强调快速起步，有的强调状态与流程控制，有的更偏多 Agent 协作，也有的更偏企业集成和平台化。

## 推荐先读

- [如何选择合适的 AI Agent 框架](/frameworks/how-to-choose-agent-framework)

## 已收录框架

- [LangChain](/frameworks/langchain/)

## 你会在这里看到什么

- 主流 AI Agent 框架的定位差异
- 适用场景与学习门槛
- 代码驱动和平台驱动路线的区别
- 框架选型时最容易被忽视的判断点
- 各个框架的基础使用说明与能力拆解
- 围绕具体框架的进阶能力、最佳实践和实战示例

## 这一栏适合什么时候看

- 已经理解 Agent 基本概念，准备开始做技术选型
- 想知道不同框架到底差别在哪里
- 已经确定某个框架，想系统查看它的使用文档
- 想避免一开始就盲目跟风某个热门框架

## 阅读建议

更推荐先从选型文章建立判断标准，再进入具体框架文档。对于 Java 团队，也建议优先比较通用开发框架、Spring 集成层和更偏 Agent 运行时的方案，而不是默认把 AI 部分拆去 Python。
