# Test

Agent 应用会让模型自行决定下一步动作，因此灵活性很强，但也更难预测一次小改动会怎样影响整体行为。要把 Agent 用到生产环境，测试不能省略。

常见测试方式有三类：

- **单元测试**：隔离测试小而确定的逻辑，用内存假对象替代真实模型，速度快、成本低、结果稳定
- **集成测试**：调用真实模型 API 和外部服务，验证组件之间是否正确协作
- **评估（Evals）**：对 Agent 的执行轨迹进行打分，可以用规则匹配，也可以用 LLM 作为评审

相比传统软件，Agent 应用通常更依赖集成测试，因为它往往会串联多个组件，而且 LLM 输出本身就存在非确定性。

## 三种测试方式的区别

| 类型 | 适用场景 | 主要目标 |
| --- | --- | --- |
| 单元测试 | 测某个函数、工具调用流程、状态逻辑 | 快速验证精确行为 |
| 集成测试 | 测真实模型、真实工具、真实网络调用 | 验证系统能否联动工作 |
| 评估 | 测轨迹质量、推理质量、行为是否退化 | 发现回归与质量变化 |

## 建议的测试顺序

1. 先写单元测试，确保 Agent 的局部逻辑正确
2. 再写集成测试，确认真实模型与外部依赖能正常工作
3. 最后补评估，用于衡量复杂行为质量和版本回归
