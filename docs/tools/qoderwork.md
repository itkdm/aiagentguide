## QoderWork 是什么？

<div class="tool-prose-card">
  <p>QoderWork 是一款桌面端智能工作助手。官方文档说明，它将 Qoder 的 Agent 能力从代码场景扩展到日常工作场景，用户可以通过自然语言完成文件整理、数据处理、文档生成等任务；官网则强调其“本地优先、自主规划、可控执行”的产品定位。</p>
  <p>和普通聊天式 AI 工具相比，QoderWork 更强调直接接任务并交付结果，而不是只输出建议文本。它适合需要处理本地文件、生成办公交付物和调用扩展能力的人，整体更接近一个桌面端通用 Agent 助手。</p>
</div>

## 主要功能

<div class="tool-feature-grid">
  <article class="tool-feature-card">
    <h3>自主规划执行</h3>
    <p>能够理解复杂任务意图，自动拆解步骤，并在执行过程中与用户沟通确认结果，不只是被动回答问题。</p>
  </article>
  <article class="tool-feature-card">
    <h3>内置浏览器自动化</h3>
    <p>官方页面明确写到可自动浏览网站、抓取信息，帮助完成研究、资料整理等任务。</p>
  </article>
  <article class="tool-feature-card">
    <h3>多格式结果输出</h3>
    <p>可直接生成 Word、Excel、PPT、PDF 等交付物，并支持 AI 图像生成，适合日常办公场景。</p>
  </article>
  <article class="tool-feature-card">
    <h3>可扩展能力</h3>
    <p>支持内置 MCP 工具与自定义 Skills，用于扩展专属工作流和任务场景。</p>
  </article>
</div>

## 如何使用？

<div class="tool-step-list">
  <article class="tool-step-card">
    <span class="tool-step-no">01</span>
    <h3>下载桌面应用</h3>
    <p>访问 QoderWork 官网下载桌面端应用。官方当前标注支持 macOS 14+ 和 Windows 10+。</p>
  </article>
  <article class="tool-step-card">
    <span class="tool-step-no">02</span>
    <h3>登录账号并描述任务</h3>
    <p>登录 Qoder 账号后，在应用中通过自然语言描述需求，让 QoderWork 自主规划和执行任务。</p>
  </article>
  <article class="tool-step-card">
    <span class="tool-step-no">03</span>
    <h3>接入 Skills 或 MCP</h3>
    <p>根据任务类型接入内置扩展能力、MCP 工具或自定义 Skills，形成更贴合个人工作的执行流程。</p>
  </article>
  <article class="tool-step-card">
    <span class="tool-step-no">04</span>
    <h3>在授权目录中处理文件</h3>
    <p>在授权目录范围内处理本地文件、生成交付结果，并按需导出 Word、Excel、PPT、PDF 等产物。</p>
  </article>
</div>

## 价格与套餐

<div class="tool-prose-card">
  <p>官方文档明确写明：QoderWork 支持 Qoder 账号登录，并与 Qoder 账号共享 Credits。Qoder 当前个人方案包括 <code>Free</code>、<code>Pro</code>、<code>Pro+</code>、<code>Ultra</code> 四档，公开价格页显示限时优惠价分别为 <code>Pro 10 美元/月</code>、<code>Pro+ 30 美元/月</code>、<code>Ultra 100 美元/月</code>，Free 为免费。</p>
  <p>不过，官方并没有单独给出 QoderWork 独立套餐价格表。因此如果问题是“QoderWork 是否有自己单独的订阅价”，当前更准确的表述是：官网暂未明确说明，实际计费主要依附于 Qoder 账号和 Credits 体系。</p>
</div>

## 补充说明

<div class="tool-note-list">
  <article class="tool-note-card">
    <h3>文件操作偏本地执行</h3>
    <p>官方文档明确说明，QoderWork 的文件操作在本地完成，但为了实现 AI 理解和分析，相关文本内容会发送给大模型 API 服务商。</p>
  </article>
  <article class="tool-note-card">
    <h3>与 Qoder 账号体系打通</h3>
    <p>QoderWork 不强调独立收费，而是直接复用 Qoder 账号、Credits 和计费体系，因此更适合作为 Qoder 产品能力的桌面延伸。</p>
  </article>
  <article class="tool-note-card">
    <h3>独立博客未明确区分</h3>
    <p>Qoder 官网存在统一官方 Blog 入口，但当前没有看到一个明确独立命名为“QoderWork 官方博客”的栏目，因此若特指 QoderWork 独立博客，更适合记为官网暂未明确说明。</p>
  </article>
</div>
