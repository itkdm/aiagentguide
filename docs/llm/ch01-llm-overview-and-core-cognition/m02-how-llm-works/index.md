# 1.2 LLM 的工作本质

这部分会带你理解 LLM 最核心的工作方式，先看它为什么常被概括成“下一个 token 预测”，再看这种训练目标为什么会进一步涌现出更复杂的能力。

这一节最大的价值，是把一个非常容易被误解的问题讲清楚：

- 说 LLM 是“下一个 token 预测”，不是在贬低它
- 说它“不止于下一个 token 预测”，也不是在否定这个基础定义
- 模型表现出来的理解、推理、总结、翻译能力，和这个训练目标并不冲突

很多人会在这里产生两个极端理解：

- 要么觉得“既然只是预测下一个词，那它根本不可能真的有复杂能力”
- 要么觉得“既然它能做复杂任务，那一定不是在做 token 预测”

这两个判断都不准确。更合理的理解是：**下一个 token 预测是底层机制，复杂能力是这个机制在大规模训练和上下文条件下表现出来的结果。**

## 学完这一节后，你应该建立的几个判断

- 当你说“LLM 在预测下一个 token”时，你说的是模型底层推理过程，不是用户视角下的任务描述
- 当你看到模型会问答、翻译、写代码时，不应该觉得这和 token 预测矛盾，而应该理解这些任务都能在语言序列建模里统一表示
- 当你看到模型输出像在“思考”，要知道那更接近统计建模和上下文推演的结果，而不是人类意义上的意识活动
