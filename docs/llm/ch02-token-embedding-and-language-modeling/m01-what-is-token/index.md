# 2.1 Token 是什么

这部分会先讲清楚 token 是什么、它和字词子词的关系，以及为什么 token 数量会直接影响上下文长度、成本和输出质量。

这一节非常基础，但也非常重要。很多后面看起来复杂的问题，其实都和 token 有关，比如：

- 为什么同样长度的中文和英文，token 数可能差很多
- 为什么一段 Prompt 明明不长，却很快就把上下文窗口占满了
- 为什么模型计费常常按 token 算，而不是按字数算
- 为什么有些输出质量问题，本质上是上下文被 token 挤爆了

如果这里没有建立准确理解，后面看 embedding、上下文窗口、长文本处理、RAG、Prompt 优化时，就很容易一直用错单位。

## 学完这一节后，你应该建立的几个判断

- token 不是“词”的同义词，而是模型处理文本时使用的离散单位
- token 的切分方式由 tokenizer 决定，不同模型可能切得不一样
- Prompt、上下文窗口、价格、延迟、长文处理，很多时候本质上都是 token 预算问题
