# 2.2.2 为什么模型需要把 token 映射成向量表示？

先给结论：

**因为 token id 只是离散标签，不具备可学习的几何关系；把 token 映射成向量后，模型才能在连续空间里学习相似性、组合模式和上下文关系。**

如果不做向量表示，现代神经语言模型几乎没法有效工作。

## token id 为什么不够用

假设有三个 token：

```text
猫 -> 102
狗 -> 874
汽车 -> 5531
```

如果你只看这些 id，模型并不能知道：

- 猫和狗在语义上更接近
- 猫和汽车差得更远
- 102 和 874 的数字距离没有语言意义

也就是说，离散编号只能区分“不是同一个 token”，却不能表达“有多像、在哪些方面像”。

而语言建模最需要的，恰恰就是这种可泛化的关系表示。

## 向量表示能多做什么

当 token 被映射成向量后，模型就可以在连续空间里学习很多模式，例如：

- 哪些 token 常出现在相似上下文
- 哪些 token 在语义或功能上更接近
- 哪些组合会形成特定表达结构
- 某个 token 在不同上下文中该如何调整表示

这对语言建模非常关键。因为语言不是“查表题”，而是高度依赖泛化和组合。

## 为什么连续空间更适合泛化

如果两个 token 在向量空间里足够接近，模型就更容易把某些学到的模式迁移过去。

例如，模型在大量文本里学到：

- “猫”经常和“宠物”“喂食”“毛发”这类上下文有关
- “狗”也经常出现在类似上下文里

那么它就可能在向量空间中把两者的某些表示模式拉近。

这样做的价值是：

- 即使某个具体句子没见过
- 模型也可能通过相似模式做出合理判断

这就是表示学习相对于纯规则或纯离散记忆的重要优势之一。

## one-hot 为什么不够

理论上，你当然也可以把每个 token 表示成 one-hot 向量。

但 one-hot 的问题很明显：

- 维度极高
- 非常稀疏
- 任意两个 token 之间几乎同样“远”
- 很难自然表达相似性结构

而 embedding 的优势是：

- 维度更低
- 表示更紧凑
- 相似模式可以在空间里形成结构
- 更适合后续矩阵计算和深层网络建模

所以从工程和学习效率上看，embedding 几乎是现代神经语言模型的基础选择。

## 为什么“向量”比“词典解释”更适合模型

人类喜欢用概念和定义理解语言，比如：

- “猫是一种动物”
- “银行有金融机构和河岸两个含义”

但模型不是靠人类式词典条目工作。它更适合通过大量上下文样本，在向量空间里自己学出：

- 词形相近关系
- 语义相近关系
- 功能角色关系
- 上下文依赖变化

这也是为什么向量表示虽然不直观，却特别适合机器学习。

## 一个简单直觉：向量空间像“可计算的语义地形”

你可以把向量空间粗略想成一张不是给人看、而是给模型算的地图。

在这张地图上：

- 某些 token 更接近
- 某些 token 更容易被同一类模式影响
- 某些方向变化可能对应某类语义或结构变化

这张地图不要求每个坐标轴都有人类可解释名称，但只要它能让模型更容易学到规律，它就是有价值的表示。

## 这和上下文化表示有什么关系

这里还要区分两层：

- `静态 embedding`：token 初始查表得到的向量
- `上下文化表示`：经过模型多层计算后，结合上下文更新过的表示

初始 embedding 提供起点，但真正支撑理解和生成的，往往是后面不断被上下文改写过的表示。

所以 embedding 很重要，但它不是全部。更准确地说，它是整个表示学习链条的起点。

## 你可以先这样记住

- token id 只能区分身份，不能表达相似性
- 向量表示让模型能在连续空间里学习关系和模式
- embedding 解决的是“怎么把离散符号变成可学习表示”
- 后续更复杂的语义理解，则建立在这些表示继续被上下文化的基础上
