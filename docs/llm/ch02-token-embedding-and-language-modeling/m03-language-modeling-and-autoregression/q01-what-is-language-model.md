# 2.3.1 什么是语言模型（Language Model）？

先给结论：

**语言模型，本质上是一个对语言序列概率分布进行建模的系统。它的核心问题是：在给定前文的条件下，后续 token 出现的概率有多大。**

所以“语言模型”这个名字虽然看起来很宽泛，但在技术上有一个相当明确的含义：

- 它不是在背词典
- 也不是先理解“句意摘要”再回答
- 而是在学习语言序列怎样更可能出现

## 什么叫“对语言序列建模”

假设有一句话：

```text
今天的会议已经
```

语言模型要做的事情，就是判断在这个前文下：

- “结束”是不是高概率后续
- “开始”是不是相对低概率后续
- 标点、语气词、结构延续分别大概有多可能

也就是说，它不是只给一个唯一答案，而是在整个词表上形成一个条件概率分布。

## 为什么这叫“语言模型”

因为它在建模语言的统计规律和结构规律。

这里的“模型”不是泛泛地指一个 AI 程序，而是更具体地指：

**一个能估计语言序列概率的参数化系统。**

如果写得更抽象一点，语言模型在做的是：

```text
P(token_t | token_1, token_2, ..., token_{t-1})
```

也就是：

给定前面所有 token，预测当前 token 的概率。

一个完整句子、段落、回答，最终都可以被拆成很多这样的条件预测。

## 语言模型为什么重要

因为大量语言任务都可以转化为“给定上下文后生成合理后续”。

例如：

- 问答：根据问题生成答案
- 翻译：根据源语言和指令生成目标语言文本
- 总结：根据原文生成压缩后的表达
- 代码生成：根据需求和已有代码生成后续代码

所以语言模型不是很多任务中的一个组件，而是整个 LLM 能力体系的根基。

## 一个最小直观例子

假设输入是：

```text
法国的首都是
```

语言模型会对词表里很多候选 token 做评分，例如：

- 巴黎：高概率
- 伦敦：低一些
- 苹果：很低
- 因为：很低

这不是因为模型“知道一道问答题答案”，而是因为它在语言建模过程中学到了：

- 国家与首都的关系
- 这类句式通常怎么续写
- 在这个上下文下什么后续最合理

## 语言模型和聊天机器人是什么关系

聊天机器人只是语言模型的一种应用形态，不是语言模型的定义本身。

如果没有后续的：

- 指令微调
- 对齐训练
- 系统提示词
- 工具调用
- 检索增强

一个语言模型本质上仍然只是一个“根据上下文继续生成”的系统。

所以先理解语言模型，再理解聊天、Agent、RAG，认知顺序会更稳。

## 为什么“建模概率”能长出复杂能力

很多人会觉得：只是预测概率，为什么最后能做这么多任务？

原因在于，语言里本来就包含：

- 事实知识
- 表达模式
- 推理轨迹
- 问答结构
- 代码结构

当模型不断学习这些条件概率关系时，它实际上也在吸收大量更高层的模式。

所以语言模型看起来像在做“补全”，但真正学到的远不止局部补词。

## 你可以先这样记住

- 语言模型的核心任务，是估计语言序列的条件概率
- 它的基本单位是 token，而不是抽象句意
- 很多高层语言任务，都可以统一成语言建模问题
- LLM 本质上仍然是语言模型的扩展形态
