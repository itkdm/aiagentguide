# 2.3.2 什么是自回归生成（Autoregressive Generation）？

先给结论：**自回归生成，就是模型在推理时每次只预测下一个 token，并把刚生成的结果再接回上下文，继续预测后面的 token。**

“自回归”这个词听起来有点数学化，但对 LLM 来说，可以先把它理解成一句更直白的话：

**模型一边生成，一边把自己刚写出来的内容继续当作后续输入。**

这就是今天大量 GPT 类模型和聊天模型的核心生成方式。

## 为什么叫“自回归”

“回归”这里可以粗略理解成：

- 用已有值去预测后续值

“自回归”则表示：

- 后续预测依赖于序列自身已经生成出来的部分

放到 LLM 里，就是：

1. 先看用户输入和已有上下文
2. 生成第一个 token
3. 再把这个 token 也算进上下文
4. 基于更新后的上下文再预测第二个 token
5. 如此循环

所以“自”这个字，强调的是：

- 模型后续生成会依赖自己前面已经生成出的结果

## 它和语言模型是什么关系

语言模型负责学习的是：

```text
P(token_t | token_<t)
```

也就是：

- 给定前文，当前 token 的概率分布

而自回归生成则是在推理阶段，真正把这个能力跑起来：

- 先算下一个 token 的概率
- 再选出一个 token
- 再继续算下一步

所以两者关系可以简单理解成：

- 语言模型：定义“怎么判断下一个 token 的可能性”
- 自回归生成：把这个判断机制一步步变成真实输出

## 一个最小流程长什么样

假设输入是：

```text
请把“今天天气很好”翻译成英文：
```

一个简化后的自回归过程大概是：

1. 模型先看整段上下文，预测第一个输出 token，例如 `The`
2. 把 `The` 拼接回上下文
3. 再预测下一个 token，例如 `weather`
4. 再继续拼回去
5. 持续重复，直到遇到停止条件

最后你看到的一整句英文，就是这一连串自回归步骤累积出来的结果。

## 一个简化伪代码

```python
context = tokenize("请把“今天天气很好”翻译成英文：")
output = []

while True:
    probs = model(context + output)
    next_token = choose_token(probs)
    output.append(next_token)
    if next_token == EOS:
        break
```

这段代码省略了很多细节，例如：

- 采样策略
- KV cache
- 停止词
- 长度限制

但已经足够说明本质：

- 一次只生成一个 token
- 新 token 会立刻影响后续生成

## 为什么这种方式特别重要

因为它决定了 LLM 的很多表面行为和工程特征。

例如：

- 为什么回答会一点点往外吐
- 为什么前面一句话写歪了，后面可能一路跟着歪
- 为什么长输出延迟更高
- 为什么“先分析再回答”的提示有时会改变结果

这些现象都和自回归生成直接相关。

## 自回归生成最大的特点：路径依赖

由于每一步都会把已生成内容接回上下文，所以前面一旦选了某条表达路径，后面通常会沿着这条路径继续展开。

比如开头如果已经决定：

- 用正式口吻
- 先给定义再展开
- 或者先走了某个错误假设

那后面的 token 预测都会持续受到它影响。

这也是为什么生成式模型既能形成连贯表达，也会出现：

- 前面错一步，后面一路自圆其说

的现象。

## 自回归不是唯一生成方式，但它是今天主流 LLM 的核心方式

从更宽的模型世界看，生成并不只有自回归一种方法。  
但对于主流 `decoder-only` LLM 来说，自回归生成就是最核心的工作机制。

所以只要你在理解：

- GPT 类模型
- 聊天模型
- 大部分代码模型

基本都绕不开它。

## 为什么工程上必须理解它

如果不理解自回归生成，很多系统现象就会判断错层。

例如：

- 为什么输出会中途跑偏
- 为什么长回答更容易出错
- 为什么前文一句错误会放大到后文
- 为什么控制采样参数会明显改变风格和稳定性

这些都不是随机副作用，而是自回归生成带来的自然结果。

## 一个常见误区

一个常见误区是把自回归理解成：

- 模型在很笨地“一个字一个字试”

这不准确。

更贴近事实的说法应该是：

- 它虽然一次只输出一个 token
- 但每一步决策时用到的上下文并不小

也就是说，输出粒度很小，但判断依据并不局部。  
这也是为什么它虽然是逐 token 生成，却仍然能写出段落、结构和长链条内容。

## 你可以先这样记住

- 自回归生成就是“生成一个 token，再基于它继续生成下一个”
- 它是语言模型推理阶段的实际运行方式
- 已生成内容会持续影响后续输出，所以天然存在路径依赖
- LLM 的很多优点和很多问题，都和这种逐步生成机制直接相关
