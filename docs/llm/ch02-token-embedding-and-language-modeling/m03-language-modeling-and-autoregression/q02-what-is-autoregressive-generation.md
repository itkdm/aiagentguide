# 2.3.2 什么是自回归生成（Autoregressive Generation）？

先给结论：

**自回归生成，就是模型在生成时每次只预测下一个 token，并把刚生成的结果再喂回上下文，继续预测后面的 token。**

“自回归”这个词听起来有点数学化，但对 LLM 来说，它可以先被理解成一句更直白的话：

**一边生成，一边把自己刚写出来的内容接着当作后续生成的输入。**

## 为什么叫“自回归”

“回归”这里可以粗略理解成“依赖已有值去预测后续值”。

“自回归”则表示：后续预测依赖于序列自身已经出现的部分。

放到 LLM 里，就是：

- 先看用户给的上下文
- 生成第一个 token
- 再把这个 token 也算进上下文
- 继续预测第二个 token
- 如此循环

所以“自”指的是，模型会不断利用自己已经生成出来的内容。

## 它和语言模型是什么关系

语言模型负责学习：

```text
P(token_t | token_<t)
```

而自回归生成就是在推理阶段，按这个规则把序列一步一步实际生成出来。

你可以把两者关系简单理解成：

- 语言模型：定义了“如何评估下一个 token 概率”
- 自回归生成：把这个能力真正跑起来，一步步产出文本

## 一个最小流程

假设上下文是：

```text
请把“今天天气很好”翻译成英文：
```

自回归生成过程大概像这样：

1. 模型先看整段上下文，预测第一个输出 token，比如 `The`
2. 把 `The` 接到上下文后面
3. 再预测下一个 token，比如 `weather`
4. 再接回去
5. 持续重复，直到遇到结束标记或达到长度上限

最后模型输出的整句英文，就是这一连串自回归步骤累积出来的结果。

## 一个简化伪代码

```python
context = tokenize("请把“今天天气很好”翻译成英文：")
output = []

while True:
    probs = model(context + output)
    next_token = choose_token(probs)
    output.append(next_token)
    if next_token == EOS:
        break
```

这段代码省略了采样、缓存、停止条件等很多细节，但已经够说明本质：

- 一次只生成一个 token
- 新 token 会立刻影响后续生成

## 自回归生成为什么重要

因为它决定了 LLM 的很多表面行为：

- 为什么回答是逐步出现的
- 为什么前面一句话写歪了，后面可能跟着一起歪
- 为什么输出具有强路径依赖
- 为什么“先想再答”的提示有时会改变结果

这些现象都和自回归生成直接相关。

## 自回归生成的一个核心特点：前面对后面有持续影响

由于每一步都会把已生成内容接回上下文，所以前面一旦选了某种表达路径，后面往往会沿着这条路径继续展开。

比如回答开头如果已经选了：

- 某种语气
- 某种解释结构
- 某种错误假设

那后面的 token 预测都会受到它影响。

这也是为什么生成式模型既能形成连贯表达，也会出现“前面错一步，后面一路错下去”的情况。

## 自回归生成不是唯一生成范式，但它是主流 LLM 的核心方式

从更宽的模型世界看，生成不只有自回归一种办法。

但对于主流 decoder-only LLM 来说，自回归生成是非常核心的工作方式。

所以只要你在理解 GPT 类模型、聊天模型、大多数代码模型，基本都绕不开这个机制。

## 你可以先这样记住

- 自回归生成就是“生成一个 token，再基于它继续生成下一个”
- 它是语言模型推理阶段的实际运行方式
- 已生成内容会持续影响后续输出
- LLM 的很多优点和很多问题，都和这种逐步生成机制有关
