# 3.3.1 Position Encoding 是什么？

先给结论：**位置编码的作用，是把“顺序信息”注入到模型表示里，让 Transformer 不只知道 token 是什么，还知道它们在序列中的位置关系。**

如果没有这一步，self-attention 更像是在看一堆无序元素之间的相关性，而不是在理解一句有先后顺序的语言。

这也是为什么位置编码不是一个“可有可无的小配件”，而是 Transformer 能处理语言序列的基础前提之一。

## 为什么 attention 本身不够

self-attention 的核心是：

- 每个位置拿自己的 `Q`
- 去和全序列的 `K`
- 计算相关性
- 再用权重聚合 `V`

这个过程非常擅长解决“谁和谁有关”的问题。  
但它天然不擅长解决“谁在前、谁在后”的问题。

如果不给任何位置信号，模型更容易看到的是：

- 哪些 token 一起出现
- 哪些 token 在语义上相关

却很难稳定地区分：

- “狗咬人”
- “人咬狗”

因为两者的 token 集合很相似，但顺序完全不同。

所以位置编码的本质，就是给 attention 一个顺序坐标系。

## 位置编码到底在补什么

位置编码补的不是“更多语义”，而是“语义排列方式”。

也就是说，它告诉模型的不只是：

- 这个 token 是什么

还包括：

- 它位于第几个位置
- 它和其他 token 相隔多远
- 它在整段上下文中大致处于什么相对位置

语言里很多含义都依赖这种排列关系。  
比如：

- 主谓宾顺序
- 修饰词靠近谁
- 代词指代前面还是后面的内容
- 条件和结论的先后

没有位置信息，attention 仍然能建立相关性，但很难稳定恢复序列结构。

## 最常见的两类思路：绝对位置与相对位置

位置编码的主流思路，大致可以分成两类。

### 1. 绝对位置编码

绝对位置编码更关心：

- 这是第 1 个 token
- 这是第 2 个 token
- 这是第 3 个 token

也就是说，它直接给每个位置一个编号表示，再把这个表示加入 token embedding。

经典做法包括：

- 正弦 / 余弦位置编码
- 可学习的位置向量

这类方法的优点是：

- 直观
- 简单
- 容易实现

但它更偏“坐标型理解”，也就是强调“我在第几位”。

### 2. 相对位置编码

相对位置编码更关心：

- 两个 token 相距多远
- 谁相对谁更近
- 当前 token 和目标 token 的距离关系是什么

这种方式更贴近很多语言现象，因为语言里很多关系天然更关心“距离”，而不是“绝对编号”。

例如一个修饰词和它修饰的词之间，通常更重要的是它们相隔几步，而不是它们分别处在第 17 位和第 21 位。

这也是为什么现代 LLM 更常见相对位置思路，`RoPE` 就是其中非常典型的一类。

## 为什么位置编码会影响长上下文能力

位置编码不只是一个“结构细节”，它还会直接影响：

- 模型如何理解长距离依赖
- 长上下文时位置关系是否还能稳定泛化
- 超出训练长度后模型会不会退化

这也是为什么不同位置编码方案，在长上下文上的表现差异会很明显。

简单说：

- 位置编码不只决定“能不能看见顺序”
- 还会影响“顺序关系在长距离下还能不能保持好用”

所以它和后面会遇到的长上下文问题，其实是直接连着的。

## 位置编码是怎么和表示合并的

工程上常见有两种合并方式：

- 直接把位置向量加到 token embedding 上
- 在 attention 计算过程中，把位置信息融进 `QK` 相似度里

前者更容易理解，后者通常更适合表达“位置关系如何影响注意力分配”。

这两类方法没有绝对高下，但它们强调的重点不同：

- 前者更像“给 token 一个位置标签”
- 后者更像“让位置关系直接影响注意力”

## 一个常见误解：位置编码是不是在给模型加记忆

不是。

位置编码不是在给模型增加额外记忆容量，它更像是在给模型提供一个坐标系统。  
它解决的是：

- 表示如何区分顺序

而不是：

- 模型如何永久记住更多内容

这两件事很容易混淆，但本质完全不同。

## 一个更直观的类比

可以把 token 理解成城市名，把位置编码理解成地图坐标。

如果只有城市名，没有坐标，你可能知道有哪些城市，却很难理解：

- 谁在谁的左边
- 谁挨着谁
- 从哪到哪是一条路径

语言序列也是类似的。  
只有 token 没有位置信息，模型也很难稳定恢复“结构”。

## 你可以先这样记住

- self-attention 天然擅长建模相关性，但不天然带顺序感
- 位置编码的作用，是给模型一个理解先后和距离的坐标系
- 绝对位置更像“第几位”，相对位置更像“相隔多远”
- 位置编码方案会直接影响长上下文下的表现和泛化能力
