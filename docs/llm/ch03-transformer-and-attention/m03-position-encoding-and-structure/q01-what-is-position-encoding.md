# 3.3.1 Position Encoding 是什么？

先给结论：

**位置编码的作用，是把“顺序信息”注入到模型表示里，让模型知道各个 token 的相对或绝对位置，而不是只看到一堆无序的 token 集合。**

Transformer 的 self-attention 本身是“顺序不敏感”的。换句话说，如果没有位置编码，模型很难区分：

- “狗咬人”
- “人咬狗”

这两句话 token 集合几乎一样，但语义完全不同。位置编码就是为了解决这个问题。

## 为什么 attention 需要位置编码

self-attention 的核心操作是：

- 对序列中每个位置计算相关性
- 按权重聚合上下文

这个过程并不天然包含“第几个位置”这样的信息。

所以如果你只给模型 token embedding，它知道“是什么”，但不知道“在哪”。

位置编码的作用，就是让模型同时拿到：

- token 内容
- token 位置

这样才有可能建模出真实语义中的顺序和结构。

## 位置编码的常见两类思路

### 1. 绝对位置编码

把位置当作一个固定编号，比如：

- 第 1 个 token
- 第 2 个 token
- 第 3 个 token

然后把这个位置信息映射成向量，加到 token embedding 上。

早期 Transformer 使用的就是这种思路，例如：

- 正弦 / 余弦位置编码
- 可学习的位置向量

这类方法简单直观，训练也稳定，但在超长上下文扩展时会遇到一些泛化问题。

### 2. 相对位置编码

相对位置编码更关注“两个 token 之间距离是多少”，而不是“它们各自处在第几位”。

这种方式更贴近语言结构，因为很多关系关心的是：

- 这个词离那个词有多远
- 是否在同一短语范围
- 是否跨越句边界

RoPE 就是相对位置信息注入的一种常见做法，我们会在下一节详细讲。

## 位置编码是怎么和 token embedding 合并的

最常见的方式是：

- 把 token embedding 和 position embedding 相加
- 或者在 attention 计算中把位置信息融入相似度计算

前者更简单，后者在长上下文场景往往表现更好。

## 位置编码不是“记忆”，而是“坐标系统”

容易误解的一点是：位置编码不是让模型“记住”前文内容，而是让模型拥有一个可以区分顺序的坐标系。

它的作用更像是：

- 给每个 token 一个位置标签
- 让模型学会“顺序会影响含义”

这也是为什么位置编码是 Transformer 结构里不可缺的一部分。

## 一个直观类比

可以把位置编码理解成：

- token 是城市
- 位置编码是坐标

如果只有城市名没有坐标，你无法理解“城市之间的距离和顺序”。

模型也是类似，没有位置编码，它只能看到“哪些 token 出现了”，却难以理解“它们按什么顺序出现”。

## 你可以先这样记住

- self-attention 本身不带顺序信息
- 位置编码是为序列提供顺序和结构的坐标系
- 常见做法包括绝对位置编码和相对位置编码
- 位置编码不是额外记忆，而是让模型知道“谁在前，谁在后”
