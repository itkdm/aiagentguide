# 3.3.3 Transformer 的基本结构是什么？

先给结论：**Transformer 的基本结构可以理解成“输入表示 + 位置处理 + 多层重复的 Transformer block”。每个 block 里再包含 attention、MLP、残差连接和 LayerNorm。**

如果再压缩一点，可以把它记成一句话：

**Transformer 是一个由“关系建模 + 非线性加工”反复堆叠形成的深层序列建模架构。**

这个定义比死记结构图更重要，因为它直接解释了：

- 为什么它适合大规模训练
- 为什么它能不断堆深
- 为什么后面的预训练和 LLM 基本都建立在这套骨架之上

## 从输入到输出，主干长什么样

最简化的主干可以写成这样：

```text
输入文本
  -> tokenizer
  -> token embedding
  -> 位置处理（position encoding / RoPE）
  -> N 层 Transformer block
       - attention
       - MLP
       - residual
       - LayerNorm
  -> 输出表示
  -> 任务头 / 下一个 token 预测
```

这就是你在大多数 Transformer 架构图里看到的主脉络。

看上去很简单，但它的关键价值就在于：

- block 结构统一
- 便于重复堆叠
- 很适合并行化训练和扩展

## Transformer block 里到底有什么

一个标准 block 通常至少包含两类核心子层：

- `Attention`：负责从上下文里按相关性聚合信息
- `MLP / FFN`：负责对表示做非线性加工

而为了让深层网络训练稳定，通常还会搭配：

- 残差连接
- LayerNorm

也就是说，block 不是一个单一模块，而是一组配合工作的结构单元。

可以粗略理解成：

- `Attention` 决定“看哪里”
- `MLP` 决定“怎么加工”
- 残差和归一化决定“能不能稳地堆起来”

## 为什么是“多层重复堆叠”这么重要

Transformer 能成为大模型主干，一个很关键的原因就是：

**它的结构非常适合标准化堆叠。**

这意味着你可以比较自然地做这些事：

- 加深层数
- 增大隐藏维度
- 增加头数
- 扩大训练规模

这种“重复 block”的设计，对工程很友好。  
因为它让模型扩展变成一条相对清晰的路径，而不是每次都要重新发明结构。

这也是 Transformer 和很多早期序列模型相比，特别适合走到大规模时代的重要原因之一。

## 为什么说它不是只有 attention

很多人学 Transformer 时，会把所有注意力都放在 attention 上。  
这当然合理，因为 attention 是结构里最有辨识度的部分。

但如果只看到 attention，就会忽略一个关键事实：

Transformer 的能力不是单个模块决定的，而是整个 block 协同决定的。

如果没有：

- `MLP` 的特征加工
- 残差的信息保留
- LayerNorm 的训练稳定

attention 本身并不能支撑今天的大模型训练规模。

所以更准确的说法不是：

- Transformer = attention

而是：

- Transformer = attention 为核心，但能力来自整个 block 结构协作

## 三种常见结构范式和这套主干的关系

你在后面会见到三种常见范式：

- `Encoder-only`
- `Decoder-only`
- `Encoder-Decoder`

它们虽然任务形态不同，但都共享这套基本骨架。

差别主要不在 block 长得完全不同，而在：

- 信息可见范围不同
- mask 方式不同
- 输入输出组织方式不同

这也是为什么理解“基本结构”很重要。  
因为只有先看清共同骨架，后面看三种范式时才不会觉得它们像三套完全无关的东西。

## 为什么现代 LLM 大多是 decoder-only

这部分在下一节会展开，但先建立一个直觉：

现代 LLM 大多走 `decoder-only`，不是因为它“唯一正确”，而是因为它更自然地匹配：

- 自回归训练目标
- 下一个 token 预测
- 统一的生成式任务形态

也就是说，Transformer 的基本结构提供了骨架；  
而 `decoder-only` 是这套骨架在生成型大模型路线上的一种主流落地方式。

## 一个更贴近工程的理解方式

如果从工程角度理解 Transformer 的结构价值，可以总结成三点：

1. block 结构统一，便于大规模堆叠
2. attention 负责建立全局关系
3. 残差、LayerNorm、MLP 共同保证表达能力和可训练性

你后面看到的很多大模型差异，往往不是“骨架完全重写”，而是在这套骨架上做：

- 位置处理方式变化
- attention 细节变化
- 归一化和残差细节变化
- 训练和对齐方式变化

所以先把骨架看懂，后面很多变化都会更容易看懂。

## 一个常见误区

一个很常见的误解是：

- Transformer 的结构图很简单，所以原理应该也很简单

这不准确。

结构图简单，只说明骨架统一；  
真正复杂的地方在于：

- 这些模块为什么能协同工作
- 为什么它们在大规模下还能稳定训练
- 为什么少改一点细节就可能影响性能、稳定性和扩展性

所以“基本结构简单”不等于“工程含义简单”。

## 你可以先这样记住

- Transformer 的主干是：输入表示 + 位置处理 + 多层 block 堆叠
- 每个 block 由 attention、MLP、残差和 LayerNorm 组成
- 它最重要的工程价值之一，是结构统一、便于扩展和并行训练
- 现代 LLM 的很多变化，都是在这套基本骨架上的演化
