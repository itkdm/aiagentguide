# 3.4.3 BERT、T5、GPT 这几类模型分别代表什么思路？

先给结论：

**BERT 代表的是 encoder-only 的理解范式，GPT 代表的是 decoder-only 的生成范式，T5 代表的是 encoder-decoder 的统一文本到文本范式。**

所以这三个名字不是“模型排名”，而是三条不同结构路线的典型代表。

## BERT：理解优先的 encoder-only 路线

BERT 的核心特征是：

- 双向编码输入
- 以理解为核心目标
- 训练目标以掩码语言模型（MLM）为代表

因此它更适合：

- 文本分类
- 命名实体识别
- 语义匹配
- 检索重排

它的代表性贡献不在于“生成”，而在于把 Transformer encoder 打造成强表征模型。

## GPT：生成优先的 decoder-only 路线

GPT 的核心特征是：

- 单向自回归生成
- 统一“下一个 token 预测”范式
- 更自然地扩展到对话、代码、指令

因此它更适合：

- 开放式生成
- 对话交互
- 长文本输出
- 代码生成

GPT 的代表性贡献在于把“生成式预训练 + 大规模扩展”变成一条可持续的主线。

## T5：序列到序列的 encoder-decoder 路线

T5 的核心特征是：

- encoder 负责理解输入
- decoder 负责生成输出
- 统一为“text-to-text”范式

因此它非常适合：

- 翻译
- 摘要
- 改写
- 标准 seq2seq 任务

T5 的代表性贡献在于用统一的“文本到文本”框架把很多 NLP 任务纳入一个标准接口。

## 结构范式对比（直觉版）

- BERT：读懂输入
- GPT：生成输出
- T5：先读懂再生成

## 为什么它们都还重要

尽管今天通用大模型主线偏向 GPT 风格，但：

- BERT 仍然是理解和检索任务的强基座
- T5 仍然在翻译和严格序列映射任务里很稳

所以选择它们的理由不是“谁更新”，而是“任务更匹配”。

## 工程上怎么快速判断该看哪一类

- 你要做理解、分类、检索：先看 BERT 路线
- 你要做生成、对话、代码：先看 GPT 路线
- 你要做明确输入到输出映射：先看 T5 路线

## 你可以先这样记住

- BERT = encoder-only = 理解优先
- GPT = decoder-only = 生成优先
- T5 = encoder-decoder = 序列到序列统一范式

它们不是互相取代，而是代表三种不同的结构思路和任务侧重点。
