# 12.3.2 参数量是否仍然是核心指标？

先给结论：

**参数量今天仍然有参考价值，但已经不能单独作为模型强弱和选型优先级的核心结论。**

它更像一个粗线索，而不是最终判断标准。

## 为什么参数量以前很重要

参数量曾经很重要，是因为它大致反映了：

- 模型容量
- 表达能力上限
- 可学习复杂模式的潜力

在训练方法和数据条件相近时，更大模型通常更容易表现更强。

## 为什么今天不能只看参数量

因为现在影响模型表现的变量已经很多：

- 训练数据质量
- 后训练方式
- 指令微调和对齐
- 推理优化
- 长上下文能力
- 工具和系统集成方式

这意味着：

- 参数更多，不一定更好用
- 参数更少，也不一定就不够强

## 参数量今天还能说明什么

它仍然可以帮助你粗略判断：

- 大致能力上限
- 推理资源需求
- 显存和部署压力

也就是说，它更适合做“资源和规模感知”，不适合直接做“业务结论”。

## 一个常见误区

很多团队会默认：

- 大参数模型一定更强
- 小参数模型一定只能做简单任务

但现实里，经过更好训练和对齐的小模型，完全可能在某些稳定任务上比大模型更划算、更稳。

## 工程上更该看什么

比单纯参数量更值得看的，通常是：

- 你的核心任务评测结果
- 延迟和成本表现
- 长上下文下的实际效果
- 稳定性和波动情况

这些维度和业务更直接相关。

## 你可以先这样记住

- 参数量是线索，不是答案
- 它更适合帮助你判断资源需求和大致容量
- 真正选型时，要让业务评测结果压过参数崇拜
