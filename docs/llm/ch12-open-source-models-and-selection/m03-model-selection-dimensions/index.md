# 12.3 模型选型维度

先给结论：**模型选型真正难的地方，不是记住要看哪些维度，而是知道这些维度之间会互相拉扯，不能各自单独最优。**

这部分会把模型选型时最容易被忽略的判断框架拆开说明。重点不是列参数表，而是让你知道这些指标在真实业务里分别意味着什么。

- [上下文长度、推理速度、价格、稳定性该怎么平衡？](./q03-how-to-balance-context-speed-price-stability)

## 读完后你应该能判断

- 哪些维度是真正的主约束，哪些只是辅助参考
- 为什么参数量已经不再足够代表模型好坏
- 什么时候小模型是更优解，什么时候强模型不可替代
