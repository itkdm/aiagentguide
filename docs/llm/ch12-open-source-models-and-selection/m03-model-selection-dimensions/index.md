# 12.3 模型选型维度

先给结论：**模型选型真正难的地方，不是记住要看哪些维度，而是知道这些维度之间会互相拉扯，不能各自单独最优。**

这一节要把“选型维度”从一张参数表变成一个判断框架。  
很多团队在选模型时，表面上列了很多维度，但最后还是会被一个单一指标牵着走，比如：

- 只看参数量
- 只看上下文长度
- 只看榜单效果
- 只看单 token 成本

这些做法的问题不是指标错了，而是它们都只覆盖了真实选型的一部分。  
在业务里，真正难的地方通常在于：

- 效果和成本会互相拉扯
- 上下文长度和延迟会互相拉扯
- 稳定性、控制权和运维能力也会互相影响

所以这一节的重点不是“列出所有维度”，而是让你知道这些维度在真实场景里分别意味着什么，又该怎么一起看。

## 读完后你应该能判断

- 哪些维度是真正的主约束，哪些只是辅助参考
- 为什么参数量已经不再足够代表模型好坏
- 什么时候小模型是更优解，什么时候强模型不可替代
