# 第 12 章 开源模型与模型选型

先给结论：**模型选型不是“谁更强就选谁”，而是看它在你的业务里是否同时满足效果、成本、延迟、控制权、合规和运维能力等多重约束。**

这一章的重点，是让你建立“选型判断力”，而不是记住几个热门名字。真正有用的选型，通常来自对任务、数据边界、团队能力和长期维护成本的综合判断。

## 你应该建立的判断

- 模型效果只是选型维度之一，不是唯一维度
- 开源与闭源的核心差别，不只是价格，而是控制权、部署方式和治理成本
- 真正成熟的选型，必须结合任务复杂度、数据边界、团队能力和长期运维成本
