# 12.2.2 通用模型和垂类模型怎么选？

先给结论：

**通用模型的优势在覆盖面广、迁移性强，垂类模型的优势在特定领域更贴任务、更省资源、更容易对齐行业要求。**

所以关键不是谁绝对更好，而是你的场景到底更需要“广”还是“深”。

## 通用模型更适合什么

通用模型通常更适合：

- 任务类型很多
- 用户问题分布很杂
- 还在探索产品边界
- 需要较强的泛化能力

这类情况下，通用模型的价值在于：你还没有把问题空间收窄，所以先要有更大的覆盖面。

## 垂类模型更适合什么

垂类模型通常更适合：

- 领域边界清晰
- 术语密度高
- 正确性要求高
- 任务长期稳定

例如法律、医疗、金融、工业文档等场景，垂类模型常常更容易在特定术语和领域判断上贴业务。

## 为什么很多团队会高估垂类模型

因为“更专业”听起来很有吸引力。  
但垂类模型也有明显边界：

- 泛化能力通常更弱
- 一旦问题超出领域，表现可能明显下滑
- 维护多个垂类模型会增加系统复杂度

所以如果你的真实流量并不够垂，通用模型往往更稳。

## 为什么很多团队也会高估通用模型

因为通用模型看起来什么都能答。  
但现实里，在专业要求高的场景里，通用模型常见问题是：

- 术语不够稳定
- 领域细节幻觉多
- 难以长期满足行业规范

这时垂类模型或者“通用模型 + 垂类数据 / RAG / 适配”的方案，往往更现实。

## 一个更实用的判断法

你可以用三个问题来判断：

1. 业务输入是不是大部分都集中在固定领域？
2. 错误是不是主要出在领域术语和专业判断上？
3. 领域外泛化能力对你是不是还很重要？

如果前两项都很强、第三项不强，垂类模型通常更值得认真考虑。  
反过来，如果场景很杂，通用模型通常更稳。

## 你可以先这样记住

- 通用模型强在覆盖面
- 垂类模型强在领域贴合
- 任务越集中、越专业，垂类模型的价值通常越大
