# 12.2.3 为什么不同模型的“默认行为风格”差异很大？

先给结论：

**不同模型的默认行为风格差异很大，主要不是因为“智商不同”，而是因为它们在训练数据、后训练方式、对齐策略、系统提示模板和产品目标上都可能不同。**

所以同样一个 prompt，在不同模型上经常不会得到同样的“助手人格”和行为边界。

## 一个最直观的差异来源：后训练不同

哪怕两个模型共享相近的基础能力，只要后训练路径不同，默认行为就会明显变化。

比如会表现成：

- 一个更愿意直接回答
- 一个更倾向先解释边界
- 一个更保守
- 一个更喜欢展开

这不是偶然，而是训练目标带来的默认倾向。

## 训练数据也会塑造默认风格

模型接触过什么样的对话、文档和任务数据，会影响它默认学到什么样的表达和判断习惯。

常见体现包括：

- 更书面还是更口语
- 更简洁还是更铺陈
- 更偏工具型还是更偏陪伴型

所以“风格”并不是表面装饰，而是训练分布的结果。

## 对齐和安全策略也会直接影响行为

模型的拒答方式、保守程度、是否先提醒风险、是否喜欢给步骤，都可能受到对齐策略影响。

这也是为什么有些模型：

- 更容易说“不确定”
- 更容易提醒风险
- 更容易拒绝边界请求

这些差异在产品体验里非常明显。

## 系统提示和模板也会放大这种差异

真实调用里，模型并不是裸跑的。  
很多产品会给模型加默认系统提示、聊天模板、格式约束，这会进一步强化某种行为风格。

所以你看到的“默认行为”，往往不是纯模型本体，而是：

- 模型本身
- 后训练
- 默认模板

三者叠加的结果。

## 为什么这件事对选型重要

因为用户感知到的“这个模型好不好用”，很多时候先感知到的不是能力上限，而是默认行为是不是顺手。

例如：

- 是否爱跑题
- 是否过度保守
- 是否太啰嗦
- 是否格式稳定

这些都会直接影响产品体验和后续 prompt 成本。

## 一个务实判断

如果两个模型能力接近，但一个默认行为更贴你的产品风格和任务要求，那它往往更值得优先选。

因为这意味着：

- prompt 成本更低
- 调整更容易
- 长期维护更稳

## 你可以先这样记住

- 默认行为风格差异，主要来自训练和对齐，不只是参数大小
- 这种差异会直接影响用户体验和系统调试成本
- 选模型时，默认行为是否贴产品，和能力分数一样重要
