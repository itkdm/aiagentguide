# 12.2 模型类型的区分

先给结论：**模型类型的差异，不只是“参数多少”或“分数高低”，还体现在默认行为、任务适配方式、回答风格和适用边界上。**

这一节的重点，是把最容易混淆的几组模型类型重新拆开。  
很多人看到模型名字时，只会先看参数量和排行榜位置，但真正影响你能不能把模型用对的，往往还包括：

- 它是 Base 还是 Instruct
- 它偏通用能力还是偏特定领域
- 它的默认行为风格是保守、直接、啰嗦还是偏执行

这些差异在模型卡片里不一定一眼就能看明白，但在真实系统里会直接表现出来。  
所以这一节不是在背类别，而是在建立一种使用视角：  
**模型类型决定了你拿到的不是同一种“原材料”，而是不同形态的能力底座。**

## 读完后你应该能判断

- 一个模型更像“续写底座”，还是“可直接当助手用的模型”
- 什么时候应该优先选通用模型，什么时候垂类模型更划算
- 为什么同样的 prompt，在不同模型上会呈现完全不同的默认行为
