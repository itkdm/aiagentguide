# 14.2 面试表达与方法论

先给结论：**LLM 面试里真正拉开差距的，不是你能不能背出几个术语，而是你能不能把一个问题从“概念”讲到“系统”，再讲到“业务取舍”。**

很多人面试时的问题，不是完全不会，而是回答停留在一个层面：

- 要么只讲原理，不讲工程
- 要么只讲工程技巧，不讲底层原因
- 要么只讲概念，没有判断标准

这一节的目标，就是把这些层拆开，给你一套更稳定的讲法。它不只是用于面试，也适用于技术沟通、方案评审和系统分析。

## 读完后你应该能判断

- 为什么“会背底层原理”还不等于“会回答 LLM 面试题”
- 一个问题应该先讲概念边界，还是先讲系统链路，还是先讲业务取舍
- 如何把不同类型的题目收敛到一套稳定结构里，而不是每题都临场拼接
