# 14.2.1 面试中讲 LLM，为什么不能只背 Transformer？

先给结论：**Transformer 必须会讲，但只讲 Transformer，通常拿不到高分。**  
原因很简单，Transformer 主要回答的是“模型内部如何做序列建模”，而 LLM 面试真正想看的，往往是你能不能把这个结构放回完整链路里理解：能力怎么形成、助手行为怎么得到、系统效果为什么会波动、上线问题最后落在哪一层。

所以问题不在于“要不要讲 Transformer”，而在于**你是不是把 Transformer 当成了全部答案**。

## Transformer 解决了什么

Transformer 当然是 LLM 的基础主干结构，这一部分必须讲清楚，否则说明底层没打稳。至少要知道：

- 它用 attention 解决长距离依赖建模问题
- 它把整段上下文一起编码，而不是像早期 RNN 那样主要按顺序传递状态
- 它适合并行训练，因此更容易扩展到大规模预训练

也就是说，Transformer 解释的是：**为什么现代语言模型能以一种适合扩展的方式处理上下文。**

这部分很重要，但它只是“模型骨架”。

## 只讲 Transformer，为什么会显得浅

### 1. 它解释不了能力为什么会出现

今天大家说的 LLM 能力，并不是“有了 Transformer 就自动出现”的。  
真正让能力长出来的，是下面这几件事一起作用：

- 大规模预训练数据
- next token prediction 这类训练目标
- 参数、数据、算力一起扩展
- 长时间训练带来的模式压缩与泛化能力

所以如果只背 attention、QKV、残差、FFN，却说不清预训练、数据规模、scaling law 和训练目标，回答会停在“结构知识”，而不是“能力形成机制”。

### 2. 它解释不了模型为什么会更像助手

很多面试官问 LLM，本质上并不是在问论文结构，而是在问：

- 为什么 Base Model 和 Instruct Model 行为差这么大
- 为什么模型会更愿意遵循指令
- 为什么它会更安全、更保守、或者更稳定

这些问题主要落在后训练和对齐上，包括：

- SFT
- RLHF
- DPO
- 系统提示词和推理策略

也就是说，Transformer 解释不了“助手感”从哪里来。  
如果你一谈 LLM 就只讲结构，面试官很容易判断你对“预训练模型”和“产品可用模型”的差别理解不深。

### 3. 它解释不了真实系统为什么还需要 RAG 和工具

真实业务里最常见的问题不是“attention 公式写不出来”，而是：

- 知识过时
- 私有知识拿不到
- 结果不可追溯
- 需要调接口、查数据库、执行动作
- 同题质量波动

这些问题的主要解法，通常不是去重新解释 Transformer，而是去做：

- RAG
- Tool Calling
- Workflow / Agent 编排
- 评测与监控
- 系统级兜底

所以面试里如果你能从 Transformer 自然过渡到 RAG、工具、评测和上线治理，说明你理解的是“LLM 系统”；如果你一直停在 attention block，说明你理解的还是“模型结构”。

## 面试官通常真正想看什么

很多题目表面上像原理题，实际上考的是下面三层：

1. 你知不知道底层原理是什么
2. 你知不知道它在整条系统链路里处于什么位置
3. 你能不能把技术解释推进到工程决策

比如问“你怎么理解 LLM”，很多人会从 Self-Attention 开始一路讲到 Position Encoding。  
这不能算错，但如果只停在这里，面试官通常还会继续怀疑：

- 你知不知道能力为什么会形成
- 你知不知道为什么上线系统还需要 RAG 和工具
- 你知不知道模型表现为什么会被 prompt、采样、评测、观测链路影响

这也是为什么现在很多 LLM 面试题，本质上已经不是“背结构”，而是“看你能不能把结构放回系统里”。

## 一个更稳的回答顺序

如果面试题是“你怎么理解 LLM”，一个更稳的讲法通常是：

1. 先用 1 到 2 句话交代 Transformer 是底层主干结构，负责上下文建模与可扩展训练
2. 再补一句：LLM 的关键不只是结构，而是大规模预训练带来的通用能力
3. 接着说明：真正像助手的行为来自 SFT、RLHF、DPO 这类后训练和对齐
4. 最后补上：真实系统效果还取决于 RAG、工具、推理配置、评测与治理

这四步的作用是把你的回答从“论文复述”推进到“系统理解”。

## 一个常见失分模式

很多人一紧张就开始堆术语：

- Self-Attention
- Multi-Head Attention
- Position Encoding
- LayerNorm
- FFN

问题不是这些词不该讲，而是**只剩这些词的时候，回答就会显得没有主线**。  
面试官听完以后，知道你背过结构，但不知道你有没有工程判断。

更稳的做法是：

- 结构讲短
- 作用讲准
- 边界讲清
- 系统链路补全

## 自查标准

如果你想判断自己有没有掉进“只背 Transformer”的坑，可以用这几个问题自查：

1. 你能不能用一句话说清 Transformer 解决什么，不解决什么？
2. 你能不能把回答自然接到预训练、对齐、推理和系统设计？
3. 你能不能解释为什么真实知识型系统还需要 RAG 和工具？
4. 你能不能说出影响上线效果的因素不只在模型结构？

只要这几条里有两三条答不上来，面试时就很容易显得“底层知道一点，但系统理解不够完整”。

## 你可以先这样记住

- Transformer 是 LLM 的重要基础，但不是完整答案
- LLM 面试考的不只是结构，而是结构、训练、推理、系统之间的关系
- 讲 Transformer 最稳的方式，不是讲得越细越好，而是讲完后能自然过渡到完整系统链路
