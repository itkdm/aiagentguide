# 14.1.3 Prompt 写得好，是不是就能解决大部分问题？

先给结论：**Prompt 很重要，但它主要解决的是“怎么把任务讲清楚”和“怎么约束模型输出”。只靠 Prompt，解决不了知识缺失、实时数据、模型上限、工具执行和系统治理这些根因。**

把 Prompt 当成万能钥匙，是 LLM 学习和落地里最常见的误区之一。

更准确的说法应该是：

- Prompt 决定输入接口质量
- 但它不是所有问题的最终解法

## 为什么 Prompt 特别容易被神化

因为 Prompt 的反馈非常快。

你改一句话，结果往往立刻就会变化：

- 表达方式变了
- 输出结构变了
- 口吻变了
- 稳定性看起来也可能变好了

而且 Prompt 还有几个很诱人的特点：

- 不需要重新训练
- 上手门槛低
- 演示效果明显
- 成本相对低

这会很容易让人产生一种错觉：

“只要我 Prompt 写得足够细，很多问题都能调出来。”

这也是为什么很多团队在项目早期，几乎所有问题都会先去改 Prompt。

## Prompt 真正擅长解决什么问题

Prompt 最擅长解决的是输入表达层的问题，例如：

- 任务目标没讲清楚
- 输出格式不明确
- 缺少必要上下文
- 缺少 few-shot 示例
- 语气、结构、口径不稳定

这些问题的共同点是：

- 模型大致具备这个能力
- 但需要你把任务接口设计得更清楚

例如：

- 让模型按固定 JSON 输出
- 要求先总结再给结论
- 给几条好例子让模型学输出风格
- 明确哪些内容必须回答、哪些不能编造

这些都属于 Prompt 的主战场。

## Prompt 不擅长解决什么问题

Prompt 的边界同样很清楚。

### 1. 知识缺失和知识过时

如果模型根本不知道某家公司制度、某个私有文档、某个最新接口变化，Prompt 再精细也无法凭空补知识。

这时真正该考虑的是：

- `RAG`
- 外部知识接入
- 文档治理

而不是继续打磨提示词措辞。

### 2. 实时状态和真实动作

如果任务需要：

- 查订单状态
- 看库存
- 调接口
- 创建工单
- 执行审批

那核心是工具调用和系统集成，不是 Prompt 技巧。

Prompt 可以帮助模型更好地决定“怎么调用工具”，但不能替代工具本身。

### 3. 模型能力上限

有些问题的难点本来就在能力上限，例如：

- 长链路复杂推理
- 复杂代码生成
- 高难度约束协同
- 多源材料深度整合

这时 Prompt 可能能改善一点，但通常不是根治手段。

### 4. 系统稳定性和回归问题

如果你面对的是：

- 同题不同答
- 边界样本不稳定
- 某类问题持续误答
- 模型升级后质量回退

那么真正需要的是：

- 评测
- 监控
- 版本对比
- 分层排查

而不是无休止地叠加 Prompt 规则。

## 一个特别常见的错误路径

很多团队会走上这样一条路：

1. 先改 Prompt
2. 不行就再加规则
3. 再加 few-shot
4. 再加“必须”“严禁”“一定要”
5. 最后 Prompt 越来越长，链路越来越脆

这条路的问题不在于 Prompt 不该改，而在于：

**它把所有问题都压到了输入层，却没有先判断根因到底在哪一层。**

最后常见结果是：

- Prompt 很长，但还是不稳
- 可维护性越来越差
- 一次修改会影响别的场景
- 新版本模型一换，很多“调出来的效果”又失效

## 更稳的处理顺序是什么

更实用的做法通常是：

1. 先判断问题是在表达、知识、工具、模型能力还是系统流程
2. 如果是表达和约束问题，优先优化 Prompt
3. 如果是知识问题，上 `RAG`
4. 如果是实时状态或动作问题，接 `Tool`
5. 如果是稳定性问题，补 `Evals` 和监控
6. 如果是能力上限问题，再考虑换模型或微调

只有这样，Prompt 才会被放在它最擅长的位置上。

## Prompt 该怎么用才更像工程手段

更工程化的做法，不是追求“写出一个万能 Prompt”，而是：

- 把任务输入结构化
- 把输出要求明确化
- 把约束条件写清楚
- 把示例控制在必要范围
- 配合评测做迭代，而不是纯凭感觉调

这样 Prompt 才更像接口设计，而不是玄学技巧。

## 一个快速判断法

如果你在调 Prompt 时遇到下面这些信号，就该警惕是否用错层了：

- Prompt 越写越长，但质量提升越来越小
- 明明是知识问题，却总想靠描述更细来补
- 明明是实时状态问题，却不接工具
- 一换模型版本，原有 Prompt 效果大幅波动

这通常说明问题已经不只是 Prompt 问题了。

## 你可以先这样记住

- Prompt 很重要，但不是万能钥匙
- Prompt 擅长解决“任务讲清楚”和“输出约束清楚”
- 知识、状态、执行、稳定性和治理问题，都需要更高层的系统手段一起补
