# 14.1 常见误区

先给结论：**LLM 领域最容易踩的坑，不是完全不懂，而是“好像懂了”。知道 Transformer、Prompt、RAG、Agent 这些词，并不等于已经建立起可用的工程判断。**

这一节会集中拆解几类最常见的误区。它们的共同特点是：表面上都像是合理直觉，实际上却会把人带到错误的优化方向上。比如把 LLM 看成只是更大的 NLP、把参数量当成唯一标准、把 Prompt 当成万能解法，或者把会背概念误认为会做系统。

如果你能把这些误区拆开，后面不管是做选型、排错、面试表达还是系统设计，都会稳很多。

## 读完后你应该能判断

- 哪些说法只是顺口的行业话术，哪些才是对系统有用的判断
- 为什么很多“看起来很高级”的方案，本质上只是在错误层级上用力
- 当模型表现不理想时，应该优先回到哪几层排查，而不是盲目换模型或堆 prompt
