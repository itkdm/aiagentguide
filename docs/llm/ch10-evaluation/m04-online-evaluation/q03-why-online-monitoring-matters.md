# 10.4.3 为什么线上监控对 LLM 产品至关重要？

先给结论：

**线上监控对 LLM 产品至关重要，是因为离线评测只能告诉你“上线前大概怎样”，而线上监控才能告诉你“真实用户、真实流量、真实依赖链路下，现在到底怎样”。**

如果没有线上监控，你的系统就算离线评测做得不错，也很容易在真实环境里失控。

## 为什么 LLM 产品比传统系统更需要线上监控

因为它的变化来源更多。

传统系统很多问题来自代码逻辑本身；  
LLM 系统还会受到这些因素影响：

- 模型版本变化
- prompt 调整
- 检索数据变化
- 工具接口波动
- 用户输入分布漂移
- 安全策略变化

所以同一个功能，今天和下周的表现可能并不一样。

## 没有线上监控会发生什么

最常见的后果包括：

- 质量退化很晚才被发现
- 出问题时无法快速定位
- 团队只能靠用户投诉知道系统变差
- 版本迭代没有闭环

本质上，你会失去对系统健康状况的可见性。

## 为什么离线评测不能替代它

离线评测很重要，但它有天然边界：

- 样本有限
- 分布固定
- 很难完整覆盖真实用户行为
- 看不到真实延迟和依赖链路波动

所以离线评测更像“上线前体检”，线上监控更像“上线后持续生命体征监测”。

## 线上监控真正带来的价值

它至少能提供四种价值：

1. **及时发现退化**  
2. **帮助定位原因**  
3. **支持版本比较和回滚**  
4. **让迭代有闭环数据支撑**  

如果缺少这四点，很多优化工作都会停留在猜测层面。

## 对业务方真正重要的是什么

业务方往往不关心你离线某个分数涨了多少，真正关心的是：

- 用户有没有更满意
- 问题有没有更少
- 成本有没有失控
- 风险有没有被压住

这些问题最终都要靠线上监控来回答。

## 一个务实做法

即使是中小团队，也至少应该做到：

- 记录核心请求链路
- 采样保存失败案例
- 持续看质量、延迟、成本主指标
- 为异常波动设置告警

这已经能比“出了事再看日志”强很多。

## 你可以先这样记住

- 离线评测决定你能不能放心上线
- 线上监控决定你上了线之后能不能稳住
- 没有线上监控，LLM 产品很难形成真正的优化闭环
