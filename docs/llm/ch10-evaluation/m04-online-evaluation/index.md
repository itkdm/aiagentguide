# 10.4 线上模型评估

先给结论：**LLM 系统上线后，真正决定成败的不是某次离线分数，而是线上质量、稳定性、延迟、成本和错误恢复能力能不能长期被看见、被定位、被修正。**

这一节把视角从离线评测切回真实系统运行。  
很多团队前面会花不少时间做离线数据集、基准测试和人工抽检，但一旦模型上线，真正决定系统是否能持续优化的，其实是线上观测能力。

因为线上问题往往不是静态暴露的，而是动态出现的：

- 用户问题类型变了
- prompt 改动影响了分布
- 检索链路波动了
- 工具调用成功率下降了
- 模型升级后某类任务突然回退了

如果系统没有把这些变化长期看见、拆开定位、及时回滚或修复，那么离线分数再漂亮也很难支撑稳定迭代。

所以这部分的重点，不只是“线上要看哪些指标”，而是建立一种生产判断：  
**没有线上观测和定位能力，LLM 系统就很难真正进入工程闭环。**
