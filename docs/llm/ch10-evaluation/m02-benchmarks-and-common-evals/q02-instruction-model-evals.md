# 10.2.2 指令模型常见评测方式有哪些？

先给结论：

**指令模型评测关注的重点，已经不只是“会不会续写”，而是“会不会按要求完成任务、输出是否可靠、行为是否可控”。**

所以它的评测方式会比预训练模型更贴近真实交互和业务流程。

## 一类核心评测：指令遵循

这是指令模型最基础的一类评测。

常见会看：

- 是否理解任务要求
- 是否遵守格式约束
- 是否按角色和风格输出
- 是否能在多约束下完成任务

这类评测的关键不是知识难度，而是模型有没有真的“按要求做”。

## 一类核心评测：事实性与可靠性

指令模型要面向用户使用，因此事实性会变得更重要。

常见会评：

- 幻觉率
- 引用是否可信
- 答案是否基于给定上下文
- 遇到不确定时是否会乱答

这些评测通常比预训练阶段更强调“行为可靠”，而不是只看知识覆盖面。

## 一类常见评测：任务完成质量

真实系统里，很多人更关心的是任务完成得怎么样。

比如：

- 摘要质量
- 抽取准确率
- 分类一致性
- 改写是否满足要求
- 多步任务完成率

这类评测会更接近最终产品目标。

## 一类重要评测：安全与拒答行为

指令模型上线后，还要面对安全、合规、风险控制问题。

所以常见还会看：

- 是否会响应不该响应的请求
- 是否会泄露不该泄露的信息
- 是否能在高风险场景下拒答或降级

这部分通常不是传统 benchmark 能完全覆盖的，需要更贴合业务场景的数据。

## 一类越来越常见的方式：模型裁判与人工评审

因为很多指令任务没有唯一标准答案，行业里常见会结合：

- 人工标注
- LLM-as-a-Judge
- Pairwise 对比
- 参考答案打分

这些方式更适合评开放式输出，但前提是评分标准必须定义清楚。

## 为什么它和预训练模型评测不同

因为目标不同。

预训练模型更像在评“底座能力”；  
指令模型更像在评“交互完成能力”和“上线行为质量”。

所以同一个模型在预训练类 benchmark 上高分，不代表在指令遵循和用户体验上也一定高分。

## 你可以先这样记住

- 指令模型更看任务完成、行为约束和事实可靠性
- 这类评测更贴近产品目标和上线场景
- 真正有用的评测，通常会把自动指标、模型裁判和人工评审结合起来
