# 10.2.1 预训练模型常见评测方式有哪些？

先给结论：

**预训练模型的评测重点，通常不是“像不像一个好助手”，而是“它的语言建模能力、知识覆盖、迁移能力和基础推理能力到底到了什么水平”。**

也就是说，预训练阶段更关心底座能力，而不是最终交互体验。

## 一类最基础的评测：语言建模指标

预训练模型最基础的评测，是看它对文本分布学得怎么样。

常见指标包括：

- loss
- perplexity（困惑度）
- token-level accuracy

这些指标的作用，是观察模型在验证集上的语言建模表现是否提升、是否过拟合、不同训练策略是否有效。

但要注意，它们只能说明“模型对文本预测得更好了”，不能直接代表真实任务一定更好。

## 一类常见评测：下游任务迁移

预训练模型还经常会做一组标准下游任务评测，用来观察它的泛化能力。

常见方向包括：

- 阅读理解
- 常识问答
- 多任务知识测试
- 分类和抽取

核心问题是：一个只做过预训练的模型，在少样本或零样本条件下，能不能把学到的能力迁移出去。

## 一类重要评测：知识与理解广度

像 MMLU 这类测试，就常被拿来观察模型在多学科知识理解上的表现。

这类评测能帮助你看到：

- 模型知识面广不广
- 不同学科之间能力是否均衡
- 它更像是“只会语言模式”，还是确实具备一定知识调用能力

## 一类补充评测：推理与专业能力

如果模型要被用于数学、代码、专业问答等场景，通常还会额外看专项任务：

- 数学题
- 代码生成
- 专业领域问答
- 长上下文理解

原因很简单：预训练模型的平均水平不等于它在关键场景下的可用水平。

## 为什么预训练评测不能只看一个指标

因为不同指标看的不是同一件事。

- 困惑度更像在看语言建模是否进步
- 多任务测试更像在看泛化和知识覆盖
- 专项 benchmark 更像在看某个能力面

如果你只看其中一个，很容易误判模型。

## 工程上该怎么理解

你可以把预训练模型评测分成三层：

1. **训练层**：loss、perplexity 等基础指标  
2. **能力层**：知识、推理、迁移类任务  
3. **专项层**：数学、代码、长文本、专业场景  

这样更容易知道每个分数到底在说明什么。

## 你可以先这样记住

- 预训练模型先看底座能力，不先看助手体验
- 基础语言建模指标很重要，但不够
- 真正判断预训练效果，通常要把训练指标和下游任务一起看
