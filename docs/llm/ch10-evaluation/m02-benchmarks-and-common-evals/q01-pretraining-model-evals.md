# 10.2.1 预训练模型常见评测方式有哪些？

先给结论：**预训练模型的评测重点，通常不是“像不像一个好助手”，而是“它的语言建模能力、知识覆盖、泛化能力和若干基础专项能力到了什么水平”。**  
也就是说，预训练阶段更关心底座能力，而不是最终交互体验。

很多读者一看到模型评测，就会先想到问答效果、聊天顺不顺、会不会听指令。但这些更接近指令模型和产品系统的评估目标。预训练模型本身首先要回答的是另外几个问题：

- 它是不是学会了更好的语言分布建模
- 它能不能把预训练中获得的统计规律迁移到下游任务
- 它在知识、推理、代码、数学等基础能力上处在什么水平
- 它的进步是训练真实带来的，还是只是在局部数据上过拟合

如果这几件事都没看清，就很难判断预训练本身到底做得好不好。

## 第一类：训练过程指标

预训练阶段最基础的一类评测，是看训练本身是否在稳定改进。最常见的就是：

- `loss`
- `perplexity`
- `token-level accuracy`

这些指标的作用，不是直接回答“模型能不能上线”，而是帮助你判断：

- 模型是否还在继续学到东西
- 当前训练是否开始过拟合
- 不同数据配比、学习率、batch 策略是否有效
- 新模型相对旧模型在语言建模上是否真的变好

这里要先建立一个边界：**训练指标主要反映“预测下一个 token 的能力”，它和真实任务效果有关，但不是一一对应。**

例如，一个模型的验证集 loss 下降了，通常说明语言建模更好了；但这不自动等于它在真实问答、代码生成或工具调用里一定同步提升。原因很简单，真实任务往往还叠加了：

- 任务格式约束
- 多步推理链路
- 安全策略
- 指令跟随
- 系统提示和工具交互

这些都不是单靠训练 loss 就能完整覆盖的。

## 第二类：通用下游能力评测

只有训练指标还不够。因为你最终关心的不是“它会不会继续拟合训练语料”，而是“它学到的能力能不能迁移出来”。

所以预训练模型经常还会做一组标准化的下游任务评测，用来观察它的泛化能力。常见方向包括：

- 阅读理解
- 常识与知识问答
- 分类与抽取
- 多任务零样本或少样本测试

这类评测关心的是：**模型是否形成了足够通用的语言能力，而不是只会记住训练分布。**

你可以把它理解为一次迁移能力检查。一个预训练模型如果在多种任务上都能保持还不错的表现，通常说明它的底座能力更扎实，后续做指令微调、领域适配或系统增强时，也更有潜力。

## 第三类：知识广度与综合理解评测

像 `MMLU` 这一类评测，经常被拿来观察模型在多学科知识理解上的表现。它的意义不在于模拟真实产品，而在于帮助你判断几个问题：

- 模型知识覆盖够不够广
- 不同学科之间能力是否均衡
- 它到底是只会语言模式，还是具备一定知识调用能力

这一类评测对预训练模型尤其重要，因为预训练本身往往依赖大规模开放语料。开放语料会带来两个结果：

1. 知识面可以很宽  
2. 质量也可能非常不均匀

所以你不仅要看“模型学了多少”，还要看“它学出来的能力分布是什么样”。有些模型在少数热门学科上分数很高，但在长尾领域明显偏弱；这在真实系统里会表现成回答风格很自信，但覆盖面并不稳定。

## 第四类：专项能力评测

如果模型未来要服务特定方向，通常还会补一组专项 benchmark。常见的有：

- 数学推理
- 代码生成
- 长文本理解
- 专业领域问答

原因很现实：**预训练模型的平均能力，不等于关键场景里的可用能力。**

举个简单例子，一个通用预训练模型在通识问答上表现不错，不代表它在数学推理上也可靠；代码能力强，也不代表长文档问答就一定稳定。专项评测的作用，就是把这些容易被平均分掩盖的短板单独拉出来看。

所以如果你后面要做的是代码助手、数据分析 Copilot、法律问答或医学问答，单看通用 benchmark 是不够的，必须补专项能力评估。

## 为什么预训练评测不能只看一个指标

因为这些指标测量的不是同一件事。

- `loss / perplexity` 更像在看语言建模是否进步
- 通用任务评测更像在看泛化与迁移
- 知识评测更像在看覆盖面和调用能力
- 专项 benchmark 更像在看某个能力面是否过关

如果你只看其中一个，很容易误判。

典型误区有两个：

- 只看 `loss`，把训练变好误当成任务能力全面变好
- 只看单一 benchmark，把局部高分误当成底座全面领先

更稳的做法，是把预训练评测拆成不同层级来看，而不是追一个总分。

## 更实用的理解方式

你可以把预训练模型评测分成三层：

1. `训练层`：loss、perplexity、训练稳定性  
2. `能力层`：知识、泛化、推理、迁移  
3. `专项层`：数学、代码、长上下文、垂类场景

这样你在看模型报告时，就不会把所有分数混成一个模糊印象，而是能回答：

- 这个模型到底是训练质量更好，还是知识覆盖更广
- 它是通用能力强，还是专项能力突出
- 它适不适合作为后续指令微调或业务系统的底座

## 小结

**预训练模型评测，核心是判断底座能力，而不是判断聊天体验。**  
训练指标帮助你看模型有没有学好，通用任务帮助你看泛化和迁移，知识与专项 benchmark 帮助你看能力分布和关键短板。真正靠谱的预训练评估，不会只盯一个分数，而是会把这几层拆开看。
