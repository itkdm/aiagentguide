# 10.3.2 主观评测和客观评测分别适合什么场景？

先给结论：

**客观评测更适合有明确正确答案或可程序化校验的任务，主观评测更适合开放式、体验型、需要整体判断的任务。**

两者不是对立关系，而是分工不同。

## 什么更适合客观评测

如果一个任务有清楚标准，通常优先做客观评测。

例如：

- 分类是否正确
- 抽取字段是否齐全
- 代码是否通过测试
- 检索是否命中目标文档
- 数值计算是否正确

这类任务的好处是：

- 稳定
- 可复现
- 易自动化
- 适合做回归测试

## 什么更适合主观评测

如果任务很难靠唯一答案或规则完全覆盖，就更适合主观评测。

例如：

- 摘要是否自然
- 回答是否有帮助
- 风格是否符合品牌口吻
- 对话是否让用户感觉清楚和顺畅

这类任务的关键不是“唯一正确”，而是“整体质量是否达标”。

## 为什么很多 LLM 任务必须有主观评测

因为大量真实输出是开放式的。

同样一段内容，可以有多种都不错的写法。  
如果你硬要把这种任务全部改写成客观规则，往往会：

- 丢掉质量细节
- 压扁真实体验
- 只剩下容易量化但不重要的部分

所以在开放任务里，主观评测是必要的。

## 为什么也不能只做主观评测

因为主观评测的问题同样明显：

- 成本高
- 一致性差
- 难规模化
- 不容易做回归

如果一个系统完全靠人工主观印象评，迭代会很慢，也很难稳定跟踪变化。

## 一个更实用的划分方法

你可以按这个问题来判断：

1. 这个任务有没有明确标准答案或可自动校验标准？  
2. 如果没有，是否能定义清晰 rubric 让人或 LLM 评审？  

如果第一个问题能答“有”，优先客观评测；  
如果第一个问题答“不好说”，第二个问题答“可以”，那就适合主观评测或 LLM-as-a-Judge。

## 你可以先这样记住

- 有明确答案，优先客观评测
- 没有唯一答案但有清晰标准，适合主观评测或模型裁判
- 真实产品里，很多任务天然就需要两类方法一起上
