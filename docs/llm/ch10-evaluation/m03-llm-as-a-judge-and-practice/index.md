# 10.3 LLM-as-a-Judge 与评测实践

先给结论：**LLM-as-a-Judge 很有用，但它不是“自动评测万能解”。它适合扩大评测规模、覆盖开放式输出，却依然会受偏差、提示设计和评分标准影响，所以必须和客观指标、人工评审一起用。**

这一节的重点，是把评测从“单一方法”推进到“组合方法”。  
随着开放式输出越来越多，很多团队会自然想到让 LLM 来做裁判，因为它比人工便宜、比规则灵活、覆盖面也更广。

这个方向很有价值，但也很容易被用偏。  
如果把 LLM-as-a-Judge 直接当成最终真相，就会忽略几个现实问题：

- 评分本身也会受提示和 rubric 影响
- 某些主观偏好会被裁判模型放大
- 开放式任务和客观任务不适合同一套评法

所以这部分不只是解释一个术语，而是要建立一套更真实的实践判断：  
**评测不是选一个最好用的方法，而是把不同方法组合成一套可信流程。**
