# 10.1.1 为什么 LLM 不能只看感觉好不好用？

先给结论：

**LLM 不能只靠“感觉好不好用”来评，是因为主观体验只会看到少量样例、局部场景和当下印象，很容易高估模型在真实系统里的稳定性与覆盖度。**

一句话说，主观感受可以帮助你发现问题，但不能替代系统化评测。

## 为什么主观感觉特别容易被误导

LLM 有一个很强的迷惑性：它在少量交互里经常会显得非常聪明。

比如它可能：

- 说话自然
- 表达流畅
- 会总结
- 会解释
- 偶尔还能答出很难的问题

这些都会让人很容易产生“这个模型已经够用了”的印象。  
但这只能说明它在你刚刚看到的样例里表现不错，不能说明它在整体任务上稳定可用。

## 主观感觉最大的问题是什么

最大的问题不是“完全没价值”，而是它有几个天然缺陷：

- 样本量太小
- 容易挑中容易被惊艳的例子
- 很难覆盖长尾和边界情况
- 不同人关注点不一样，结果不稳定

所以两个人各问十个问题，得出的模型结论可能完全不同。

## 为什么这对 LLM 尤其危险

因为 LLM 的输出不是固定的，而且任务边界通常很宽。

传统软件很多时候是“功能有没有做出来”；  
LLM 更像“在大量开放输入下，它答得够不够稳、错得多不多、错得危险不危险”。

这类问题如果只靠感觉，很容易漏掉：

- 稳定性问题
- 幻觉问题
- 长尾场景失败
- 格式不一致
- 高风险错误

## 那主观体验还有没有用

有，但它更适合做这些事：

- 早期发现明显问题
- 生成评测样例
- 辅助定义质量标准
- 给离线评测补充人工判断

也就是说，主观体验应该作为评测输入，而不是评测本身。

## 更可靠的做法是什么

更可靠的方式通常是：

1. 先明确任务目标  
2. 再准备代表性样本集  
3. 再定义评分标准  
4. 最后把人工感受和自动指标结合起来  

这样你看到的就不再只是“我觉得挺好”，而是“在这些任务上，它的表现达到什么水平”。

## 常见误区

- 用几个 demo 就决定模型选型
- 只让内部熟悉产品的人试，不看真实用户输入
- 把“说得像样”误当成“答得正确”

## 你可以先这样记住

- 主观感觉适合发现现象，不适合直接下结论
- LLM 评测要看规模、覆盖度和稳定性
- 没有结构化样本和标准，感觉再强也很容易偏
