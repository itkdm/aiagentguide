# 10.1.1 为什么 LLM 不能只看感觉好不好用？

先给结论：**LLM 不能只靠“感觉好不好用”来评价，是因为主观体验只会看到少量样例、局部场景和当下印象，很容易高估模型在真实系统里的稳定性、覆盖度和可上线性。**主观感觉可以提供线索，但不能替代系统化评测。

很多人第一次试一个模型时，最常见的判断就是：

- 这个模型挺聪明
- 回答挺顺
- 看起来比上一个强

这些印象并不一定错，但它们最多只能说明：**在你刚刚试到的那几个样例里，它表现不错。**

## 为什么主观感觉特别容易误导

因为 LLM 天生就擅长制造“它好像真的很懂”的感觉。

它可能会：

- 说话自然
- 表达流畅
- 解释得像模像样
- 偶尔给出很亮眼的回答

这些都很容易让人迅速形成“这个模型已经够用了”的判断。

但这类判断通常忽略了几个关键问题：

- 样本数量太少
- 覆盖面太窄
- 长尾场景没被看到
- 高风险错误没被暴露

所以主观好感和真实质量之间，经常隔着很大一段距离。

## 主观体验最大的局限，不是没价值，而是太局部

主观试用当然有价值，但它的价值更像“发现现象”，而不是“给出结论”。

因为它天然存在几个缺陷：

- 容易挑到容易被惊艳的例子
- 很难覆盖真实输入分布
- 不同人会关注不同点
- 很难稳定复现

结果就是，两个人各试十个问题，最后得到的模型结论可能完全不同。

这在做选型和上线判断时非常危险。

## 为什么这对 LLM 特别明显

因为 LLM 不是传统意义上的固定功能模块，它面对的是大量开放输入。

传统软件很多时候是在看：

- 功能有没有做出来
- 流程能不能跑通

而 LLM 还多了一层问题：

- 在大量不同输入下是否持续稳定
- 在边界样本上会不会失控
- 在高风险问题上错得是否危险

这些问题很难靠“试几轮对话感觉还行”看清楚。

## 主观感觉最容易漏掉什么

如果只靠感觉，最容易漏掉的通常是：

- 长尾失败样本
- 边界分类错误
- 格式不稳定
- 幻觉和高置信错误
- 多轮任务中的累计偏差

也就是说，越影响上线稳定性的东西，越不容易在少量主观试用里被看见。

## 那主观体验还有没有用

有，而且很有用，但它更适合做下面这些事：

- 早期发现明显问题
- 生成评测样例
- 帮助定义质量标准
- 补充自动指标看不到的体验信息

换句话说，主观体验应该作为评测输入，而不是评测结论本身。

## 更可靠的做法是什么

更可靠的方式通常是：

1. 先明确任务目标。
2. 再准备有代表性的样本集。
3. 再定义清楚评分标准。
4. 最后把人工体验和结构化指标结合起来看。

这样你得到的就不再只是：

- “我觉得挺好”

而是：

- “在这些任务、这些样本、这些标准下，它表现到了什么水平”

这两者完全不是一回事。

## 一个常见误区：把 Demo 效果当成可上线结论

这非常常见。

Demo 往往只展示：

- 精选样本
- 顺利路径
- 最容易出效果的场景

它可以证明“这个方向值得做”，但通常不能证明“这个系统已经可以上线”。

上线判断一定要看：

- 长尾覆盖
- 稳定性
- 高风险错误
- 回归表现

这些信息，主观感觉通常给不全。

## 小结

**主观感觉适合发现现象，不适合直接下评测结论。**对 LLM 来说，“看起来不错”只能说明它在少量场景里让人满意，不能说明它已经在真实任务里稳定可用。真正靠谱的判断，必须落到结构化样本、明确标准和可复现评测上。
