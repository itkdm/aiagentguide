# 10.1 为什么不能只靠主观感觉评模型

先给结论：**“感觉这个模型挺聪明”只能作为最初印象，不能作为评测结论。因为 LLM 很容易在少量样例里表现惊艳，却在规模化、长尾样本和真实流程里暴露问题。**

这一节的作用，是先把“评模型”这件事从直觉判断里拉出来。  
在 LLM 场景里，主观体验尤其容易误导人，因为模型非常擅长制造一种“它好像真的懂”的感觉。

这会带来几个常见误判：

- demo 跑得好，就以为可以上线
- 回答听起来聪明，就以为任务完成率也高
- 团队内部少量主观试用不错，就以为可以支持模型选型

但这些结论都不稳。  
因为真实系统里真正决定成败的，往往是长尾问题、重复稳定性、流程完成率和线上用户体验，而不是某几次对话是否让人觉得“挺会说”。

这一节会先帮你建立一个评测基本防线：  
**主观感受可以作为线索，但不能替代评测结论。**

这一步非常关键。  
如果这一层没有先立住，后面不管是 benchmark、LLM-as-a-Judge 还是线上监控，都会很容易被拿去“证明既有印象”，而不是做真正的质量判断。
