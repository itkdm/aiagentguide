# 10.1.3 为什么模型评测必须区分离线能力和线上效果？

先给结论：**离线评测回答的是“模型或系统在受控样本上表现如何”，线上评估回答的是“系统在真实流量里是否真的创造价值”。**这两个问题相关，但不能互相替代。

如果你只做其中一个，评测结论通常都会失真。

## 什么是离线评测

离线评测通常是在预先准备好的数据集上，对模型或系统做批量测试。

它的特点是：

- 输入可复现
- 标准可对齐
- 方便横向比较
- 适合做版本回归

它最擅长回答的问题是：

- 哪个模型在同一任务上更强
- 改了 Prompt / RAG / 工具后有没有变好
- 哪些样本类型依然是短板

所以离线评测更像“实验室里的能力测量”。

## 什么是线上评估

线上评估关注的是系统进入真实使用场景后的表现。

它常看的指标包括：

- 用户满意度
- 任务完成率
- 人工接管率
- 错误率和投诉率
- 延迟、成本和转化效果

它回答的问题是：

- 这个系统在真实用户和真实流量里有没有实际收益

所以线上评估更像“生产环境里的业务结果判断”。

## 为什么离线好，不代表线上也好

因为线上环境远比离线复杂。

真实流量里会多出很多变量：

- 用户提问方式更乱
- 长尾样本更多
- 检索质量会波动
- 工具调用会失败
- 用户是否接受回答风格也会影响结果

这会导致一个很常见的现象：

- 离线分数很好
- 但线上表现一般

不是离线评测没用，而是它回答的是另一个层面的问题。

## 为什么线上好，也不代表可以不做离线

因为没有离线评测，你几乎没法稳定迭代。

如果缺少离线集，团队通常会遇到这些问题：

- 改了之后不知道到底哪里变好了
- 很难复现线上问题
- 很难做版本对比
- 很难发现回归

线上数据告诉你“现在怎么样”，离线评测帮助你“稳定改下去”。

这两者的职责完全不同。

## 一个更合理的组合方式

真实项目里，更稳的评测组合通常是：

1. `离线评测`：做版本比较、回归测试、发现短板。
2. `线上评估`：看真实用户和业务指标是否改善。
3. `人工反馈`：补自动指标看不到的问题。

这三者结合，才更接近完整评测体系。

## 一个常见误区：把 benchmark 当成线上效果代理

这很容易让团队高估模型。

Benchmark 和离线集通常只能说明：

- 在这些固定样本上表现如何

但真实系统上线之后，还要面对：

- 检索和工具链
- 用户行为
- 成本和时延
- 业务约束

这些因素都可能让一个“离线高分方案”在真实环境里不够理想。

## 一个更务实的判断方法

如果你要判断评测体系是不是健康，可以问：

- 有没有能稳定复现的离线集？
- 有没有覆盖真实流量关键场景？
- 有没有线上指标能反映业务效果？
- 有没有机制把线上问题回灌到离线集？

如果这四点缺很多，评测体系通常就还不完整。

## 小结

**离线评测看能力和回归，线上评估看真实使用价值。**前者帮你比较和迭代，后者帮你判断是否真的创造业务收益。少了任何一边，模型评估都会偏。真正稳的团队，一定会把两者接成闭环，而不是只盯着其中一个分数或感觉。
