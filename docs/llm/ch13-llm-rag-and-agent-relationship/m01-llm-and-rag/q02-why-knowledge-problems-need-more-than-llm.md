# 13.1.2 为什么很多“知识型问题”只靠 LLM 不够？

先给结论：

**很多知识型问题只靠 LLM 不够，不是因为模型不会说，而是因为这类问题真正要求的是：事实新、依据准、边界清、回答可追溯。**

而这些要求，很多都超出了“靠模型参数直接回忆”能稳定满足的范围。

## 知识型问题和开放生成问题不一样

如果你让模型写一段文案、润色一句话、做一个摘要，主要考的是语言组织能力。  
但知识型问题通常更关心：

- 事实准不准
- 依据是不是最新
- 是否引用了正确材料
- 是否混入了模型自己的补全

所以它更像“事实处理问题”，而不只是“文本生成问题”。

## 为什么参数里的知识不够稳定

模型参数里的知识有几个天然局限：

- 记忆不是数据库式精确检索
- 更新速度慢
- 对长尾事实不稳定
- 容易把相似知识混在一起

所以知识型问题一旦对细节要求高，单靠模型就容易出现：

- 过时
- 幻觉
- 细节混淆
- 高置信错误

## 企业知识问题更明显

在企业场景里，这个问题会更严重，因为知识往往：

- 私有
- 动态变化
- 有权限边界
- 有版本差异

这些都不是“模型多训一点”就自然解决的。

## 一个更实用的理解

知识型问题通常需要三件事一起成立：

1. 先拿到正确知识  
2. 再约束模型基于这些知识回答  
3. 最后让答案可追溯、可验证  

只靠 LLM，通常只能做好第三步的一部分表达组织，前两步往往需要 RAG 或工具补齐。

## 你可以先这样记住

- 知识型问题的难点不是“会不会说”，而是“有没有拿到正确事实”
- 只靠参数回忆，很难长期稳定满足新、准、可追溯这三个要求
- 这就是为什么知识型问题往往需要 LLM 之外的系统能力
