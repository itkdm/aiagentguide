# 13.1 LLM 与 RAG 的关系

先给结论：**LLM 负责“理解和生成”，RAG 负责“把外部知识喂给模型”。有了强 LLM 以后，RAG 依然重要，因为很多问题的瓶颈不是语言能力，而是事实来源。**

这一节要先处理一个非常常见的误解：  
模型越来越强、上下文越来越长，是不是就意味着 RAG 的价值在下降，甚至慢慢不需要了？

这个判断在少量场景里可能成立，但放到真实系统里通常并不稳。  
因为 LLM 再强，仍然主要依赖参数中的既有知识分布；而很多业务问题真正卡住的地方，不在语言生成，而在于：

- 知识是不是最新
- 内容是不是私有
- 回答能不能追溯来源
- 输入信息是不是已经被正确筛出来

所以这一节的重点不是简单比较“谁更高级”，而是先把这两层的职责拆开：  
**LLM 解决表达和推理，RAG 解决知识供给和信息定位。**
