# 13.1.3 哪些问题更适合纯 LLM，哪些更适合 LLM + RAG？

先给结论：

**纯 LLM 更适合语言组织、总结、改写、一般推理等不强依赖外部事实的问题；LLM + RAG 更适合依赖最新知识、私有知识、可追溯来源和大规模文档语料的问题。**

换句话说，先问“任务主难点是什么”，就能大致判断该走哪条路。

## 更适合纯 LLM 的问题

纯 LLM 通常更适合：

- 改写和润色
- 摘要与归纳
- 文案生成
- 一般性解释
- 不依赖私有或最新资料的推理

这些任务的重点是语言能力本身，不是外部知识供给。

## 更适合 LLM + RAG 的问题

LLM + RAG 通常更适合：

- 企业知识问答
- 产品和制度文档查询
- 法规、条款、手册解释
- 需要回答时给出依据来源的问题
- 依赖持续更新信息的问题

这类任务的主难点，是“先把正确材料找出来”，再让模型表达。

## 最容易误判的中间地带

现实里，很多任务不是纯一边。

例如：

- 先基于内部资料回答
- 再做总结、比较、改写

这类任务其实是：

1. RAG 负责补事实  
2. LLM 负责组织表达  

所以它本质上是组合问题，而不是非此即彼。

## 一个更务实的判断法

你可以用这几个问题快速判断：

1. 回答是否强依赖外部资料？
2. 资料是否最新、私有或需要追溯？
3. 问题的主要价值是在“拿事实”，还是在“组织答案”？

如果前两项都强，通常更偏 LLM + RAG；  
如果第三项更偏“组织答案”，通常纯 LLM 就可能足够。

## 你可以先这样记住

- 主难点在表达，纯 LLM 更合适
- 主难点在知识供给，LLM + RAG 更合适
- 很多真实任务其实是“RAG 供事实，LLM 负责生成”的组合模式
