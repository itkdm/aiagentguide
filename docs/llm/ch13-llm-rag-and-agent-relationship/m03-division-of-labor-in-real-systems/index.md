# 13.3 三者在实际系统中的分工

先给结论：**真实系统里最重要的不是“有没有把概念都用上”，而是每一层都做自己最擅长的事。LLM 负责理解与生成，RAG 负责补知识，工具负责拿真实数据和执行动作，Workflow 负责稳定流程，Agent 负责处理需要动态决策的多步任务。**

这一节是前面两节的收束。  
前面已经把 `LLM vs RAG`、`RAG vs Agent` 拆开了，这里要再往前走一步：  
**把这些能力放回同一个真实系统里，看它们到底应该怎么分工。**

这一步很重要，因为很多系统设计失误都不是出在单点能力不够，而是出在职责分配错了。  
常见问题包括：

- 明明应该用工具拿真实状态，却让模型凭语言能力硬猜
- 明明是知识供给问题，却一直改 Prompt
- 明明是固定流程问题，却过早升级成 Agent

所以这部分的重点，不是再讲一次概念定义，而是帮助你建立一个更实用的架构判断：  
在真实系统里，哪些事该让模型做，哪些事该交给检索、工具、Workflow 或 Agent。
