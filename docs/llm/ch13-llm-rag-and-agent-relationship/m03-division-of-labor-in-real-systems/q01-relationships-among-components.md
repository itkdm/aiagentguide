# 13.3.1 LLM、RAG、Tool Use、Workflow、Agent 之间是什么关系？

先给结论：

**可以把这几层粗略理解成：LLM 是认知与生成核心，RAG 是知识补给层，Tool Use 是外部能力接口，Workflow 是固定流程编排，Agent 是在这些能力之上做动态决策的执行层。**

它们不是互斥关系，更像不同层级的组件。

## LLM 在系统里扮演什么

LLM 最擅长的是：

- 理解输入
- 组织语言
- 做一般推理
- 输出自然表达

所以它通常是“认知核心”，但不是完整系统本身。

## RAG 在系统里扮演什么

RAG 解决的是：

- 模型没有外部知识
- 需要基于资料回答
- 需要可追溯事实来源

所以它更像“知识增强层”。

## Tool Use 在系统里扮演什么

工具调用解决的是：

- 查数据库
- 调 API
- 执行动作
- 获取实时状态

它让系统不只是会说，还能拿到真数据、做真动作。

## Workflow 在系统里扮演什么

Workflow 适合处理：

- 规则明确
- 步骤固定
- 可以用顺序流程定义清楚的任务

它的优势是稳定、可控、可测试。

## Agent 在系统里扮演什么

Agent 通常是在前面这些能力之上，再加一层：

- 动态判断
- 任务分解
- 工具选择
- 中间状态管理

它更像“调度者”和“执行推进者”。

## 一个更实用的理解

你可以把它们看成从底到顶的分层：

1. LLM：理解与生成  
2. RAG / Tool Use：知识和动作接口  
3. Workflow：固定编排  
4. Agent：动态编排与闭环执行  

层越往上，灵活性越强，但复杂度也越高。

## 你可以先这样记住

- LLM 是核心能力
- RAG 和工具是补充外部世界的接口
- Workflow 让流程稳定
- Agent 让系统在复杂目标下动态行动
