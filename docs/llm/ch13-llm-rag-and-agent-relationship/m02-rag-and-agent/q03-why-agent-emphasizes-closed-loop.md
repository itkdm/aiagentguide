# 13.2.3 为什么 Agent 更强调任务分解、工具调用和执行闭环？

先给结论：

**Agent 更强调任务分解、工具调用和执行闭环，是因为它要解决的不是“回答一个问题”，而是“把一个目标真正推进到完成”。**

一旦目标不是单步可解，系统就必须考虑中间状态、动作选择和结果反馈。

## 为什么要做任务分解

很多任务天然不是一句话就能完成。

例如：

- 先理解需求
- 再查资料
- 再调用系统
- 再判断结果是否满足要求

如果不拆步，系统就会把所有复杂度一次性压给模型，稳定性通常会更差。

## 为什么要调用工具

因为很多任务要完成，不只是生成文字，还需要：

- 查数据库
- 发消息
- 调 API
- 做计算
- 访问文件和系统状态

这些事情不是靠“生成一段很像真的文本”就能完成的。  
所以 Agent 必须把外部工具纳入执行链路。

## 为什么要做执行闭环

闭环的意思是：  
系统不只“给出一步输出”，还要看这一步结果是否满足目标，并决定下一步要不要继续。

例如：

- 检索结果不够，就再检一次
- 工具报错了，就改参数重试
- 信息还缺失，就继续追问

这和普通问答系统最大的不同，就是它会围绕目标持续行动，而不是一次输出就结束。

## 一个更务实的理解

如果 RAG 更像“查资料后回答”，  
Agent 更像“为了完成目标，查资料只是其中一步”。

所以 Agent 天然会更强调：

- 分步
- 状态
- 工具
- 反馈
- 闭环

## 你可以先这样记住

- 任务越复杂，越需要分解
- 目标越接近真实执行，越需要工具
- 一旦要把任务推进到完成，闭环就会比单轮回答更重要
