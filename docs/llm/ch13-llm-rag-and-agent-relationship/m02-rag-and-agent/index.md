# 13.2 RAG 与 Agent 的关系

先给结论：**RAG 解决的是“怎么给模型补知识”，Agent 解决的是“怎么围绕目标做多步决策和执行”。有检索能力，并不自动等于系统已经具备 Agent 能力。**

这一节的重点，是把“检索增强”和“任务执行”这两种能力严格拆开。  
很多团队在系统演化过程中，容易出现一个典型误判：  
系统已经能查知识、能拼上下文、能回答复杂问题，于是就觉得自己已经做出了 Agent。

但真实情况通常不是这样。  
RAG 更像知识供给层，它负责把对的资料找出来、喂给模型；而 Agent 关心的是：

- 任务怎么拆
- 下一步做什么
- 要不要调用工具
- 怎样根据中间结果继续推进

这两者可以组合，但并不等价。

所以这一节不是在强调某个概念更新潮，而是在建立一个非常关键的系统边界：  
**有检索，不等于有自主任务闭环。**
