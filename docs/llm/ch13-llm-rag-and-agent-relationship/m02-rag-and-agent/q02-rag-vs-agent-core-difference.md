# 13.2.2 RAG 和 Agent 的核心差别是什么？

先给结论：

**RAG 的核心角色是“知识增强器”，Agent 的核心角色是“任务执行器”。一个主要解决模型知道什么，一个主要解决系统接下来做什么。**

这就是两者最本质的区别。

## RAG 的核心目标

RAG 最主要的目标是：

- 找到外部知识
- 把知识提供给模型
- 降低知识缺失和幻觉

它更像在解决“信息供给链路”问题。

## Agent 的核心目标

Agent 最主要的目标是：

- 围绕目标规划步骤
- 决定何时调用什么工具
- 处理多步依赖和中间状态
- 让任务形成执行闭环

它更像在解决“行动和决策链路”问题。

## 为什么两者常一起出现

因为很多复杂 Agent 在执行任务时，也需要外部知识。  
这时 RAG 往往会作为 Agent 的一个子能力存在。

例如：

- Agent 先判断要不要检索
- 检索拿到资料
- Agent 再决定是继续检索、调用工具，还是直接输出

这里的 RAG 是组件，Agent 是编排者。

## 一个更工程化的理解

你可以把两者粗略理解成：

- RAG 负责“把对的资料拿来”
- Agent 负责“决定如何利用这些资料完成任务”

这也是为什么它们看起来常常相关，但本质上不是同一层能力。

## 你可以先这样记住

- RAG 偏知识增强
- Agent 偏任务编排和执行
- 在复杂系统里，RAG 常常是 Agent 的一部分，而不是反过来
