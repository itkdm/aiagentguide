# 9.3.1 如何降低幻觉？

先给结论：

**降低幻觉最有效的思路，不是单纯让模型“更聪明”，而是同时做到三件事：减少它瞎猜的空间、给它足够依据、在输出后增加验证。**

这三件事分别对应：输入约束、信息补充、结果校验。

## 第一层：减少模型瞎猜的空间

如果你不给边界，模型就容易在不确定时继续补。

常见做法包括：

- 明确要求“只基于提供材料回答”
- 明确要求“没有依据时说不知道”
- 明确要求区分“已知事实”和“推断”
- 给出稳定输出格式，避免自由发挥过大

这类方法的作用，是先把模型的搜索空间收紧。

## 第二层：给模型足够依据

很多幻觉并不是模型突然乱来，而是它根本没有足够材料还被要求作答。

所以事实型任务通常要优先补依据，例如：

- 用 RAG 提供相关文档
- 用工具查实时数据
- 给结构化上下文而不是一段模糊描述
- 对时间敏感问题强制外部查询

如果依据没给够，只靠 prompt 往往治标不治本。

## 第三层：在输出后增加校验

即使模型前面已经拿到了材料，也不代表它一定会用对。

所以在高风险系统里，常见还会加一层后处理：

- 检查回答是否引用了给定来源
- 检查关键字段是否缺失或冲突
- 对关键事实做二次查询
- 对高风险结果触发人工审核

这层不是为了让模型“更会答”，而是为了防止错误直接流到用户面前。

## 一个实用治理顺序

如果你要排查幻觉，建议按这个顺序做：

1. 先看问题是否要求了超出已知依据的回答
2. 再看上下文里是否真的提供了足够材料
3. 再看 prompt 是否允许模型拒答和标记不确定
4. 最后看输出后是否有校验和兜底

很多团队一上来就改 prompt，但前两步才经常是主因。

## 什么方法最容易被高估

最容易被高估的是“多加几句约束词”。  
它确实有帮助，但帮助通常建立在两个前提上：

- 模型本来就接近能答对
- 系统已经给了足够依据

如果这两个前提不成立，约束词只能让错答案换一种更克制的说法。

## 你可以先这样记住

- 降幻觉靠三层：约束、补依据、做校验
- 对事实型任务，依据往往比措辞更重要
- 真正稳定的系统，通常允许“不知道”，也允许多一层验证
