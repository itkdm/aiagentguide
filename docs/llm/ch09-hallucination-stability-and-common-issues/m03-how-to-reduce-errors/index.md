# 9.3 如何降低错误

先给结论：**降低错误不是只靠“换个更强的模型”或“多写几句 Prompt”，而是要先判断错误来自哪里，再决定该用提示约束、检索补充、工具查询、结果校验，还是直接改系统链路。**

这一节是前两节的收束。  
前面已经把幻觉和稳定性问题拆开了，这一节要继续往前走一步：  
**当你已经知道错误有很多种，真实系统里应该怎么治理。**

这里最常见的误区有两个：

- 把所有问题都压给 Prompt
- 把所有问题都归结成“模型不够强”

这两种处理方式都太粗。真实系统里，错误往往是多层叠加的：

- 输入表达可能不清楚
- 资料供给可能不对
- 工具调用可能缺位
- 结果校验可能没有建立

所以这一节不是在列一堆零散技巧，而是帮你建立一套分层治理思路：  
先判断错误发生在哪一层，再决定该修输入、补知识、接工具、加校验，还是重构链路。
