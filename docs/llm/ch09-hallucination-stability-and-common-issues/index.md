# 第 9 章 幻觉、稳定性与常见问题

先给结论：**LLM 出错并不只是一种情况。你看到的“答错”，可能来自幻觉、知识过时、上下文不足、提示不清、采样波动，或者系统链路本身没有把约束建好。**

这一章解决的不是“模型偶尔会胡说”这么一句空话，而是帮你把常见错误拆开。只有先分清错误类型，后面谈评测、调参、RAG、工具调用和上线防护才不会混在一起。

## 你应该建立的判断

- 幻觉不是“所有错误”的统称，它更接近“模型生成了看似合理但并不真实或无依据的内容”
- 知识过时、检索不到、工具失败、上下文截断，都可能表现为“答错”，但治理方法不同
- 模型说得越流畅，不代表它越可靠；“高置信错误”往往比明显答不上来更危险
- 降低错误不是只靠一个 prompt，而是要同时看提示、检索、工具、评测和兜底策略
