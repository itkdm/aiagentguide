# 9.1 什么是幻觉

先给结论：**“幻觉”不是指模型所有答错的情况，而是指模型生成了看起来通顺、像是真的、但实际上不成立或没有依据的内容。**

这一节的核心任务，是先把“幻觉”这个词从泛化滥用里救出来。  
在很多讨论里，只要模型答错了，就会被统称成“幻觉”。但如果概念不分清，后面所有治理动作都会变得很粗糙。

因为不同类型的错误，根因完全不同：

- 有些是模型无中生有
- 有些是知识过时
- 有些是上下文没给够
- 有些是检索拿错资料

它们看起来都像“答错”，但系统应对方式并不一样。

所以这一节不是只做定义解释，而是先把边界立住：  
**什么才算幻觉，为什么它比普通错误更危险，以及为什么“一本正经地胡说”会成为真实系统里的高风险问题。**

## 读完后你应该能判断

- 一条回答是“无中生有”，还是“知识没更新”
- 一次错误更像模型内部生成问题，还是外部系统没有提供足够依据
- 为什么“答得很像真的”会比“直接说不知道”更难治理

## 这一节适合怎么读

- 如果你以前把所有错误都一股脑叫成“幻觉”，这一节要先读透
- 如果你后面会看错误治理、评测和系统补偿，这一节是分类基础
- 如果你关心的是高风险业务场景，这一节能帮你更准确地区分危险错误类型
