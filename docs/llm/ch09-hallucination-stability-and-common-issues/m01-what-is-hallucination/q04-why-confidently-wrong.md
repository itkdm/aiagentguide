# 9.1.4 为什么模型有时会一本正经地胡说八道？

先给结论：

**模型会“一本正经地胡说八道”，是因为它优化的是生成连贯、像样、符合指令的回答，而不是天然优化“只在确认无误时才表达得很确定”。**

所以“语气很稳”与“事实很真”之间，没有必然关系。

## 为什么错答案也能说得很像真话

模型经过预训练、指令微调和对齐后，已经学会了很多“好答案的外形”：

- 结构完整
- 语气自然
- 条理清楚
- 会补背景、原因和结论

这让它即使在事实基础不足时，也能生成一段“看上去很专业”的文本。

## 高置信错误通常怎么形成

常见链路是这样的：

1. 用户问题里有缺口，或者系统没有给足依据  
2. 模型基于相似模式先补出一个局部判断  
3. 后续 token 继续围绕这个判断展开  
4. 最终形成一段自洽但并不真实的回答  

一旦第一步偏了，后面越写越顺，错误反而更难被肉眼察觉。

## 为什么对齐后仍然会这样

很多人会以为：做了指令微调和对齐后，模型就应该更谨慎。  
但现实里，对齐通常会让模型：

- 更愿意回答
- 更像一个“会配合的助手”
- 更能把话说完整

这会提升可用性，但如果没有同时强化“不确定就停”“没证据就别补”，也可能让错误回答显得更可信。

## 为什么这类错误比普通错误更危险

因为它的误导性更强。

如果模型直接说“我不知道”，用户会继续查证；  
但如果模型给出一段完整解释，还带上术语、步骤、结论，很多人会默认它是可靠的。

在真实系统里，这类错误尤其容易出现在：

- 法律、医疗、金融等高风险问答
- 长文总结和报告生成
- 引用资料、版本信息、配置参数这类细节任务

## 工程上怎么压低这种问题

不要只要求“回答得好”，还要明确要求“有依据再答”。

更有效的控制通常包括：

- 允许并鼓励模型说“不确定”
- 对事实型回答要求引用依据
- 将回答限制在给定材料内
- 用检索或工具补充外部事实
- 对高风险任务加校验、重试或人工审核

## 自查方法

如果你发现系统常常“答得越像专家，错得越难发现”，可以重点检查：

- prompt 是否把“完整回答”放在“谨慎回答”之前
- 评测是否只奖惩“答对率”，没有奖惩“拒答质量”
- 检索和工具是否真的给到了可验证依据
- 输出是否缺少引用、证据或可回查路径

## 你可以先这样记住

- 模型的自信语气，本质上是一种生成风格，不是真实性的证明
- 高置信错误往往来自“先补一个判断，再顺着写下去”
- 要防这种问题，核心不是让模型更会说，而是让它更愿意在没把握时停下来
