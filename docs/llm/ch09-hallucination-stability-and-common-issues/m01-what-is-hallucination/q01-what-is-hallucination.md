# 9.1.1 什么是幻觉（Hallucination）？

先给结论：

**幻觉是指模型生成了看起来合理、语法也通顺，但内容并不真实、没有依据，或者与已知事实不一致的输出。**

关键点不在“它答错了”，而在“它像是真的”。如果一句话明显是乱码，读者通常不会信；但如果一句话结构完整、措辞自然、细节很多，哪怕它是错的，也很容易被当成真的。

## 幻觉通常长什么样

在实际系统里，幻觉常见有几种表现：

- 编造不存在的事实、人物、论文、接口或参数
- 把真实信息拼错，例如把两个事件、两个产品版本混在一起
- 在没有证据时补全细节，比如擅自补地点、时间、因果关系
- 引用看起来像真的来源，但来源本身并不存在或并不支持结论

这些输出的问题不是“句子生成失败”，而是“内容生成得太像正确答案了”。

## 不要把所有错误都叫幻觉

工程上更有用的做法，是把下面几类情况分开：

- **幻觉**：模型无依据地生成内容
- **知识过时**：模型记住了旧信息，但没有最新事实
- **检索失败**：系统本该取外部知识，但没有召回到关键材料
- **工具调用失败**：外部 API 返回异常、超时或参数错误
- **理解偏差**：用户问的是 A，模型按 B 去回答

如果你把这些都混成“幻觉”，后面就很难定位问题。

## 为什么这个定义重要

因为不同错误对应不同治理手段。

如果问题是知识过时，你更该补的是检索、工具或数据更新；  
如果问题是幻觉，你更该补的是依据约束、引用检查、拒答策略和输出校验。

## 一个简单判断法

看到一条可疑回答时，可以先问三个问题：

1. 这条信息有没有明确依据？
2. 这条信息能不能被当前上下文或外部来源验证？
3. 如果没有依据，模型是不是仍然给了非常确定的表达？

如果三者里前两项都答不上来，而第三项又很强，那它大概率就是幻觉。

## 一个最小例子

用户问：“某篇论文的作者是谁？”

- 如果模型回答“我不确定，当前上下文没有给出作者信息”，这不是幻觉。
- 如果模型回答出一个完整作者名单，但名单并不存在，这更像幻觉。
- 如果作者名单曾经正确，但论文后续版本有更新而模型没跟上，这更像知识过时。

## 常见误区

- 把“任何错误”都叫幻觉
- 只看语气是否自信，不看有没有依据
- 以为幻觉只是模型能力不够，忽略系统没有提供事实约束

## 你可以先这样记住

- 幻觉的核心不是“错”，而是“无依据却像真的”
- 幻觉只是错误的一种，不等于所有事实性问题
- 工程上先分型，再治理，比笼统说“减少幻觉”更有用
