# 9.2.3 为什么模型输出会受 Prompt、采样参数、上下文组织影响？

先给结论：

**模型输出受 Prompt、采样参数、上下文组织影响，是因为这些内容本来就在决定“模型此刻以什么目标、在什么信息范围里、按多大随机性去继续生成”。**

所以它不是“太敏感”，而是这些变量本来就属于输入条件的一部分。

## Prompt 为什么会影响输出

Prompt 不只是“提问文字”，它实际上在告诉模型：

- 你现在要做什么任务
- 优先遵守什么约束
- 要输出成什么格式
- 遇到不确定情况该怎么处理

如果 prompt 模糊，模型就要自己补理解；  
如果 prompt 明确，模型的搜索空间会更小，输出通常更稳。

## 采样参数为什么会影响输出

采样参数决定了模型在候选 token 之间有多“敢于变化”。

你可以把它们粗略理解成：

- `temperature`：让分布更保守还是更发散
- `top-p` / `top-k`：候选范围收多紧

参数越开放，表达越丰富，但一致性通常更差；  
参数越保守，输出越收敛，但也更容易僵硬或模板化。

所以它不是“哪个参数永远最好”，而是看任务要稳定还是要发散。

## 上下文组织为什么会影响输出

同样一堆信息，顺序和组织方式不同，也会影响模型注意到什么、先依赖什么。

常见影响包括：

- 关键约束放太后面，模型容易忽略
- 长上下文里中间信息容易被弱化
- few-shot 示例顺序会暗示输出模式
- 检索结果混杂时，模型可能抓错重点

这就是为什么“给了同样材料”不代表“模型一定会同样使用”。

## 一个工程化理解

可以把这三类变量看成三层控制：

1. **Prompt** 决定任务目标和行为规则
2. **上下文** 决定事实依据和参考材料
3. **采样参数** 决定最终生成时的发散程度

只要其中任意一层变了，最终输出都可能变。

## 什么时候最容易被放大

这种敏感性在以下场景里更明显：

- 任务本身定义模糊
- 要求很多但优先级没讲清
- 长上下文材料质量参差不齐
- 边界案例很多
- 输出格式约束很严

因为这时模型本来就需要在多个可能方向里做选择。

## 怎么让它更稳

更稳的常见做法包括：

- 把任务目标写清楚
- 把格式要求写清楚
- 把优先级写清楚
- 把关键依据前置
- 减少无关上下文
- 根据任务下调采样随机性

## 你可以先这样记住

- Prompt 决定“怎么做”
- 上下文决定“基于什么做”
- 采样参数决定“做得有多收敛”
- 输出变化，往往就是这三层里有一层没被控住
