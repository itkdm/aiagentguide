# 9.2.3 为什么模型输出会受 Prompt、采样参数、上下文组织影响？

先给结论：**模型输出会受 Prompt、采样参数、上下文组织影响，是因为这些内容本来就在决定模型“现在要做什么、基于什么做、以及以多大发散度继续生成”。**它不是“太敏感”，而是这些变量本来就属于输入条件的一部分。

很多人会把这种现象理解成：

- 模型不稳定
- 模型太脆弱
- 模型很玄学

更工程化的理解是：**你改了条件，输出跟着变，本来就是合理的。**真正要问的不是“为什么会变”，而是“哪些变化是合理波动，哪些变化说明系统约束不够”。

## Prompt 为什么会影响输出

Prompt 不只是“提问文本”，它实际上在告诉模型：

- 现在要完成什么任务
- 哪些约束优先
- 输出应该长什么样
- 不确定时该怎么处理

如果 Prompt 模糊，模型就需要自己补理解。

如果 Prompt 明确，模型的搜索空间就会缩小，输出通常更稳定。

所以 Prompt 的作用不是“神奇咒语”，而是任务定义接口。

## 采样参数为什么会影响输出

采样参数决定了模型在候选 token 之间有多保守或多发散。

可以粗略理解成：

- `temperature` 控制分布是更收敛还是更发散
- `top-p` / `top-k` 控制候选范围收得多紧

参数更开放时：

- 表达可能更丰富
- 创造性可能更强
- 但一致性通常更差

参数更保守时：

- 输出更收敛
- 结构更稳定
- 但也更容易僵硬、模板化

所以参数没有绝对最优，只有和任务是否匹配。

## 上下文组织为什么也会影响输出

即使给模型的是同一批材料，顺序和组织方式不同，模型的关注点也会不同。

常见影响包括：

- 关键约束放得太后，模型更容易忽略
- 长上下文中间内容更容易被弱化
- few-shot 示例顺序会暗示输出模式
- 检索结果混杂时，模型可能抓错重点

这就是为什么“给了同样材料”不等于“模型一定会同样使用这些材料”。

## 一个更工程化的三层理解

可以把这三类变量理解成三层控制：

1. `Prompt` 决定任务目标和行为规则。
2. `上下文` 决定事实依据和参考材料。
3. `采样参数` 决定最终生成时的发散程度。

只要其中任何一层变了，最终输出都可能变。

这也是为什么稳定性问题不能只盯着一个变量看。

## 哪些场景最容易被放大

这种敏感性在下面几类任务里更明显：

- 任务定义本身模糊
- 约束很多但优先级没讲清
- 长上下文材料质量参差不齐
- 边界样本很多
- 输出格式要求很严

因为这些场景本来就需要模型在多个可能方向里做选择。

一旦条件略有变化，结果就更容易偏。

## 怎么让系统更稳

更稳的常见做法包括：

- 把任务目标写清楚
- 把格式要求写清楚
- 把优先级写清楚
- 把关键依据前置
- 减少无关上下文
- 根据任务下调采样发散度

重点不是让输出完全不变，而是让关键字段、关键结论和关键格式稳定下来。

## 一个常见误区：只改 Prompt，不看上下文和参数

很多团队一发现结果变了，就只盯着 Prompt 调。

但真实系统里，经常真正变化的是：

- 检索结果
- 会话历史
- 工具返回
- 采样配置

如果这些不固定，单独讨论 Prompt 是否“有效”其实意义不大。

## 一个简单记忆法

可以先这样记：

- Prompt 决定“怎么做”
- 上下文决定“基于什么做”
- 采样参数决定“做得有多收敛”

理解了这三层，很多“为什么结果又变了”的问题就不会显得玄学。

## 小结

**模型输出受 Prompt、上下文和采样参数影响，不是异常，而是输入条件控制的自然结果。**真正的工程目标不是消灭所有变化，而是把关键变化压进可控范围，让系统在事实、格式和核心结论上保持稳定。
