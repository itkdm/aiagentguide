# 9.2 稳定性问题

先给结论：**LLM 的“不稳定”并不只是随机波动。你看到的时好时坏，可能来自采样随机性，也可能来自提示不清、上下文组织变化、边界样本过多，或者模型本身在某类任务上没有稳定决策边界。**

这一节的重点，是把“不稳定”从一句模糊抱怨变成几个可拆解、可处理的问题。

很多团队在观察到模型时好时坏后，第一反应往往是：

- 模型随机性太强
- 多试几次就行
- 继续改 Prompt

这些判断有时成立，但并不总成立。  
因为所谓稳定性问题，背后可能是完全不同的几类原因：

- 采样带来的自然波动
- 任务描述不够清楚
- 上下文组织方式变化
- 模型本身对边界样本没有稳定判别能力

这一节会把这些来源拆开，让你建立一个更工程化的判断：  
**不是先问“稳不稳”，而是先问“哪一类不稳定”。**

## 读完后你应该能判断

- 一次波动更像采样随机性，还是任务本身没有被说明白
- 同一个问题反复出错，到底是偶发抖动还是稳定缺陷
- 为什么“微小改 prompt 就变结果”往往说明系统约束还不够清楚

## 这一节适合怎么读

- 如果你经常把“偶发波动”和“系统性缺陷”混在一起，这一节要重点看
- 如果你后面会做评测、回归测试或质量排查，这一节是定位前提
- 如果你关心的是怎么让系统更可预测，而不是偶尔跑出好结果，这一节会更有价值
