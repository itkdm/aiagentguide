# 9.2.2 什么是随机性，什么是系统性错误？

先给结论：

**随机性问题是“同样条件下结果会抖”，系统性错误是“它总是在某类地方错”。前者更像波动，后者更像结构性缺陷。**

如果你分不清这两类问题，就很容易修错方向。

## 什么叫随机性

随机性通常指：在输入近似相同的前提下，模型输出存在一定自然波动。

常见来源包括：

- 采样参数不是完全确定性
- 候选 token 概率接近，选哪个都说得过去
- 上下文中存在多种合理表达方式

这类问题的特征是：

- 并不是每次都错
- 常常只是措辞、结构或局部细节变化
- 多跑几次可能有的好、有的差

## 什么叫系统性错误

系统性错误是指：只要碰到某类输入，模型就经常在类似位置出错。

比如：

- 总是漏抽某个字段
- 总是把两个概念混淆
- 一遇到边界案例就分类错误
- 长文里总是漏掉中间段的重要信息

这类问题的特征是：

- 可重复
- 有明显模式
- 即使降低随机性，也仍然会出现

## 为什么这两个要分开

因为治理方式不同。

### 如果更像随机性

优先考虑：

- 降低 temperature
- 固定 prompt 模板
- 增加 few-shot 示例
- 约束输出格式
- 在评测中使用多次采样统计

### 如果更像系统性错误

优先考虑：

- 改任务定义
- 改示例覆盖
- 改检索和上下文构造
- 增加规则校验
- 更换模型或拆任务链路

## 一个实用区分法

你可以做一个最小实验：

1. 固定完整输入和参数
2. 连续跑 10 次
3. 记录错在哪、怎么错

如果每次错误位置都不太一样，更像随机性。  
如果反复在同一类点上错，更像系统性错误。

## 混合情况最常见

现实里很多问题不是二选一，而是：

- 基础上存在系统性短板
- 外面再叠加随机波动

比如模型本来就不擅长某类抽取任务，而采样又让错误表现得忽好忽坏。  
这时候你如果只压 temperature，往往只能把“抖动”压小，压不掉“总会错”的部分。

## 你可以先这样记住

- 随机性是抖动
- 系统性错误是稳定缺陷
- 先判断是哪一类，再选修法，否则很容易白调参数
