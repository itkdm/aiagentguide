# 11.2 效果、延迟与成本的平衡

先给结论：**LLM 落地里最常见的失败，不是效果做不上去，而是团队只追效果最好，结果延迟、成本或稳定性先把系统拖垮。**

这部分会帮助你把三个指标一起看。重点不是追“最强”，而是找到“业务能接受、团队养得起、用户也愿意用”的方案。

## 读完后你应该能判断

- 一个场景该优先保效果、保延迟还是保成本
- 为什么“模型更强”通常不只是单纯多花点钱，而是连系统形态都会变
- 什么叫真正有工程价值的“够用方案”
