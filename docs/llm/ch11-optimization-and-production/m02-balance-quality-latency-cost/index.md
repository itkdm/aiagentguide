# 11.2 效果、延迟与成本的平衡

先给结论：**LLM 落地里最常见的失败，不是效果做不上去，而是团队只追效果最好，结果延迟、成本或稳定性先把系统拖垮。**

这一节的重点，是把效果、延迟和成本从三个孤立指标变成一个共同决策问题。  
很多团队在离线评测时只看效果分数，一到线上才发现：

- 好模型太慢
- 快模型不稳
- 成本一上量就失控

这不是因为三者不能兼顾，而是因为系统从一开始就没有把这三项一起设计。

所以这部分不是在讲“怎么选最便宜”或“怎么追最好效果”，而是在建立一种更真实的落地判断：  
**真正可用的方案，不是某一项最优，而是业务能接受、团队养得起、用户也愿意持续使用。**

## 读完后你应该能判断

- 一个场景该优先保效果、保延迟还是保成本
- 为什么“模型更强”通常不只是单纯多花点钱，而是连系统形态都会变
- 什么叫真正有工程价值的“够用方案”
