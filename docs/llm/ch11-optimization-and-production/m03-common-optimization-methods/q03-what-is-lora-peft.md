# 11.3.3 什么是 LoRA / PEFT？

先给结论：

**LoRA / PEFT 的核心思路，是尽量少改大模型原参数，只训练一小部分附加参数或适配模块，从而以更低成本完成任务适配。**

所以它们更像“低成本训练方案”，而不是部署压缩方案。

## PEFT 是什么

PEFT 是 Parameter-Efficient Fine-Tuning 的缩写。  
它代表一类方法：不做全参数微调，而是只训练少量参数，让模型适应特定任务。

它的价值在于：

- 训练成本低
- 显存压力小
- 训练速度更快
- 更容易维护多个任务版本

所以 PEFT 特别适合资源有限、任务很多、需要快速试错的场景。

## LoRA 是什么

LoRA 是 PEFT 里的代表性方法之一。  
它的核心不是把原模型大改，而是在某些线性层旁边加入低秩适配矩阵，只训练这些新增参数。

这带来的直接好处通常是：

- 大幅减少可训练参数量
- 降低微调显存需求
- 更容易为不同任务保存和切换适配器

所以 LoRA 往往是团队从“想微调”走向“先低成本试微调”的第一选择。

## LoRA / PEFT 最适合什么场景

它们通常更适合这些情况：

- 已有基座模型总体不错
- 只需要在某类任务上更贴合
- 想低成本做风格、格式、领域适配
- 希望同时维护多个任务版本

这类问题的共同点是：你要的不是重造一个模型，而是让现有模型更贴近你的特定任务。

## 它们不适合解决什么

LoRA / PEFT 通常不擅长直接解决：

- 实时知识更新
- 外部数据库事实缺失
- 工作流设计不合理
- 工具调用链路没建好

这些问题本质上不是“参数适配”问题。

## 一个更务实的理解

如果蒸馏更像“训练一个更小的新学生”，  
LoRA / PEFT 更像“在原老师身上加一层轻量适配器，让它更适合某门课”。

所以两者虽然都涉及训练，但目标不同：

- 蒸馏偏压缩和迁移
- LoRA / PEFT 偏低成本适配

## 你可以先这样记住

- LoRA / PEFT 是低成本微调路线
- 它们解决的是任务适配问题，不是知识补给问题
- 当你需要的是“让现有模型更贴任务”，它们通常比全参微调更现实
