# 11.3 常见优化手段

先给结论：**量化、蒸馏、LoRA / PEFT 虽然都常出现在“优化”这个话题里，但它们解决的不是同一类问题。量化更偏部署与资源效率，蒸馏更偏模型压缩与能力迁移，LoRA / PEFT 更偏低成本适配和训练。**

这一节要做的，是把“优化手段”从术语表重新变回工程工具箱。  
很多人看到量化、蒸馏、LoRA、PEFT 这些词，会有一种错觉：它们都属于优化，所以大概是可替换的。

但真实情况正相反。  
这些方法虽然都在“优化”这个大框里，却作用在完全不同的层面：

- 有些是在解决推理部署资源问题
- 有些是在解决模型压缩和能力迁移
- 有些是在解决低成本训练适配

如果不先把层次分清，很容易出现“拿训练方法解决部署问题”或者“拿部署方法期待能力提升”的错误预期。

所以这一节的重点不是背名词，而是先搞清每种方法到底在优化链路的哪一层。

## 读完后你应该能判断

- 一次优化更像部署压缩问题，还是训练适配问题
- 什么时候量化能直接带来收益，什么时候收益有限
- 什么时候该考虑蒸馏或 LoRA，而不是直接全参微调

## 这一节适合怎么读

- 如果你经常把量化、蒸馏、LoRA 混成一类方案，这一节要重点看
- 如果你后面会接触模型部署、训练适配或资源压缩，这一节是分类基础
- 如果你关心的是“当前瓶颈该用哪一类优化方法”，这一节会给出入口
