# 11.3.2 什么是蒸馏（Distillation）？

先给结论：

**蒸馏可以理解成：让一个更小的学生模型去学习大模型的输出行为、分布或中间知识，从而在更低成本下尽量逼近大模型的效果。**

它本质上是一种能力迁移和模型压缩方法。

## 为什么要做蒸馏

因为很多时候，大模型效果很好，但部署太贵、太慢、太重。  
如果能把它的部分能力迁移到更小模型，就可能同时得到：

- 更低推理成本
- 更低延迟
- 更容易部署
- 在特定任务上接近老师模型的表现

所以蒸馏常常不是追“最强”，而是追“更划算的近似最优”。

## 蒸馏通常在学什么

不同蒸馏方法会学不同层面的东西，例如：

- 最终输出答案
- token 概率分布
- 中间层表示
- 某种行为策略或风格

简单理解就是：学生模型不只是学“最终标签”，而是尽量模仿老师模型怎么看问题、怎么给答案。

## 蒸馏最适合什么场景

蒸馏通常更适合这些情况：

- 你已经有一个效果不错但太贵的大模型
- 任务范围相对稳定
- 希望把高成本能力迁到更轻量方案
- 愿意投入训练成本换长期推理收益

这类场景的关键是：你已经知道“好答案长什么样”，现在想把它压缩成更便宜的系统。

## 蒸馏的边界是什么

蒸馏不是免费午餐。

常见边界包括：

- 学生模型很难在所有能力面都接近老师模型
- 超出蒸馏数据覆盖范围时，泛化可能明显变差
- 训练和数据准备本身需要成本
- 如果老师模型本身有偏差，学生也会继承

所以蒸馏通常更适合收敛在明确任务上，而不是期待“小模型全方位复制大模型”。

## 一个务实理解

如果量化更像“把原模型压缩得更省”，  
那蒸馏更像“重新训练一个更小的模型去学会类似能力”。

两者都跟成本有关，但路径完全不同。

## 你可以先这样记住

- 蒸馏是把大模型能力迁到更小模型
- 它更适合长期稳定任务和高频场景
- 它省的是未来推理成本，但前提是你愿意先投入训练和数据成本
