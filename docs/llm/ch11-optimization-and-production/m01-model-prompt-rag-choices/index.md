# 11.1 模型、Prompt 与 RAG 的选择

先给结论：**模型、Prompt、RAG 不是同一层手段。模型主要解决能力上限，Prompt 主要解决任务表达与行为约束，RAG 主要解决外部知识供给。选错层，优化通常既贵又慢。**

这一节要先解决一个特别常见的优化误区：  
很多团队一看到效果不好，就会立刻讨论“要不要换更强模型”“要不要微调”“要不要再改 prompt”，但真正关键的问题其实是，**当前问题到底落在哪一层。**

因为这三类手段解决的问题并不一样：

- 模型升级更偏能力上限和泛化能力
- Prompt 调整更偏任务表达、约束和输出口径
- RAG 更偏知识供给、时效性和来源支撑

如果根因判断错了，后面的动作很容易既费钱又费时间。  
这一节会把这三层重新放回问题归因里看，帮助你先学会判断“问题像什么”，再决定“该改哪一层”。
