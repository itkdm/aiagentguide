# 11.4 微调与系统增强

先给结论：**真实企业落地里，微调只是可选手段之一，真正决定系统能不能长期上线的是：能力方案、数据边界、权限设计、审计可追溯性和运行时观测有没有一起建好。**

这一节的重点，是把“能力提升”和“系统治理”放到一张图里看。  
很多团队做企业落地时，最容易把注意力都放在模型能力本身，比如：

- 要不要微调
- 要不要换更大模型
- 要不要再补一点领域数据

这些问题当然重要，但如果只看能力层，企业系统往往还是落不了地。  
因为真正决定它能不能长期运行的，还包括：

- 知识和数据边界是否清楚
- 权限和调用范围是否受控
- 输出是否可审计、可追溯
- 线上运行是否可观测、可回滚

所以这一节不是单讲“微调 vs RAG”，而是把它们放回企业治理和系统增强的完整背景里看。
