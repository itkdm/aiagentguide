# 11.4 微调与系统增强

先给结论：**真实企业落地里，微调只是可选手段之一，真正决定系统能不能长期上线的是：能力方案、数据边界、权限设计、审计可追溯性和运行时观测有没有一起建好。**

这一部分会把“怎么做能力提升”和“怎么把系统做稳”放到一张图里看。因为很多团队的问题并不是不会做模型，而是没有把模型放进一个可治理的企业系统里。

## 这一节会回答什么问题

- [微调和 RAG 应该怎么选？](./q01-finetuning-vs-rag)
- [小模型、大模型、开源模型、闭源模型分别适合什么场景？](./q02-small-big-open-closed-models)
- [企业落地 LLM 时最常见的风险是什么？](./q03-common-enterprise-risks)
- [如何做安全、权限、审计和观测？](./q04-security-permission-audit-observability)

## 读完后你应该能判断

- 一个问题该先用 RAG、微调，还是先改治理和系统链路
- 不同模型类型为什么适合不同业务约束
- 企业级 LLM 系统为什么必须从第一天就设计权限、审计和观测
