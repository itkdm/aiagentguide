# 6.3.3 如何根据任务类型调整生成策略？

先给结论：**先按“任务目标”分层，再用采样参数逐步调节稳定性与多样性，最后用评测回收。** citeturn0search2turn1search0turn3search4

## 第一步：把任务按目标分层

你可以先把任务分成三类：

1. **确定性任务**：分类、抽取、结构化输出  
2. **可控生成**：摘要、报告、改写  
3. **发散生成**：创意写作、头脑风暴、方案探索

这是选择采样策略的基准层。citeturn0search2turn1search0

## 第二步：给每类任务设定默认策略

一个可用的初始配置是：

- 确定性任务：低温度 + 低 top-p 或 greedy  
- 可控生成：中等温度 + 中等 top-p  
- 发散生成：较高温度 + 较高 top-p

这些只是起点，最终参数要靠评测收敛。citeturn0search2turn3search4

## 第三步：用评测回收参数

建议至少做两类评测：

- **一致性评测**：同样问题重复生成，观察稳定性  
- **覆盖性评测**：看候选是否足够多样且可用  

只有评测能验证“参数是否符合任务目标”。citeturn1search0

## 常见误区

- 只调温度不调 top-p，导致噪声控制不足
- 只靠主观感觉，不做回归评测
- 一个参数配置覆盖所有任务，导致系统体验不稳定

## 结论收束

**生成策略不是“拍脑袋”，而是“任务分层 + 参数回收”。**
