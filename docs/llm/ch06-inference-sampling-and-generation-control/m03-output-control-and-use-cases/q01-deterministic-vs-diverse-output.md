# 6.3.1 什么场景适合确定性输出，什么场景适合发散性输出？

先给结论：**确定性输出适合“可验证、可执行”的任务，发散性输出适合“需要探索与创意”的任务。** citeturn2view0turn0search2

## 确定性输出适合的场景

确定性输出强调“可复现与可控”，常见场景包括：

- 结构化抽取与分类
- 规则性强的报告生成
- 需要稳定格式的接口输出

这些场景更关注一致性而不是多样性。citeturn0search2turn3search4

## 发散性输出适合的场景

发散性输出强调“多样性与探索”，常见场景包括：

- 创意写作、标题生成
- 方案发散与头脑风暴
- 多方案对比与选项输出

这类任务更重视覆盖面和新颖性。citeturn2view0turn1search0

## 关键判断标准

你可以用三条标准快速判断：

1. **可验证性**  
   输出是否有明确的对错标准？有的话偏确定性。

2. **容错性**  
   输出“差一点”是否仍可接受？容错高时可适当发散。

3. **用户目标**  
   用户是想“得到唯一答案”，还是“看到多个候选”？

## 常见误区

- 把所有任务都用低温度，导致输出过度保守
- 把所有任务都调高温度，导致输出不可控

你可以先记住一句话：**确定性追求稳定，发散性追求探索。**
