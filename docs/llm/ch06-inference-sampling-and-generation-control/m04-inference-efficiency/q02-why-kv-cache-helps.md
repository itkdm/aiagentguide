# 6.4.2 KV Cache 为什么能提升推理效率？

先给结论：**KV Cache 让模型在解码时复用历史 token 的注意力表示，从而把“重复计算”变成“直接读取”。** 

## 推理里最大的“浪费”来自哪里

自回归生成每一步都需要使用全部历史 token。  
如果不缓存，每一步都要重新计算过去的 key/value。

## KV Cache 如何节省成本

启用 KV Cache 后：

- 历史 token 的 key/value 被缓存  
- 新 token 只需计算新增部分  

这样推理时间会随生成长度线性增长，而不是反复做全量计算。

## 为什么有时还会慢

KV Cache 不是万能：

- **缓存占用显存**：序列越长，缓存越大
- **带宽限制**：读写缓存会占用显存带宽

如果显存紧张或带宽瓶颈突出，速度提升会变小。

## 工程上的落地建议

如果你要充分利用 KV Cache：

1. 控制上下文长度，避免缓存膨胀  
2. 控制输出长度，减少解码步数  
3. 做批处理，提升吞吐率

你可以先记住一句话：**KV Cache 提升速度，但前提是你能承担显存成本。**
