# 6.4 推理效率优化

先给结论：**推理成本主要被“上下文长度 + 解码步数”决定，KV Cache 能减少重复计算但会增加显存占用。** 

这一节会把“推理为什么贵”进一步落到优化手段上。  
在真实系统里，很多性能问题并不是模型本身“太慢”，而是上下文太长、输出太多、缓存管理做得差，或者服务策略没有围绕推理特点设计。

所以这一节的重点不是只记住 KV Cache 这个词，而是理解几条最核心的工程关系：

- 为什么缓存可以减少重复计算
- 为什么缓存会换来更高显存压力
- 为什么长上下文和长输出会把成本一起拉高
- 为什么很多推理优化本质上是在“显存、延迟、吞吐”之间做权衡

把这些逻辑看清之后，后面不管是理解长上下文问题、服务系统设计，还是做成本优化，都会更稳。

## 学完后你应该建立的判断

- KV Cache 是“算力换显存”的典型优化
- 上下文越长，注意力计算越贵
- 输出越长，解码循环越慢
