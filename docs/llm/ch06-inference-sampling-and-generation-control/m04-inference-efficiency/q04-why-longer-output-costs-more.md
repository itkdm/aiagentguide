# 6.4.4 为什么输出越长，延迟和成本通常越高？

先给结论：**输出越长，解码循环次数越多；每个 token 都要一次前向计算，因此延迟与成本线性上升。** 

## 自回归生成的特点

大多数 LLM 采用自回归方式生成：  
每次只生成一个 token，再把它拼回上下文继续下一轮。

这意味着：

- 生成 200 token 就是 200 次前向  
- 输出越长，延迟越高  

## 为什么这会影响成本

推理成本通常按 token 计费或按算力资源消耗计费。  
输出长度直接决定你要花多少算力。

## 工程上的控制方式

你可以用三类手段控制输出成本：

1. **限制 max tokens**  
2. **明确输出格式**：避免无意义冗长  
3. **分段输出**：大任务拆成多段生成

## 结论收束

**输出长度是最直接的成本杠杆。**
