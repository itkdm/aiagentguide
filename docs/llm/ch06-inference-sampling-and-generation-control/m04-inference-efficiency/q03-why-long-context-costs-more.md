# 6.4.3 为什么长上下文会导致推理成本变高？

先给结论：**长上下文让注意力计算量快速上升，同时 KV Cache 占用显存更大，因此延迟和成本都会上升。** citeturn1search0turn0search0turn0search2

## 原因一：注意力复杂度与长度平方相关

Transformer 的自注意力在序列长度上具有二次复杂度，长度翻倍时计算量远不止翻倍。citeturn1search0

这就是长上下文推理速度明显下降的根本原因。

## 原因二：KV Cache 随长度增长

长上下文意味着需要缓存更多的 key/value，显存占用随长度线性增长。citeturn0search0turn0search2

当缓存占用过大时：

- 单卡可并发请求数下降  
- 延迟提升  

## 工程上的控制手段

常见做法包括：

- 对输入做裁剪或摘要  
- 分段检索，避免一次塞入太长上下文  
- 为不同任务设置上下文预算

## 结论收束

**长上下文不是“免费能力”，而是“计算与显存的显式成本”。**
