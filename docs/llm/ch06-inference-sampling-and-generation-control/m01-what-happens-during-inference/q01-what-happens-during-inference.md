# 6.1.1 LLM 推理时到底在做什么？

先给结论：**推理就是在当前上下文上前向计算，得到下一 token 的概率分布，然后按某种解码策略选出 token，循环生成完整回答。** 

## 推理和训练的本质区别

训练阶段会更新参数；推理阶段参数固定，只做前向计算。  
但“只做前向”不代表便宜，因为每生成一个 token 都要做一次完整前向计算。

## 你可以用这条链路理解推理

1. **输入被编码为 token**  
2. **模型前向计算，得到 logits**  
3. **通过解码策略选出下一个 token**  
4. **把新 token 拼回上下文，继续下一轮**

这就是为什么推理看起来像“一个循环”，而不是一次性输出。

## 推理结果为什么会不同

同一个提示，推理输出会不同，主要因为：

- 解码策略不同（greedy、top-k、top-p）
- 采样参数不同（temperature、top_p）

这些参数并不会改变模型能力，只会改变“从分布里挑出哪个答案”。
