# 6.1 模型推理时到底在做什么

先给结论：**推理阶段就是“用当前上下文计算下一 token 分布，并一步步解码成文本”。成本主要来自注意力计算与多次解码循环。** 

这一节的目标，是把“模型在推理时到底在干什么”这件事讲得足够具体。  
很多人知道训练很贵，却不理解为什么模型上线后、明明已经不再更新参数，推理依然会消耗大量算力和延迟预算。

原因在于，推理并不是“把答案一次性取出来”，而是在当前上下文上反复做矩阵计算、构造注意力、生成概率分布，再一 token 一 token 地继续向后解码。

只要这层机制没有看清，后面很多问题都会容易混：

- 为什么长上下文会更贵
- 为什么输出越长延迟越高
- 为什么 KV Cache 这种优化会有效
- 为什么同样模型在不同服务配置下速度差很多

这一节会把这些问题的底层逻辑先立住。

## 学完后你应该建立的判断

- 推理的核心是“逐 token 预测 + 解码”
- 注意力计算与上下文长度决定了推理成本
- 采样策略决定“回答风格”，而非“模型能力”

## 这一节适合怎么读

- 如果你以前把推理简单理解成“模型跑一遍前向计算”，这一节要重点看
- 如果你后面会关注延迟、吞吐、KV Cache 和服务成本，这一节是必要前提
- 如果你总觉得“推理很贵”只是经验结论，这一节会把成本来源拆清楚
