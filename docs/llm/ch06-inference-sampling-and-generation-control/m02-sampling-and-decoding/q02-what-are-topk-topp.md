# 6.2.2 什么是 top-k / top-p？

先给结论：**top-k 和 top-p 都是在做候选裁剪。top-k 保留概率最高的前 K 个 token，top-p 保留累计概率达到阈值 p 的那部分 token。它们都不是直接提高模型能力，而是在限制“哪些候选还有资格被抽到”。**

很多人第一次看到这两个参数时，会把它们理解成“多样性控制器”。这个方向不算错，但更准确的说法应该是：**它们首先是在控风险边界，只有在边界控住之后，多样性调节才有意义。**

## 为什么采样前要先裁剪候选

模型在每一步都会给出整个词表上的概率分布，但真正有意义的高概率候选通常只集中在很小一部分 token 上。

如果完全不裁剪，直接在整个分布里采样，就会带来两个问题：

- 低概率噪声 token 也有机会混进来
- 输出更容易出现跑偏、异常表达或逻辑跳跃

所以 top-k / top-p 的作用，首先不是“让模型更会发散”，而是先把明显不靠谱的长尾候选切掉。

## top-k 是什么

top-k 的规则最直观：

1. 按概率从高到低排序。
2. 只保留前 K 个 token。
3. 在这 K 个 token 里继续做采样。

例如 `top_k=10`，就意味着只有当前最可能的 10 个 token 还能参与后续抽样，其余全部直接丢弃。

它的特点是很直接，也很好理解：

- `K` 小：候选范围窄，输出更稳
- `K` 大：候选范围宽，输出更发散

所以 top-k 的本质是用“固定数量”来限制候选空间。

## top-p 是什么

top-p 又叫 nucleus sampling。它不是固定保留多少个 token，而是：

1. 先按概率从高到低排序。
2. 从前往后累加概率。
3. 当累计概率达到阈值 `p` 时停止。
4. 只保留这部分 token。

它的特点是候选集合大小会动态变化。

例如：

- 如果当前分布特别尖，高概率 token 很集中，那么达到 `p=0.9` 可能只需要很少几个 token。
- 如果当前分布比较平，达到同样的 `p=0.9` 可能要保留更多 token。

所以 top-p 的本质是用“概率质量覆盖率”来限制候选空间。

## 两者最核心的差别

如果要用一句话区分：

- `top-k` 控的是候选数量
- `top-p` 控的是候选概率覆盖范围

这导致它们在面对不同分布时，行为并不一样。

### top-k 的特点

优点是简单、直观、容易预期。  
缺点是它不关心分布形状，只关心“前 K 个”。

如果某一步分布非常尖，只保留固定 K 个候选，可能已经很多了；  
如果某一步分布很平，同样的 K 又可能太少，裁得过死。

### top-p 的特点

优点是更自适应，会随着分布形状变化自动调整候选大小。  
缺点是直觉上比 top-k 稍微难一点，也更容易让人误把 `p` 当成“随机性参数”，而忽略它本质是在裁剪候选边界。

## 为什么它们不是“能力增强器”

这是最常见的误区之一。

top-k 和 top-p 能做的是：

- 去掉明显低概率噪声
- 控制候选空间大小
- 在稳定性和发散性之间做边界控制

它们做不了的是：

- 让模型凭空知道更多知识
- 让错误推理自动变正确
- 让差的 Prompt 自动变好

如果模型本来就在胡猜，那么你只是决定“在多大范围里胡猜”。  
如果模型本来就不懂，那么你只是改变“不懂时会怎么抽样”。

## 真实系统里通常怎么选

更稳的做法不是死记一个默认值，而是按任务来想。

### 适合更保守候选空间的任务

- 事实问答
- 信息抽取
- 分类打标
- 结构化输出
- 高约束代码或配置生成

这些任务更怕：

- 噪声词混入
- 输出不稳定
- 格式跑偏

因此候选空间通常要收得更紧。

### 适合更宽候选空间的任务

- 创意写作
- 文案变体
- 多版本 brainstorming
- 风格化表达

这些任务接受一定探索空间，所以候选范围可以放宽一些，但仍然不代表要把长尾噪声完全放开。

## top-k 和 top-p 要不要一起用

可以一起用，但要知道自己在干什么。

一起用时，本质上是叠加两层限制：

- 一层限制最大候选数量
- 一层限制累计概率范围

这在某些系统里有价值，但如果你同时把它们调得很激进，也可能互相打架，让候选空间既不自然也不稳定。

工程上更常见、更容易理解的做法是：

- 先明确你更想控制的是“数量”还是“概率覆盖”
- 选其中一个作为主参数
- 再配合 temperature 微调内部发散度

比起同时乱调三个参数，这样更容易定位问题。

## 一个更直观的理解

可以把采样想成“在一间候选会议室里投票”。

- `top-k` 是规定“只允许前 K 名候选进会议室”
- `top-p` 是规定“只允许累计代表 90% 票数的候选进会议室”

会议室外的人，后续再优秀也没有发言权。  
这就是为什么它们对边界控制很重要。

## 常见坑

### 把 top-k / top-p 当成越大越好

候选空间太大，经常只是把低概率噪声放进来，不代表更聪明。

### 同时把 top-k、top-p、temperature 都开得很松

这通常不是“更有创意”，而是更难控。

### 只记参数名，不看任务目标

参数本身没有意义，只有放回“我要稳定输出还是要探索输出”这个任务判断里才有意义。

## 一个简单的排查方法

如果你觉得模型输出太乱，可以先这样排查：

1. 固定 Prompt 和 temperature。
2. 逐步收紧 top-k 或 top-p。
3. 观察三个变化：
   - 异常词是否减少
   - 格式是否更稳
   - 输出是否开始过度模板化

这样比上来就同时改一堆参数更容易看出问题到底出在哪。

## 收束一句话

**top-k 和 top-p 的本质，都是在采样前先裁掉一部分候选。top-k 按数量裁，top-p 按累计概率裁。它们控制的是候选边界，不是模型能力上限。**
