# 6.2.3 temperature 和 top-p 有什么区别？

先给结论：**temperature 是在调整概率分布的形状，top-p 是在裁剪候选空间。前者决定候选之间的相对随机程度，后者决定哪些候选还能参与抽样。**

很多人会把这两个参数混着用，觉得它们都在“让输出更随机”或“让输出更稳定”。这个方向没错，但如果不把两者拆开，很容易调出一组看似合理、实际互相打架的配置。

## temperature 在做什么

temperature 的核心作用，是把原始分布变尖或变平。

它不会直接删掉任何 token，而是调整：

- 高概率 token 的优势有多大
- 中低概率 token 还有没有机会被选到

所以它影响的是“在保留的候选里，模型有多敢偏离最优项”。

## top-p 在做什么

top-p 又叫 nucleus sampling。它不是去改概率形状，而是按累计概率阈值，把候选集截断到一段“概率质量足够高”的核心区域。

也就是说：

- 累计概率进入阈值范围内的 token 保留
- 阈值之外的低概率尾部直接被丢掉

所以 top-p 影响的是“哪些 token 连参与抽样的资格都没有”。

## 一句话区分

如果只记一句，可以这样记：

- `temperature` 决定“怎么抽”
- `top-p` 决定“从哪一堆里抽”

这两者看起来都在影响随机性，但一个是调内部权重，一个是裁候选边界。

## 为什么它们经常一起出现

因为真实生成里，光做其中一个往往不够。

### 只调 temperature 的问题

如果你把 temperature 调高，但完全不限制候选空间，模型可能把低概率尾部噪声也一起放进来。结果不一定是“更有创意”，更常见的是：

- 跑题
- 词语异常
- 逻辑跳跃
- 事实漂移

### 只调 top-p 的问题

如果你只裁剪候选集，但在保留集合内部仍然使用很保守的采样方式，输出虽然不会太离谱，但也可能非常僵硬，变化不够。

所以常见工程思路是：

1. 先用 top-p 去掉明显低质量的尾部噪声。
2. 再用 temperature 控制保留候选中的发散程度。

这比单独粗暴地拉高某一个参数，通常更稳。

## 两者分别更适合解决什么问题

### temperature 更适合回答的问题

- 为什么同一组候选里有时总选第一名，有时愿意尝试第二第三名
- 如何让输出更保守或更发散
- 如何控制风格变化程度

### top-p 更适合回答的问题

- 如何防止极低概率噪声 token 混进来
- 如何把采样空间限制在“相对合理”的范围内
- 如何在发散和失控之间加一道边界

换句话说，temperature 更像是油门，top-p 更像是道路护栏。

## 一个更直观的例子

假设当前有一组 token 概率大致如下：

- A: 0.40
- B: 0.25
- C: 0.15
- D: 0.08
- E: 0.05
- 其余更低

如果你只调 temperature：

- 所有 token 还都在
- 只是 A 的领先优势会被放大或压缩

如果你调 top-p：

- 例如只保留累计概率前 0.9 的 token
- 那么某些尾部 token 会直接消失，后续不再参与采样

这就是“调分布”和“裁分布”的差别。

## 为什么很多配置会互相打架

最常见的问题是：

- 高 temperature
- 再配很高的 top-p

这等于同时放大随机性，又把候选空间放得很大。结果往往不是“更聪明”，而是“更难控”。

另一种常见问题是：

- 很低 temperature
- 再配很低的 top-p

这会把输出收得过死，容易出现：

- 语言过度模板化
- 多轮回复重复
- 创意任务缺乏变化

所以参数不是越激进越好，也不是越保守越好，而是要匹配任务目标。

## 任务里该怎么判断

### 事实型、规则型任务

通常更看重：

- 正确率
- 一致性
- 低风险

这时更常见的思路是：

- 候选空间不要太宽
- 发散程度不要太高

也就是让 top-p 和 temperature 都偏保守。

### 创意型、探索型任务

通常更看重：

- 多样性
- 风格变化
- 候选探索空间

这时可以适当放宽 top-p，并提高 temperature，但仍要避免把尾部噪声完全放开。

## 一个更实用的组合思路

如果你不想死记参数值，可以按这个顺序调：

1. 先确定这个任务能接受多大风险。
2. 用 top-p 先收住候选边界。
3. 再用 temperature 去调风格和发散度。
4. 最后用多轮测试看稳定性，而不是只看单次结果。

这个顺序比一上来就同时乱调两个参数更稳。

## 常见误区

### 误区一：temperature 和 top-p 是同一类参数

不是。它们都影响随机性，但工作机制不同。一个调形状，一个裁范围。

### 误区二：两个都开大，创造力就一定更强

很多时候只是噪声更大、跑偏更多。

### 误区三：只要调参数，就能解决输出质量问题

参数只能改采样行为，不能补底层知识、推理或检索缺陷。

## 收束一句话

**top-p 负责决定“哪些 token 还在候选池里”，temperature 负责决定“候选池里抽样有多激进”。前者控边界，后者控发散。**
