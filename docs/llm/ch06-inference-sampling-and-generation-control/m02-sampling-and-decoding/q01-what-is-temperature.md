# 6.2.1 什么是 temperature？

先给结论：**temperature 用来调节采样分布的“尖锐程度”。它不会改变模型知道什么，只会改变模型在多个候选 token 之间有多激进地偏向高概率项。temperature 越低，输出越接近确定性；temperature 越高，输出越发散。**

很多人第一次调这个参数时，会把它理解成“创造力开关”。这个说法方便记忆，但不够准确。更工程化的理解应该是：**temperature 不是给模型加创造力，而是在改“模型如何从已有概率分布里做选择”。**

## 它到底改了什么

模型在生成每一个 token 时，都会先算出一组 logits，再把它们转成概率分布。temperature 的作用，就是在转成概率前，重新拉开或压平这些分数之间的差距。

直觉上可以这样理解：

- `temperature` 低：高概率 token 的优势会被放大，分布更尖，模型更倾向选最稳的词。
- `temperature` 高：高概率 token 的优势会被削弱，分布更平，模型更容易在多个候选之间做更发散的选择。

所以它控制的不是“模型会不会”，而是“模型在已经会的情况下，愿意多大程度偏离最可能答案”。

## 为什么低温更稳定

当 temperature 下降时，概率最高的几个 token 会进一步占优。这会带来两个直接结果：

- 同样输入更容易得到相似输出
- 输出更少冒险，更少跳到低概率词

这就是为什么事实问答、结构化生成、字段抽取这类任务，通常会更偏好低 temperature。因为这些任务首先追求的是：

- 结果可复现
- 格式可控
- 错误风险低

在这些场景里，系统不需要模型“有灵感”，而是需要它稳定地走在高概率、低风险轨道上。

## 为什么高温会更发散

当 temperature 升高时，原本差距没那么大的候选 token 会被拉近，模型更愿意尝试次优甚至更边缘的选项。这会让输出：

- 更有变化
- 更不容易固定成一种模板
- 更适合发散式探索

所以创意文案、头脑风暴、故事延展、风格变体这类任务，往往会适当提高 temperature，给模型留出更多组合空间。

但要注意，高 temperature 带来的不是“更高级”，而是“更不保守”。这既可能产生新意，也可能引入噪声、跑偏和幻觉。

## 它为什么不等于“质量调节器”

这是最常见的误解。

temperature 改的是采样行为，不是底座能力。它不能：

- 让模型凭空知道更多事实
- 让弱推理突然变强
- 让错误知识自动变正确

如果底座模型本来就不懂某件事，调 temperature 只是在“错误答案集合”里换一种采样方式。低温可能让它更稳定地错，高温可能让它更花样地错。

所以遇到质量问题时，先分清这是：

- 能力问题
- 知识问题
- 检索问题
- 采样问题

不要把所有输出不理想都归因于 temperature。

## 真实系统里通常怎么用

更稳的做法不是背一个“万能温度值”，而是按任务目标来定。

### 适合低 temperature 的任务

- 事实问答
- 信息抽取
- 分类和打标
- SQL / JSON / 代码片段生成
- 高约束格式输出

这些任务更关心正确性、可复现性和格式稳定，所以采样空间通常要收紧。

### 适合中高 temperature 的任务

- 广告文案
- 创意改写
- 头脑风暴
- 故事延展
- 多版本候选生成

这些任务接受一定发散，甚至需要发散，否则输出很容易模板化。

### 一个更实用的判断方式

不要问“temperature 设多少最好”，而要问：

- 这个任务是更怕重复，还是更怕跑偏
- 是更怕保守，还是更怕出错
- 是要单次稳定产出，还是要多候选探索

这些问题想清楚，temperature 才有意义。

## 为什么它经常和 top-p 一起调

因为 temperature 和 top-p 管的不是同一件事。

- `temperature` 调的是分布形状
- `top-p` 控的是候选范围

实际系统里常见做法是：

1. 先用 `top-p` 或其他裁剪手段去掉过低概率噪声。
2. 再用 `temperature` 在保留下来的候选空间里控制发散程度。

如果只把 temperature 调高，但完全不限制候选空间，模型更容易采到低概率噪声 token。这样看起来“更自由”，实际常常只是更乱。

## 常见坑

### 把 temperature 当成能力增强器

它只能改采样行为，不能补能力短板。

### 在高风险任务里把 temperature 调太高

例如知识问答、规则执行、结构化输出。如果此时还追求高发散，通常只会把错误率和不稳定性一起拉高。

### 只看单次输出，不看分布表现

temperature 是分布级参数。一次回答看起来不错，不代表这个配置长期稳定。真正该看的是多轮、多样输入下的整体表现。

## 一个简单的自查方法

如果你怀疑问题出在 temperature，可以做一个最小排查：

1. 固定 Prompt 和上下文。
2. 用低、中、高三组 temperature 各跑多次。
3. 观察三个维度：
   - 事实是否更稳定
   - 格式是否更稳定
   - 内容是否更容易跑偏或重复

如果降温后只是“更稳定地错”，说明核心问题大概率不是采样，而是知识、推理或检索。

## 收束一句话

**temperature 控制的是“模型在已有概率分布里有多敢偏离最稳答案”。它决定输出的发散程度，不决定模型能力上限。**
