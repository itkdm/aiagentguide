# 6.2.4 什么是 greedy decoding？

先给结论：**greedy decoding 就是在每一步都直接选当前概率最高的 token。它的优点是稳定、简单、可复现，缺点是容易过早锁死路径，缺乏多样性，也不保证最终序列全局最优。**

很多人会把 greedy decoding 理解成“最理性”或“最高质量”的解码方式，因为它每一步都在选概率最大项。但这里有一个很容易忽略的点：**每一步局部最优，不等于整段输出全局最优。**

## 它到底怎么工作

模型在生成每个 token 时，都会给出一个概率分布。greedy decoding 的规则非常直接：

1. 看当前所有候选 token 的概率。
2. 选概率最大的那个。
3. 把它接到上下文后，继续下一步。

整个过程中没有随机采样，也没有为低概率候选保留探索空间。

这意味着同样输入、同样模型、同样上下文下，输出通常高度一致。这就是它最大的特点。

## 为什么它看起来很稳

因为它把“选择自由度”几乎压到了最小。

这会带来几个明显好处：

- 同输入下结果更可复现
- 输出更容易稳定遵守格式
- 调试更方便，因为采样噪声更少

所以在很多强约束任务里，greedy decoding 很有吸引力。例如：

- 分类
- 抽取
- 结构化字段生成
- 高格式约束输出

这些任务通常不需要探索多个表达版本，反而更关心一次就按格式给对。

## 为什么它不一定是“最好”

问题出在它太早做决定。

语言生成不是独立选词，而是一个连续决策过程。某一步虽然最优的 token 看起来概率最高，但选了它之后，后续路径可能会被锁进一个更平庸、更重复、甚至更差的序列里。

也就是说，greedy decoding 的局限是：

- 它只看眼前一步
- 不为后续更优路径预留探索空间

这也是为什么它经常会出现：

- 用词保守
- 句式重复
- 风格发死
- 创意不足

在创意生成、开放式对话、文案写作这类任务里，这种问题尤其明显。

## greedy 和采样的根本区别

可以把两者简单对比成：

- `greedy`：每一步都拿第一名
- `sampling`：在合理候选里做带随机性的选择

前者追求的是确定性，后者追求的是在质量和多样性之间找到平衡。

因此，greedy decoding 更像是在说：“我只要最稳的那条路径。”  
而采样更像是在说：“我接受合理范围内的多个版本。”

## 它适合什么任务

更适合 greedy 的，通常是这些任务：

- 结构化抽取
- 固定格式生成
- 分类和标签预测
- 对一致性要求很高的系统输出

这些任务的共同点是：

- 用户不需要多个候选版本
- 输出容错空间小
- 格式稳定比表达变化更重要

比如生成 JSON 字段、SQL 片段、规则映射结果时，greedy 往往比高随机采样更好控。

## 它不适合什么任务

不太适合 greedy 的任务通常包括：

- 创意文案
- 多版本 brainstorming
- 故事延展
- 风格改写
- 需要自然变化的对话生成

这些任务如果使用 greedy，常见问题是：

- 语言太模板化
- 版本之间几乎没有差异
- 模型总是走“最常见套路”

在这些场景里，稳定反而会变成缺点，因为系统需要的是探索空间，而不是单一路径锁死。

## 为什么它常被拿来做基线

尽管它有局限，greedy decoding 仍然很重要，因为它是非常好的基线。

原因很简单：

- 行为稳定
- 结果容易复现
- 便于排除采样噪声

如果你在调 Prompt、调模型、调上下文时想先看“最保守条件下模型到底会怎么走”，greedy 往往是最干净的起点。

换句话说，它不一定是最终配置，但常常是最适合拿来做问题定位的配置。

## 一个常见误区：稳定不等于质量更高

很多人看到 greedy 输出更稳定，就会误以为它质量也更高。其实不是。

更准确的说法是：

- greedy 的方差更低
- 但它不代表平均质量一定最高

在很多开放式任务里，采样策略虽然不够稳定，却可能更容易产出更自然、更灵活的回答。只是你需要接受输出波动，并用任务、系统和评测手段去约束它。

## 一个实用判断

如果你面对的任务更怕这些问题：

- 跑偏
- 格式不稳
- 多次结果不一致

那么 greedy 通常值得优先试。

如果你更怕这些问题：

- 回答死板
- 创意不足
- 候选版本过少

那么 greedy 往往就不是合适选择。

## 常见坑

### 把 greedy 当成“最聪明的解码”

它只是最保守，不代表最聪明。

### 在创意任务里强行用 greedy

结果通常不是更优，而是更像模板填空。

### 把 greedy 的稳定误判成模型能力强

有时它只是更稳定地暴露模型默认高概率套路，不代表模型真的理解得更深。

## 收束一句话

**greedy decoding 的本质是“每一步都选当前最可能的 token”。它追求稳定和可复现，但代价是探索空间小、输出容易保守，也不保证整段结果全局最优。**
