# 8.1.1 什么是 Prompt？

先给结论：**Prompt 就是你在当前请求里提供给模型的任务上下文，包括指令、背景、约束、示例和期望输出形式。** citeturn0search0turn0search4

## 不要把 Prompt 理解成一句“提问”

在真实系统里，Prompt 往往不是一句自然语言问题，而是一整段结构化上下文，例如：

- 系统角色说明
- 用户任务描述
- 约束条件
- 示例输入输出
- 输出格式要求

这些内容共同决定模型如何理解当前任务。citeturn0search0turn0search4

## Prompt 在做什么

Prompt 的作用可以概括成三件事：

1. **告诉模型当前任务是什么**
2. **告诉模型输出应该长什么样**
3. **把当前最相关的信息放进可见上下文**

所以 Prompt 的本质不是“施法”，而是“组织上下文”。citeturn0search0turn0search1

## 一个最小示意

```text
请把下面这段用户反馈总结成三条要点，并按“问题 / 影响 / 建议”格式输出。
```

这条 Prompt 至少同时提供了：

- 任务：总结
- 数量约束：三条
- 格式约束：问题 / 影响 / 建议

## 结论收束

**Prompt 不是单纯发问，而是给模型搭建当前任务的工作环境。**
