# 8.1 Prompt 的基本作用

先给结论：**Prompt 的本质是“把当前任务、约束和期望输出组织进上下文”，它会显著影响模型行为，但不会凭空给模型增加不存在的知识或工具能力。** 

这一节的目标，是先把 Prompt 从两个极端里拉回来：

- 一个极端是把 Prompt 看成只是“随便输入一句话”
- 另一个极端是把 Prompt 神化成“几乎能解决所有问题的万能技巧”

这两个理解都不准确。  
Prompt 的真实位置更接近“任务接口层”: 你怎么描述任务、给什么上下文、加哪些约束，都会直接影响模型当前这一次推理的行为表现。

但它能做的事情也有明确边界。它可以重组任务、压实输出要求、减少歧义，却不能替代知识更新、工具接入、评测和系统设计。

所以这一节不是在教你背几个 Prompt 技巧，而是在建立一个更稳的基线判断：  
**Prompt 很重要，但它解决的是任务表达问题，不是所有问题。**

## 学完后你应该建立的判断

- Prompt 是任务组织手段，不是知识注入魔法
- Prompt 好坏主要影响稳定性、可控性和任务对齐
- 真实系统里，Prompt 必须和检索、工具、评测一起看

## 这一节适合怎么读

- 如果你以前把 Prompt 当成“模型输入的自然语言描述”，这一节会帮你把它提升到系统接口层理解
- 如果你容易在“Prompt 很重要”和“Prompt 没那么神”之间摇摆，这一节要重点看
- 如果你后面会做提示结构设计、工具调用或 Agent 编排，这一节是基础入口
