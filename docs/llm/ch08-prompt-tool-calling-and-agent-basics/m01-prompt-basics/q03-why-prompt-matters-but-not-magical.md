# 8.1.3 为什么 Prompt 很重要，但又不能神化 Prompt？

先给结论：**Prompt 很重要，因为它直接影响任务对齐和输出稳定性；但不能神化，因为它不能突破模型的知识边界、工具边界和推理边界。** 

## 为什么它重要

同一个模型，在 Prompt 组织不同的情况下，输出质量可能差很多。  
原因不是模型“突然变聪明了”，而是任务描述更清楚、约束更明确、示例更有效。

## 为什么不能神化

Prompt 无法解决这些问题：

- 模型不知道的知识
- 模型拿不到的实时数据
- 模型做不到的外部动作

这也是为什么真实系统里，Prompt 通常要和 RAG、工具调用、评测体系一起工作。

## 一个工程判断

当你发现任务效果不好时，先区分问题类型：

1. **任务没说清楚**：优先改 Prompt  
2. **知识不够或过时**：优先加检索  
3. **需要外部动作**：优先加工具调用  
4. **模型底座不够**：优先换模型或微调  

## 结论收束

**Prompt 是高杠杆手段，但不是万能补丁。**
