# 8.3.4 为什么工具调用能显著降低模型“硬猜”的概率？

先给结论：**因为工具调用把“模型自己编答案”的空间，替换成了“先拿真实结果，再组织输出”的流程。** 

## 模型为什么会硬猜

当模型缺少实时数据、缺少精确结果、又必须给出回答时，它往往会生成“看起来合理”的内容。  
这就是很多幻觉的来源。

## 工具调用如何改变这件事

工具调用会把流程改成：

1. 先判断需不需要外部信息  
2. 真的去取数、计算或执行  
3. 再基于结果回答

这会显著减少“无依据生成”。

## 工程上的额外收益

工具调用除了降低硬猜，还带来：

- 更容易审计
- 更容易追踪错误来源
- 更容易对结果做校验

## 结论收束

**工具调用的核心价值，不只是“更强”，而是“更少瞎猜”。**
