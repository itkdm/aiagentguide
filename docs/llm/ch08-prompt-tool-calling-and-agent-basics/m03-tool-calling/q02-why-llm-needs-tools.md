# 8.3.2 LLM 为什么需要调用工具？

先给结论：**因为 LLM 擅长语言理解与生成，但不擅长保证实时事实、精确计算和外部动作执行。工具正是用来补这三个缺口。** citeturn0search3turn0search4

## 缺口一：实时数据

模型参数里的知识会过时。  
如果问题依赖最新价格、库存、政策或搜索结果，工具调用比“靠记忆猜”更可靠。citeturn0search4

## 缺口二：确定性计算

计算器、数据库查询、代码执行器这类工具可以提供可校验结果。  
相比让模型口算或硬猜，工具结果更稳定。

## 缺口三：外部动作执行

像发邮件、调接口、查询系统状态这类事，本质上不是语言生成，而是系统操作。  
这些能力天然应该交给工具，而不是交给模型“想象自己做了”。citeturn0search3turn0search4

## 结论收束

**模型需要工具，不是因为模型太弱，而是因为很多任务本来就不该只靠语言生成完成。**
