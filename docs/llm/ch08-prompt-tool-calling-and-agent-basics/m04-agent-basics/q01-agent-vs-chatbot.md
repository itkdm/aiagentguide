# 8.4.1 Agent 和普通聊天机器人有什么区别？

先给结论：**普通聊天机器人主要负责“接收输入并回复”，Agent 则会围绕目标做多步判断、调用工具、处理中间状态，并根据结果继续行动。** citeturn0search0turn0search1

## 聊天机器人更像“单轮或多轮回复器”

普通聊天机器人通常做的是：

- 理解用户问题
- 生成回答
- 在有限上下文里继续对话

它的核心能力是对话与文本生成，而不是执行复杂任务。

## Agent 更像“会规划的执行器”

Agent 典型会做这些事：

1. 理解目标  
2. 决定下一步做什么  
3. 调用工具或访问外部系统  
4. 根据结果继续规划  

这意味着 Agent 的重点不在“会聊天”，而在“会推进任务”。citeturn0search0turn0search1

## 一个工程判断

如果你的系统只是：

- 回答问题
- 做摘要
- 基于固定输入生成文本

那它更像聊天机器人或文本系统。  
如果它需要：

- 多步决策
- 工具调用
- 中间状态管理
- 根据结果调整下一步

那它才更接近 Agent。

## 结论收束

**会对话不等于 Agent，会推进任务才更接近 Agent。**
