# 8.4.3 Workflow 和 Agent 的边界在哪里？

先给结论：**Workflow 是“流程预先写好，模型只是其中一环”；Agent 是“系统会根据中间结果动态决定下一步”。** citeturn0search1turn0search2

## Workflow 的特点

Workflow 更适合：

- 步骤固定
- 分支可预见
- 输入输出结构稳定

例如固定的：抽取 -> 分类 -> 生成报告。  
这种情况下，系统行为应该由工程逻辑主导，而不是交给模型临场决定。

## Agent 的特点

Agent 更适合：

- 步骤不固定
- 需要根据结果决定下一步
- 可能多次调用不同工具

例如复杂调研、跨系统任务处理、多轮决策。

## 一个实用判断标准

你可以问两个问题：

1. 下一步流程能不能在写代码时就确定？
2. 如果中间结果变化，系统是否需要重新规划？

如果第一个答案是“能”，优先 Workflow。  
如果第二个答案是“需要”，才更接近 Agent。

## 结论收束

**能写死流程，就别先上 Agent。**
