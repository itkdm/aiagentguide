# 8.4.2 Agent 为什么通常建立在 LLM + 工具 + 记忆 / RAG 之上？

先给结论：**因为 LLM 负责理解与规划，工具负责执行与取数，记忆 / RAG 负责补足状态与外部知识。缺任何一层，Agent 都很容易退化成“只会说话”。** citeturn0search0turn0search3turn0search4

## LLM 负责什么

LLM 主要负责：

- 理解目标
- 决定下一步动作
- 组织语言与结果

没有 LLM，Agent 就缺少通用推理与任务组织能力。

## 工具负责什么

工具负责：

- 搜索
- 查询
- 计算
- 执行外部操作

没有工具，Agent 很多时候只能“想象自己完成了任务”。citeturn0search3turn0search4

## 记忆 / RAG 负责什么

记忆或 RAG 负责：

- 补足当前上下文之外的信息
- 保持跨步骤状态
- 提供最新、可检索的事实来源

没有这一层，Agent 的多步任务容易丢状态，也容易依赖过时知识。citeturn0search4

## 结论收束

**LLM 决策，工具执行，记忆补状态。这三层加起来，Agent 才真正可用。**
