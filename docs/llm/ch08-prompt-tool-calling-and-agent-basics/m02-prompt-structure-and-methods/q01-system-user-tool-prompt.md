# 8.2.1 System Prompt、User Prompt、Tool Prompt 分别起什么作用？

先给结论：**System Prompt 负责定义全局规则，User Prompt 负责表达当前任务，Tool Prompt 或工具结果负责补充外部信息与执行反馈。三者混在一起，系统就更容易失控。**

## System Prompt 在控制什么

System Prompt 更像“全局运行规则”，通常包含：

- 角色设定
- 行为边界
- 输出原则
- 安全与拒答要求

它不应该塞太多具体业务数据，而应主要负责“长期稳定约束”。

## User Prompt 在表达什么

User Prompt 的职责是表达当前这次请求：

- 用户要完成什么任务
- 当前有哪些输入材料
- 输出需要满足什么约束

它是任务级上下文，不是系统级规则。

## Tool Prompt / 工具结果在补什么

工具调用场景里，模型往往需要读取外部系统返回的结果。  
这部分信息的职责不是“规定规则”，而是“提供事实依据”。

例如：

- 搜索结果
- 数据库查询结果
- API 返回值

## 工程上的一个常见错误

最常见的问题是把三种角色混写：

- 把系统规则塞进用户输入
- 把工具结果写成模糊自然语言
- 把临时任务要求写进长期系统提示

这会直接降低可控性。

## 结论收束

**System 管规则，User 管任务，Tool 管事实。角色分工越清楚，系统越稳定。**
