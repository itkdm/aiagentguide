# 8.2.4 为什么 Prompt 设计会直接影响模型输出风格和稳定性？

先给结论：**Prompt 会改变模型对任务的理解边界，也会改变它更倾向生成哪类回答，所以会直接影响风格和稳定性。** 

## 影响风格：因为你在改“回答偏好”

当 Prompt 明确要求：

- 用正式语气
- 按步骤输出
- 只给结论不展开

模型就会把这些约束当成当前任务的一部分，从而改变输出风格。

## 影响稳定性：因为你在减少歧义

很多不稳定不是模型“突然失常”，而是任务表达太模糊。  
Prompt 越清楚，歧义越少，输出越稳定。

例如这些信息都会提升稳定性：

- 明确输出格式
- 给出边界条件
- 提供参考示例

## 工程上的直接做法

如果你想提高稳定性，优先改这些：

1. 明确任务目标
2. 明确输出格式
3. 明确不该做什么
4. 必要时加示例

## 结论收束

**Prompt 影响风格，是因为它在塑造当前任务；Prompt 影响稳定性，是因为它在减少歧义。**
