# 8.2.3 为什么不同任务类型需要不同 Prompt 结构？

先给结论：**因为不同任务失败的原因不同，所以 Prompt 结构也必须围绕不同风险点来设计。** 

## 分类任务和生成任务的问题不一样

例如：

- 分类任务容易出错在标签边界
- 摘要任务容易出错在遗漏重点
- 代码任务容易出错在格式与约束不明确

既然失败模式不同，Prompt 当然不能一套打天下。

## Prompt 结构本质上在解决“任务风险”

你可以把 Prompt 结构理解成一个控制手段：

- 规则多的任务：需要更强约束
- 格式敏感的任务：需要更明确输出模板
- 推理复杂的任务：需要更清晰步骤拆解

## 一个实用的设计方法

先不要问“用哪种模板”，先问：

1. 这个任务最容易错在哪里？
2. 这些错误能不能靠示例、约束或步骤拆解降低？
3. 输出是否能被自动校验？

这样设计出来的 Prompt 才是有针对性的。

## 结论收束

**Prompt 结构不是风格问题，而是任务控制问题。**
