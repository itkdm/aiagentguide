# 4.1.3 预训练和传统监督学习有什么区别？

先给结论：

**预训练关注通用表示学习，监督学习关注特定任务优化；前者依赖大规模无标注语料，后者依赖高质量标注数据；前者强调迁移能力，后者强调任务内性能。**

所以它们不是互相取代，而是职责不同。

## 区别一：训练目标不同

- 预训练：学习通用语言模式（例如 next token prediction）
- 监督学习：学习具体任务映射（例如分类、序列标注）

预训练的目标更通用，监督学习的目标更具体。

## 区别二：数据来源不同

- 预训练：依赖海量无标注语料
- 监督学习：依赖人工标注数据

这意味着：

- 预训练可以吃下更大规模的数据
- 监督学习的规模往往被标注成本限制

## 区别三：迁移方式不同

- 预训练通过迁移或微调，把通用能力转为任务能力
- 监督学习通常只能在任务内部优化，跨任务迁移有限

这也是为什么预训练成为 LLM 的基础，而监督学习更多作为后续对齐和任务适配工具。

## 区别四：工程成本结构不同

- 预训练成本高，但可复用性强
- 监督学习成本相对低，但容易重复建设

换句话说：

- 预训练是“重投入、广复用”
- 监督学习是“轻投入、窄收益”

这决定了它们在真实工程中的不同定位。

## 区别五：评估方式不同

- 预训练更关注整体能力与迁移效果
- 监督学习更关注特定任务指标

所以评估角度也不同：

- 预训练看“能不能泛化到更多任务”
- 监督学习看“在这个任务上是不是最好”

## 什么时候该重预训练，什么时候该重监督

常见判断是：

- 如果你需要通用能力和长期迁移，预训练更关键
- 如果你只关心某个稳定任务，监督学习更直接

但在 LLM 工程里，这两者往往不是二选一，而是组合使用：

- 先预训练出基础能力
- 再通过监督或对齐把能力拉到目标场景

## 你可以先这样记住

- 预训练学通用能力，监督学习学具体任务
- 预训练靠数据规模，监督学习靠标注质量
- 预训练提供迁移基础，监督学习做精准对齐
- 工程上通常是“预训练打底 + 监督对齐”
