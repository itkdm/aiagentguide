# 4.1.1 什么是预训练（Pretraining）？

先给结论：

**预训练是指在大规模通用语料上，用自监督目标训练模型，让它先学到通用语言表示和模式，再迁移到具体任务。**

这里的关键词是：

- `大规模通用语料`
- `自监督目标`
- `通用表示`
- `迁移到下游任务`

## 预训练到底在训练什么

以 LLM 为例，预训练最常见的目标就是：

- 给定前文，预测下一个 token

这类目标不需要人工标注，因为训练信号直接来自文本本身。

比如一句话：

```text
今天的天气很
```

模型的训练目标就是预测下一个 token 可能是“好”“不错”“糟糕”等。

通过海量文本反复训练，模型逐步学会：

- 语言规律
- 语义关联
- 常见知识表达模式
- 任务指令的语言形式

这些能力共同构成了“通用表示”。

## 预训练和“自监督”是什么关系

预训练通常使用自监督学习（self-supervised learning）。

它的特点是：

- 不依赖人工标注标签
- 标签来自输入本身的结构

例如：

- 预测下一个 token
- 填空预测
- 对比学习式的相似性判断

这使得预训练可以使用极大规模的语料，而不被标注成本限制。

## 一个最简训练流程

```text
1. 收集大规模通用语料
2. tokenization
3. 构造自监督训练目标（例如 next token prediction）
4. 大规模训练模型
5. 得到预训练模型
```

注意：这个流程强调的是“先学通用能力”，不是“直接解决某个具体业务任务”。

## 预训练模型能直接用吗

部分任务可以直接用，但通常效果不够稳。

原因是：

- 预训练学的是通用模式，不是针对某个具体任务
- 模型可能知道很多知识，但不一定知道如何按用户意图表达

所以在真实系统中，预训练通常只是第一步。

后面还会有：

- 指令微调
- 对齐训练
- 任务微调或 RAG 组合

这些步骤负责把通用能力对齐到具体场景。

## 一个常见误解

误解：预训练只是“规模大一点的训练”。

更准确的理解是：

**预训练的关键不在“规模”，而在“目标是通用表示学习”。**

规模当然重要，但它只是让这种通用表示学习更充分。

## 你可以先这样记住

- 预训练是用自监督目标在大规模语料上训练模型
- 目标是让模型学到通用语言表示
- 预训练是 LLM 能力的起点，不是终点
- 后续任务对齐通常还需要微调或系统级组合
