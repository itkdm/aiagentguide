# 4.4.2 为什么预训练模型“会续写”，但不一定“会做助手”？

先给结论：

**预训练模型学到的是“在给定上下文下的合理续写分布”，而助手能力需要“按用户意图与规范执行行为”。两者目标不同，所以会续写不等于会当助手。** citeturn2academia12turn1search4turn1academia14

## “续写能力”到底是什么

自回归预训练把任务定义为“预测下一个 token”。citeturn2academia12  
这意味着模型学会了：

- 在某种语境下，语言通常如何继续
- 哪类回答在训练语料里更常见

它并不知道“你想要它做什么”，它只在模拟“像训练语料里那样继续”。citeturn2academia12turn1search8

## “助手能力”需要额外的对齐目标

要让模型真正像“助手”一样工作，至少要对齐到这些目标：

- **遵循指令**：明确区分用户指令与背景文本
- **偏好一致**：在多种可行回答中选“更有用、更安全”的那类
- **拒绝或纠错**：识别不合规请求并给出替代策略

这就是为什么后续会有“指令微调 + 偏好学习（RLHF）”这样的对齐过程。citeturn1search4turn1academia14

## 你可以把它理解成“行为策略升级”

预训练学到的是“语料统计分布”。  
对齐训练学到的是“符合人类偏好的行为策略”。citeturn1search4turn1academia14

如果没有对齐步骤，模型会：

- 按语料中常见风格续写
- 在模糊指令下给出“看似合理但不一定有用”的内容
- 缺少统一的拒答规范

这就是“会续写但不会当助手”的本质。

## 工程上怎么把模型变成“助手”

一条常见路径是：

1. **指令微调（SFT）**：用高质量“指令-回复”样本，让模型学会遵循指令。
2. **偏好学习（RLHF / DPO）**：用人类偏好或比较样本，让模型学会选更好的回答。
3. **系统约束**：通过 system prompt、工具调用规则、安全策略把行为锁到可控范围。

这些步骤不会改变“语言能力”的底座，但会改变它的行为策略。citeturn1search4turn1academia14

下面是一个最小化的指令微调样本格式示意：

```json
{"instruction": "把这段话改写成更正式的语气", "input": "今天超忙，项目要延期了", "output": "近期工作量较大，项目进度需要适当调整。"}
```

## 常见坑

- 只用少量指令样本就期待“助手行为稳定”
- 偏好数据口径不一致，导致输出风格漂移
- 把 system prompt 当成“万能对齐”，忽视训练分布

你可以记住一句话：**预训练解决“会说”，对齐训练解决“会做”。**
