# 5.1.2 为什么“会补全文本”和“会遵循用户意图”是两件事？

先给结论：**“补全文本”是在模拟语料分布，“遵循意图”是在执行任务。前者依赖语言建模目标，后者依赖指令对齐目标。** 

## 一个最小对比例子

同样一句输入，模型可能走两条完全不同的路径：

```text
输入：请用三条要点总结下面这段话：
（后面跟一段业务描述）
```

如果模型只会“补全文本”，它更可能产生“像原文那样继续写”的内容。  
如果模型会“遵循意图”，它会输出三条清晰要点。

这个差异不来自“语言能力强弱”，而来自**任务定义的不同**。

## 从目标函数角度看差别

预训练目标是“预测下一词”。  
指令对齐目标是“在多个可行输出中选择更符合用户意图的那一个”。

这意味着：

- 预训练让模型“像语料那样写”
- 对齐训练让模型“像任务那样做”

## 从输入格式角度看差别

“补全文本”的输入通常是自然文本上下文。  
“遵循意图”的输入更像“指令 + 约束 + 期望输出格式”。

如果模型从未见过这种“指令结构”，它就无法稳定把指令当成约束，而会把它当成普通文本继续写。

## 这对工程有什么影响

你需要把“完成文本”和“执行意图”当成两类能力来评估：

- 评估补全文本：看连贯性、语法与风格一致性
- 评估执行意图：看格式遵循、步骤完整性与约束满足度

如果你只用“文本流畅度”来衡量助手，就会误判它“可用”。

## 结论收束

**会补全文本是语言建模能力，会遵循意图是行为对齐能力。**  
这两件事需要不同的数据与训练目标支撑。
