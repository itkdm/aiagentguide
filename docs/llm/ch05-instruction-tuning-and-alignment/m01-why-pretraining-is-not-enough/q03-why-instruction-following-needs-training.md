# 5.1.3 为什么指令跟随能力需要单独训练？

先给结论：**指令跟随不是语言能力的自然副产物，它需要“指令格式理解 + 目标选择偏好”，而这些在预训练中几乎没有被系统学习。** 

## 原因一：指令分布与预训练分布不同

预训练语料大多是自然文本，而指令数据是“任务描述 + 约束 + 输出格式”。  
这两类数据的结构差异很大，所以模型必须显式学习“指令的语用规则”。

## 原因二：需要学习“偏好选择”

同一条指令通常对应多个合理回答。  
指令跟随能力的关键在于“选更符合意图的那个”，而这需要偏好信号或对比数据来教会模型。

## 一条最小可行的训练路径

如果你要从零把模型推到“会听指令”，可以按这个顺序：

1. **指令微调（SFT）**：用高质量指令-回复样本让模型学会格式与任务映射。
2. **偏好学习（RLHF / DPO）**：用比较样本让模型学会选择更好的回答。
3. **系统约束**：用系统提示、工具调用与安全策略把行为限制在可控范围。

## 最小指令数据格式示意

```json
{"instruction": "把这段话改写为正式语气", "input": "今天开会推迟了", "output": "今日会议延期举行，具体时间另行通知。"}
```

关键不是“样本数量越多越好”，而是样本能覆盖你的真实任务结构与约束方式。

## 常见误区

- 只做少量指令样本训练，却希望行为稳定
- 只靠 system prompt 约束，忽视训练分布
- 偏好数据口径混乱，导致风格漂移

你可以先记住一句话：**指令跟随是行为学习，不是语言建模。**
