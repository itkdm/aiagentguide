# 5.2.5 In-Context Learning 和微调分别适合解决什么问题？

先给结论：**ICL 适合“低成本试验与临时迁移”，SFT/Instruction Tuning 适合“稳定行为与长期可控”。** citeturn1search58turn2view0turn2view4

## 先看三条最关键的判断标准

1. **稳定性要求**  
   如果任务需要持续一致输出，优先微调。citeturn2view0turn2view4

2. **成本与速度**  
   如果你需要快速试验、没有训练预算，优先 ICL。citeturn1search58

3. **可控性与可评测性**  
   如果你需要明确的格式与约束，微调更可控。citeturn2view0turn2view4

## 一个工程化的选型流程

你可以按这个顺序做选择：

1. **先用 ICL 试验**：验证任务是否可行、输出是否有潜力。
2. **再决定是否微调**：当你需要稳定行为或规模化交付时再投入微调成本。
3. **最后做对齐**：如果涉及安全或规范性要求，再引入偏好学习或系统约束。citeturn2view0turn2view4

## 常见误区

- 把 ICL 当成“可以长期稳定上线”的方案
- 过早微调，导致成本高却没明确任务边界
- 用“语言流畅度”替代“任务完成度”评估

## 结论收束

**ICL 是“先验证”，微调是“再规模化”。**  
当任务稳定、对输出一致性有要求时，微调几乎是必选项。citeturn1search58turn2view0turn2view4
