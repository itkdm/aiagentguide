# 5.2.1 什么是 SFT（Supervised Fine-Tuning）？

先给结论：**SFT 就是用“有监督样本”对预训练模型做微调，让模型在指定任务或风格上更稳定。** 

它本质上是一种“行为塑形”：用成对的输入-输出样本，把模型拉向你想要的回答分布。

## SFT 的数据长什么样

SFT 常见的样本格式是：

```json
{"input": "请把这段话改写成正式语气：今天太忙了，项目要延期。", "output": "近期工作量较大，项目进度需要适当调整。"}
```

如果你做的是指令类任务，输入可以拆成 `instruction`、`input`、`output` 三段，效果更稳定。

## SFT 解决什么问题

你可以把 SFT 用在这些场景：

- **行为一致性**：让模型在同类问题上输出稳定格式
- **风格与口吻**：让模型学会你的写作或品牌风格
- **任务对齐**：把通用语言能力落地成可执行任务

如果你已经有明确任务目标、样本可控，SFT 是最快的工程路径。

## 一个最小可行的训练流程

1. **确定任务边界**：定义“哪些输入应该给出什么样的输出”。
2. **构建样本集**：优先覆盖真实业务场景与边界条件。
3. **设定评测集**：保留一批不参与训练的样本做回归验证。
4. **微调与验证**：观察是否提升稳定性、格式遵循与可执行性。

这能避免“训练损失下降，但线上表现不稳定”的错觉。

## 常见坑

- **数据分布偏窄**：只覆盖理想输入，导致上线后崩溃
- **样本质量不稳**：标注口径不一致，输出风格漂移
- **只看训练指标**：忽视真实任务表现

你可以记住一句话：**SFT 不是让模型更聪明，而是让模型更像你希望的“工作方式”。**
