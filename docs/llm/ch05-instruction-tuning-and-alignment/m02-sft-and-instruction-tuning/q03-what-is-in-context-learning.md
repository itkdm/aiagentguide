# 5.2.3 什么是 In-Context Learning？

先给结论：**In-Context Learning（ICL）是模型在推理时通过提示中的示例学习任务模式，但参数不更新。** 

它是“把小样本放进提示里，让模型临时迁移”的能力，而不是一次真正的训练。

## ICL 是怎么工作的

典型形式是：

```text
示例1：输入A -> 输出A
示例2：输入B -> 输出B
现在：输入C -> ?
```

模型通过上下文中的示例来推断“这是什么任务、应该用什么格式输出”。

## ICL 的优势

- **零训练成本**：不需要更新参数或准备训练集
- **快速试验**：适合探索任务可行性
- **低风险**：不会影响模型原始能力

## ICL 的局限

- **稳定性弱**：示例稍改，输出波动明显
- **上下文限制**：样本数量受窗口长度限制
- **难以形成长期一致的行为**

这些问题来自“它只在当前上下文临时生效”。

## 什么时候优先用 ICL

你可以在这些场景优先考虑 ICL：

- 任务不稳定、还在探索阶段
- 你没有足够标注数据
- 希望快速验证需求或原型

如果任务已经稳定、需要持续一致的输出，SFT 或 Instruction Tuning 通常更可靠。

你可以先记住一句话：**ICL 适合“快试”，微调适合“稳用”。**
