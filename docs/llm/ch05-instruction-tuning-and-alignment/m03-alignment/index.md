# 5.3 对齐（Alignment）

先给结论：**对齐是让模型行为更贴近“人类意图与可接受规范”的过程，它不是提升能力，而是把能力约束到可用、可控与安全的范围。** 

这一节的重点不是把“对齐”讲成一个抽象口号，而是把它还原成真实系统里的行为控制问题。

很多团队在讨论对齐时容易两头跑偏：

- 一种是把它理解成纯安全话题，觉得和产品体验关系不大
- 另一种是把它理解成“模型更听话了”，却忽略它其实会直接改变拒答策略、风格口径、风险边界和可上线程度

这一节会让你看到，对齐并不是一个附加层，而是把模型从“可能有能力”收敛成“可以稳定交付”的关键环节。  
如果你后面要理解 RLHF、DPO、模型规范、拒答行为和风格差异，这里是中间桥梁。

## 学完后你应该建立的判断

- 对齐解决的是“行为策略”，不是“语言能力”
- 没有对齐的模型在真实系统里很难稳定交付
- 对齐程度会直接改变产品体验与风险边界
