# 5.3.2 为什么大模型不仅要“能力强”，还要“行为可控”？

先给结论：**能力强只代表“会生成”，但产品需要“稳定、可控、可解释的生成”。这就是行为可控的重要性。** citeturn0search0turn0search3

## 现实系统里“不可控”的具体代价

如果没有对齐，模型常见问题包括：

- 同一问题风格漂移、答案自相矛盾
- 在边界场景下给出不合规输出
- 对指令的格式与约束不稳定

这些问题会直接体现在上线系统的成本上：更多人工校验、更高安全风险、更差用户信任。citeturn0search0turn0search2

## 对齐为什么能改善“可控性”

对齐的核心是用偏好信号与策略约束，把模型输出分布收敛到“人类更能接受的范围”。citeturn0search0turn0search1  
这会让模型在同一任务上更稳定，也更容易被系统约束控制。

## 工程上如何判断“可控性”

你可以用三类检查来判断模型是否可控：

1. **一致性检查**  
   对相同问题多次提问，观察输出风格与结论是否稳定。

2. **约束遵循检查**  
   给定格式、长度、步骤等硬约束，观察是否稳定遵循。

3. **边界与拒答检查**  
   给出不合规或越权请求，观察是否稳定拒答并给出替代方案。citeturn0search0turn0search2

如果这些检查表现不稳定，即使“能力看起来很强”，也很难上线。citeturn0search0

## 结论收束

**能力强是必要条件，但行为可控才是可交付条件。**
