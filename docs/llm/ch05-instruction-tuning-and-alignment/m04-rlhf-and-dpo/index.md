# 5.4 RLHF 与 DPO

先给结论：**RLHF 用“人类偏好 + 奖励模型 + 强化学习”对齐行为，DPO 用“偏好对比”直接优化模型，不需要显式奖励模型与 RL。** 

这一节会把两条路线拆开讲清：它们各自的训练链路、工程代价与风险边界，以及为什么对齐方式会直接改变用户体验。

- [什么是 DPO？](./q02-what-is-dpo)

## 学完后你应该建立的判断

- RLHF 与 DPO 都是“偏好对齐”，但工程路径与成本不同
- 数据质量决定了对齐上限，算法只是放大或收敛偏好
- 对齐方法会改变风格、拒答策略与稳定性
