# 5.4 RLHF 与 DPO

先给结论：**RLHF 用“人类偏好 + 奖励模型 + 强化学习”对齐行为，DPO 用“偏好对比”直接优化模型，不需要显式奖励模型与 RL。** 

这一节要解决的是一个更细的问题：  
**既然已经知道模型需要对齐，那对齐到底怎么做，为什么后来很多团队又从 RLHF 转向了更直接的偏好优化方法。**

RLHF 和 DPO 看上去像两个算法名，但读者真正需要建立的是工程判断：

- 它们依赖什么类型的数据
- 训练链路有多复杂
- 各自容易把问题做成什么样
- 为什么它们最后会体现在用户看到的风格、稳定性和拒答策略上

这一节会把这两条路线放在同一张图里看，不只是讲原理，也会讲为什么很多团队在工程上更偏好某一类方法。

## 学完后你应该建立的判断

- RLHF 与 DPO 都是“偏好对齐”，但工程路径与成本不同
- 数据质量决定了对齐上限，算法只是放大或收敛偏好
- 对齐方法会改变风格、拒答策略与稳定性
