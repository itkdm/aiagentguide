# 5.4.2 什么是 DPO？

先给结论：**DPO（Direct Preference Optimization）是一种直接用偏好对比优化模型的方法，不需要显式奖励模型与强化学习优化。** 

## DPO 的核心思想

DPO 用成对偏好数据训练，让模型“更偏向被选择的回答、远离被拒绝的回答”。  
它等价于“直接拟合偏好分布”，而不是先训练奖励模型再做 RL。

## DPO 的数据长什么样

与 RLHF 相同，DPO 也使用成对偏好数据：

```json
{
  "prompt": "给出一段简洁的用户引导",
  "chosen": "请先填写资料，再点击提交。",
  "rejected": "你最好把资料填了。"
}
```

只要偏好口径一致，DPO 可以直接在这些对比样本上训练。

## DPO 的工程优势

和 RLHF 相比，DPO 常见优势是：

- **链路更短**：不需要训练奖励模型
- **更易稳定**：没有 RL 的不稳定优化过程
- **成本更低**：数据结构更简单、训练更直接

这也是 DPO 在中小团队中更容易落地的原因。

## 但它也有边界

DPO 的效果高度依赖偏好数据质量。  
如果偏好口径不一致，DPO 会稳定地学到“错误偏好”。

## 你可以先记住一句话

**DPO 是“用偏好对比直接对齐”的方法。**
