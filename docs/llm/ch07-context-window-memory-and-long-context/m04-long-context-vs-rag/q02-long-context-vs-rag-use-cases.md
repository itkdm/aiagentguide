# 7.4.2 长上下文和 RAG 分别适合什么场景？

先给结论：**长上下文适合“需要同时参考整段上下文关系”的任务，RAG 适合“从大规模知识中找少量相关证据”的任务。** citeturn0search0turn0search1

## 更适合长上下文的场景

长上下文更适合这些任务：

- 长对话连续跟进
- 整篇文档改写、总结、审阅
- 需要跨段理解上下文关系的问题

这类任务的重点是“保留整体语境”。citeturn0search1

## 更适合 RAG 的场景

RAG 更适合这些任务：

- 从知识库中定位事实答案
- 面向经常更新的数据源问答
- 多文档检索、引用与证据定位

这类任务的重点是“从大集合里先找对内容”。citeturn0search0

## 最常见的真实做法：组合使用

很多真实系统会这样做：

1. 先用 RAG 检索最相关内容  
2. 再把检索结果放进上下文窗口  
3. 用模型做整合、解释和生成

这通常比“全靠大窗口”更稳，也比“只检索不组织”更好用。citeturn0search0turn0search1

## 一个简单判断标准

你可以这样选：

- **重点在整体语境**：优先长上下文
- **重点在精准找证据**：优先 RAG
- **两者都要**：组合使用

## 结论收束

**长上下文偏“保留全局”，RAG 偏“缩小搜索空间”。**
