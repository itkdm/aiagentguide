# 7.4.3 为什么“上下文窗口很大”不等于“就不需要检索”？

先给结论：**上下文窗口大，只代表“可以放进去更多内容”；检索的核心价值是“先找到该放进去什么”。这两件事不是一回事。** 

## 大窗口解决的是容量，不是选择

就算模型能处理很长的上下文，你仍然要回答一个问题：  
**在大量候选文档里，哪些内容值得放进去？**

这个步骤本质上还是检索问题。

## 没有检索会出现什么问题

如果只靠大窗口，常见风险包括：

- 塞入过多噪声信息
- 关键信息被淹没
- 成本和延迟快速上升

这些问题在长上下文场景里会被进一步放大。

## 检索的真实价值

检索不是因为“窗口不够大”才存在，  
而是因为真实系统里永远存在：

- 数据太多
- 数据在变化
- 查询只需要其中一小部分

所以只要你的知识源足够大，检索就几乎一直有价值。

## 结论收束

**大窗口降低了“放不下”的问题，但没有消灭“先找什么”的问题。**
