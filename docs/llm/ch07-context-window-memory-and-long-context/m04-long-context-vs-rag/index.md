# 7.4 长上下文与 RAG 的关系

先给结论：**长上下文解决的是“能放多少信息”，RAG 解决的是“该取哪些信息”。两者不是替代关系，而是常见的互补关系。** 

这一节会把长上下文和 RAG 放在一起看，帮助你理解它们分别适合什么场景，以及为什么“上下文窗口很大”仍然不等于“就不需要检索”。

## 学完后你应该建立的判断

- 长上下文提供容量，但不自动提供筛选能力
- RAG 的核心价值是检索、定位与时效更新
- 真实系统里常见做法是“检索 + 合理上下文预算”
