# 7.1 上下文窗口是什么

先给结论：**上下文窗口就是模型一次推理能“看到”的最大序列长度，它是模型结构与配置决定的硬限制。** 

这一节的重点不是只给出一个定义，而是把上下文窗口放回真实系统里理解。  
很多人第一次接触这个概念时，只记住了“某个模型支持多少 K token”，但没有意识到这其实会直接限制：

- 一次能塞进多少资料
- 系统提示和用户输入如何分配预算
- 多轮对话为什么会越来越贵
- 为什么长文档、长对话、长工具轨迹都会挤占有效空间

所以这部分要建立的不只是一个概念认识，而是一种产品和工程判断：  
**上下文窗口不是宣传参数，而是系统资源预算。**

只有先把这点想清楚，后面你再看长上下文、记忆、RAG 和 Agent 时，才不会把“能放进去”和“能用得好”混成一回事。

## 学完后你应该建立的判断

- 上下文窗口是“硬上限”，不是软建议
- 超出窗口的信息不会参与当前推理
- 长上下文带来成本与质量风险，需要工程控制
