# 7.1.2 上下文窗口为什么会限制模型一次能处理的信息量？

先给结论：**上下文窗口是硬上限，超出的内容不会进入模型计算；即使在窗口内，信息也会出现“注意力竞争”。** citeturn2search3turn0search4

## 原因一：超过上限的内容无法进入模型

当输入超过模型配置的最大序列长度时，系统只能截断、分段或压缩。citeturn2search3  
这意味着一部分信息会被直接丢弃，模型根本“看不到”。citeturn2search3

## 原因二：窗口内也存在注意力稀释

即使内容没有超长，长上下文仍可能让关键信息“被淹没”。  
研究表明，当关键信息位于长上下文中间位置时，模型容易忽略它。citeturn0search4

## 工程上的直接影响

你会发现这些现象：

- 提示过长，模型忽略开头或中间细节  
- 信息越多，输出反而越不稳定  
- 关键约束被“挤掉”导致偏离任务

## 结论收束

**上下文窗口不仅限制“能不能放得下”，也影响“能不能用得好”。**
