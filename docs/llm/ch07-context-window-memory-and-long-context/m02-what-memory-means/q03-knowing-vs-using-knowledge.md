# 7.2.3 为什么模型“记得某些知识”不等于它能稳定调用这些知识？

先给结论：**模型可能在参数里“知道”，但在具体任务中无法稳定调用；因为调用依赖提示、上下文位置与解码策略。** citeturn0search1turn0search4

## 原因一：调用依赖上下文组织

即使模型“知道某个事实”，如果提示里没有明确触发点或约束，它也可能不调用。citeturn0search1  
这就是为什么同样知识在不同提示下表现不一致。

## 原因二：长上下文会让关键信息失效

长上下文中，关键信息可能被“中间丢失”，导致模型忽略本该使用的知识。citeturn0search4

## 原因三：解码策略会放大不确定性

如果采样设置偏发散，模型可能选择“看似合理但不准确”的路径。citeturn0search1  
因此“记得”并不保证“稳定调用”。

## 工程上的解决路径

你可以用这些手段提高稳定调用率：

1. **提示结构化**：把关键知识放在显著位置，避免被淹没  
2. **检索增强**：让模型在回答前读取相关文档  
3. **约束输出**：降低温度或使用更确定的解码策略  

## 结论收束

**模型“知道”是基础，“能稳定用”靠工程组织。**
