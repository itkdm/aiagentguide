# 7.2 模型的“记忆”到底是什么

先给结论：**大模型的“记忆”主要有两类：参数记忆（知识内化在参数里）与上下文记忆（提示中临时可见的内容）。两者性质完全不同。** 

这一节要解决一个非常常见、也非常容易误导系统设计的问题：  
我们嘴里说模型“记住了什么”，到底是在说参数里学到的统计知识，还是在说当前对话里暂时看到的信息。

如果这两个概念不分清，后面会出现很多错误判断，比如：

- 把参数记忆当成可追溯知识库
- 把上下文记忆当成长期状态存储
- 看到模型偶尔能答出来，就误以为它能稳定调用

这一节会把“记忆”拆成更接近工程现实的两类来源，并解释为什么“模型知道”和“系统能稳定用出来”之间还隔着提示组织、检索、上下文预算和推理稳定性这些环节。

## 学完后你应该建立的判断

- 参数记忆是“长期但不一定可控”的知识来源
- 上下文记忆是“临时但可控”的任务信息来源
- 稳定调用需要提示组织或检索机制支持

## 这一节适合怎么读

- 如果你以前把“记忆”当成一个模糊概念，这一节要先读透
