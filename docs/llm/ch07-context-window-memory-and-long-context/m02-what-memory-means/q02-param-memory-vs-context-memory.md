# 7.2.2 参数记忆和上下文记忆有什么区别？

先给结论：**参数记忆来自训练，是“长期存在但不易精确编辑”的；上下文记忆来自当前请求，是“短期存在但高度可控”的。**  
两者不只是来源不同，而是在可控性、时效性、可验证性和系统用途上几乎都不同。

很多教程会简单地说：

- 参数记忆是模型学到的
- 上下文记忆是 prompt 里给的

这句话本身没错，但如果只停在这里，还是不够指导工程决策。  
真正重要的是理解：**这两类记忆分别适合解决什么问题，不适合解决什么问题。**

## 第一类：参数记忆

参数记忆是模型在训练过程中内化进权重里的内容。  
它通常包括：

- 语言模式
- 常见世界知识
- 常见任务形式
- 一些通用推理和表达习惯

它的特点是：

- 不需要你每次重新喂
- 天然跟着模型一起存在
- 可以覆盖很广的通用领域

但它也有明显局限：

- 很难逐条修改
- 可能过时
- 来源通常不可追溯
- 某条知识是否能稳定调出，并不完全可控

所以参数记忆更像一个“底座知识分布”，而不是可直接管理的知识库。

## 第二类：上下文记忆

上下文记忆来自当前请求里显式放入的内容。  
它包括：

- system prompt
- 用户问题
- few-shot 示例
- 对话历史
- 检索结果
- 工具输出

它的特点和参数记忆正好相反：

- 每次请求都可以重写
- 可以精确控制给什么、不给什么
- 更新速度快
- 更容易做出处、权限和审计控制

但它也有天然限制：

- 只能在当前窗口内生效
- 会被上下文长度限制
- 组织不好就容易被淹没
- 不会自动长期保留

所以它更像“运行时工作记忆”，而不是模型的长期底座。

## 最核心的区别不是来源，而是可控性

如果只从来源看，两者的区别似乎很学术。  
但从工程角度看，最重要的其实是可控性。

### 参数记忆的可控性低

你不能轻易对它说：

- 删除某一条旧知识
- 精确加入一条新知识
- 保证以后每次都引用这一条事实

要改它，通常得靠：

- 再训练
- 微调
- 对齐调整

这些手段成本高，而且通常也不是逐条可控编辑。

### 上下文记忆的可控性高

你可以在当前请求里直接决定：

- 给哪些材料
- 用什么顺序给
- 哪些是必须遵守的约束
- 哪些证据必须优先使用

这也是为什么在真实系统里，凡是要求：

- 最新
- 私有
- 可追溯
- 可审计

的知识，通常都更适合走上下文链路，而不是盲信参数记忆。

## 另一个关键区别：时效性

参数记忆很容易过时。  
因为它固化的是训练截止之前的统计知识。

上下文记忆则可以是最新的。  
只要你在请求里放入最新文档、检索结果、数据库状态，模型就能在当前任务中使用这些内容。

所以对于：

- 企业内部文档
- 实时业务数据
- 频繁变更规则
- 受权限控制的信息

上下文记忆通常比参数记忆更可靠。

## 再一个关键区别：可追溯性

参数记忆的问题在于，它很难告诉你：

- 这条知识是从哪学来的
- 为什么这次引用的是这个版本
- 是否能逐条做权限控制

但如果上下文记忆来自：

- RAG 检索
- 数据库查询
- 工具调用

你通常更容易做到：

- 给出处
- 做审计
- 做权限隔离
- 控制只允许使用当前证据回答

这也是为什么企业系统里，很多知识型任务宁愿多做一层检索，也不愿完全依赖模型“自己记得”。

## 对系统设计意味着什么

你可以用下面这套判断来决定该依赖哪种记忆：

### 更适合依赖参数记忆的情况

- 通用背景知识
- 通用表达能力
- 常见任务格式
- 不要求逐条追溯的基础认知

### 更适合依赖上下文记忆的情况

- 私有知识
- 最新知识
- 高风险事实
- 需要出处、权限和审计的内容
- 当前任务特定约束

换句话说：

- 参数记忆负责“底座能力”
- 上下文记忆负责“当前任务真正要用的信息”

## 为什么成熟系统通常会混合使用

真实系统很少只靠其中一种。  
更常见的是混合：

- 参数记忆提供通用语言能力、背景知识和表达能力
- 上下文记忆提供最新、私有、任务相关、可控的信息

这也是 RAG 系统的基本逻辑：  
不是否定参数记忆，而是用可控知识补足参数记忆的时效和可追溯短板。

## 一个简单自查法

如果你面对某个知识问题，不确定该放在参数侧还是上下文侧，可以问自己：

1. 这条知识是不是经常变化？
2. 这条知识是否需要权限和审计？
3. 我是否需要给出处？
4. 我是否希望能立即更新，而不改模型？

如果这几条里大多是“是”，通常就更适合走上下文记忆，而不是纯依赖参数记忆。

## 结论收束

**参数记忆决定模型的底座能力，上下文记忆决定当前任务能否拿到正确、最新、可控的信息。**  
它们不是替代关系，而是分工关系。
