# Section Visual Systems

Use only the current article's section unless the article clearly bridges two sections.

## frameworks

Purpose:
- help readers compare frameworks, abstraction layers, workflow styles, and engineering tradeoffs

Visual direction:
- editorial technical infographic
- clean light backgrounds
- structured composition with obvious visual hierarchy
- mild 2.5D or polished flat illustration

Preferred image types:
- header-image
- comparison-image
- layered-concept-image

Color direction:
- blue, teal, amber, slate

Text policy:
- allow a few framework names or short anchor labels
- prefer Chinese labels for visible text, keeping only framework names and abbreviations in English
- never use large explanatory paragraphs

## rag

Purpose:
- explain retrieval pipelines, indexing flow, chunking, embedding, reranking, query flow, and failure points

Visual direction:
- system-flow style
- precise, technical, modular
- clearer directional motion than decorative atmosphere

Preferred image types:
- workflow-image
- layered-concept-image
- comparison-image

Color direction:
- teal, cyan, dark slate accents, muted orange for retrieval hotspots

Text policy:
- short stage labels only
- prefer Chinese stage labels, keeping only necessary technical abbreviations in English
- keep the image readable without dense captions

## llm

Purpose:
- explain model concepts, training/inference distinctions, capabilities, limitations, and system roles

Visual direction:
- conceptual but disciplined
- more abstract than `rag`
- layered, networked, or field-like scenes

Preferred image types:
- header-image
- layered-concept-image
- metaphor-image

Color direction:
- deep blue, indigo, cool cyan, restrained warm highlight

Text policy:
- use extremely sparse labels
- prefer Chinese labels when labels are visible
- prefer conceptual shapes over textual explanation

## principles

Purpose:
- explain underlying agent mechanics, memory, planning, tool use, runtime behavior, and control loops

Visual direction:
- analytical, systems-oriented, slightly abstract
- clean decomposition into modules or loops

Preferred image types:
- layered-concept-image
- workflow-image
- metaphor-image

Color direction:
- slate, blue, muted green, subtle amber accents

Text policy:
- short node labels are acceptable
- prefer Chinese labels when labels are visible
- keep everything diagram-like, not presentation-slide-like

## getting-started

Purpose:
- reduce fear and help new readers build a mental map quickly

Visual direction:
- approachable, welcoming, less dense
- brighter and simpler than `principles` or `llm`
- stronger emphasis on orientation and journey

Preferred image types:
- header-image
- workflow-image
- metaphor-image

Color direction:
- warm blue, mint, soft orange, light neutral background

Text policy:
- almost no text
- if text appears, use Chinese
- use familiar metaphors and clear visual progression

## tools

Purpose:
- show where a tool fits, what kind of work it supports, and how it compares to neighbors

Visual direction:
- product-landscape or toolkit-map feel
- structured, practical, less abstract than `llm`

Preferred image types:
- comparison-image
- header-image
- layered-concept-image

Color direction:
- blue, steel, teal, practical neutral palette

Text policy:
- a few product or capability labels are acceptable
- prefer Chinese labels, preserving product names in English where needed
- avoid dense capability lists inside the image

## tutorials

Purpose:
- support execution sequence, stage transitions, and build-up of a working result

Visual direction:
- procedural, progressive, concrete
- slightly more dynamic than `principles`

Preferred image types:
- workflow-image
- header-image
- comparison-image

Color direction:
- blue-green base with one energetic accent color

Text policy:
- only step anchors or tiny labels
- prefer Chinese labels
- no paragraph text

## interviews

Purpose:
- help readers understand categories of questions, answer structure, and evaluation perspective

Visual direction:
- sharp, focused, evaluative
- more card-like or conversational than deeply technical sections

Preferred image types:
- comparison-image
- metaphor-image
- header-image

Color direction:
- navy, slate, muted gold accents, clean light background

Text policy:
- a few short category labels only
- prefer Chinese labels
- avoid turning the image into a cheat sheet
